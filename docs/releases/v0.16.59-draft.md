# v0.16.59 发布草案

**目标版本：** `0.16.59`
**提交范围：** `v0.16.58..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue artifacts 的 `create-issues.sh` 支持 dry-run 预览。

## 用户价值

`v0.16.58` 已经让 artifacts `README.md` 和 `artifact-manifest.json.validation_commands` 展示同源验证命令。维护者能看到完整复核路径，但真正运行 `create-issues.sh` 前，仍需要手动打开脚本确认每条 `gh issue create` 命令。对于开源项目维护者来说，创建 issue 是有外部副作用的操作，预览能力应该是脚本本身的一等入口。

`v0.16.59` 给生成的 `create-issues.sh` 增加 `CLIANY_CREATE_ISSUES_DRY_RUN=1` dry-run mode。维护者可以运行 `CLIANY_CREATE_ISSUES_DRY_RUN=1 ./create-issues.sh`，只打印将要执行的 `gh issue create` 命令，不运行 publication preflight，也不创建 issue。正常执行路径仍然保留 `python scripts/check_release_publication.py --strict --json` preflight，避免最新本地 release 未公开时派发新任务。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.59 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `create-issues.sh` 包含 `CLIANY_CREATE_ISSUES_DRY_RUN`、`Dry run: publication preflight and gh issue create are not executed.` 和 `CLIANY_ISSUE_COMMANDS`；artifacts `README.md` 展示 `CLIANY_CREATE_ISSUES_DRY_RUN=1 ./create-issues.sh`；`artifact-manifest.json` 包含 `create_issues_dry_run_command`。

## 变更分组

### Candidate issue create script dry-run

- 生成的 `create-issues.sh` 支持 `CLIANY_CREATE_ISSUES_DRY_RUN=1`。
- Dry-run mode 只打印 `gh issue create` 命令，不运行 publication preflight，也不创建 issue。
- 正常执行路径继续先跑 `python scripts/check_release_publication.py --strict --json`。
- `artifact-manifest.json` 新增 `create_issues_dry_run_command`，方便工具展示预览命令。
- Artifacts `README.md` 在 `Create Issues` 小节展示 dry-run 命令。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 dry-run mode 与正常 preflight 路径的差异。
- `tests/test_plan_next_iteration.py` 覆盖生成脚本、manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让候选 issue 创建脚本可以安全预览外部副作用。

## 风险与兼容性

- 默认执行路径不变，仍然先跑 strict publication preflight。
- Dry-run mode 不执行 `gh issue create`，不访问 GitHub，不要求真实 LLM key。
- `create_issues_dry_run_command` 是新增 manifest 字段，不改变现有字段语义。

## Release Notes 摘要

`v0.16.59` adds a dry-run mode to generated candidate issue creation scripts, allowing maintainers to preview `gh issue create` commands before publication preflight or any GitHub side effects.
