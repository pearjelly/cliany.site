# v0.16.243 发布草案

**目标版本：** `0.16.243`
**提交范围：** `v0.16.242..HEAD`
**当前阻塞：** 本版需要完成 release readiness、聚焦测试、完整测试、CHANGELOG / 版本号同步，并在最终 readiness 通过后创建并推送 `v0.16.243` tag。

## 用户价值

`v0.16.242` 已经把发布节奏和远端可见性门禁收紧，但生成 adapter 的安全审计仍停留在独立 `audit.py` 模块和手动检查路径。维护者虽然可以事后审计某个 adapter，但 `save_adapter()` 在写出 `commands.py` 前没有自动阻断 critical finding；这会让 README 中“生成代码安全审计”的承诺依赖额外步骤，而不是成为默认生成边界。

`v0.16.243` 把 `audit_source()` 接到 adapter 持久化入口：所有通过 `save_adapter()` 落盘的生成代码都会先经过 AST 审计。只要发现 `eval()`、`exec()`、危险 subprocess / os 调用或语法错误等 critical finding，就抛出 `SecurityError` 并阻止 `commands.py` 写入。这样 explore、SDK、atom adapter 等共用保存链路都获得同一层默认保护。

## 变更分组

- Codegen security：`save_adapter()` 在创建 adapter 目录或写入临时文件前调用 `audit_source()`，critical finding 会以 `SecurityError` 中断生成。
- Regression test：`tests/test_generator_v3.py` 新增 `eval()` 生成代码阻断测试，确认危险 adapter 不会写出 `commands.py`。
- Release test stability：`tests/test_release_readiness.py` 的 fixture repo 使用 `git init -b master`，避免本机 `init.defaultBranch` 为 `main` 时破坏 release readiness 断言。

## 案例库映射

- 本次不修改 [cases/manifest.json](../../cases/manifest.json) 的案例状态；相关说明仍落在 [cases/README.md](../../cases/README.md)。
- 当前案例库仍保持 active 4 个、candidate 3 个、known gap 1 个。
- Known gap 保持 1 个：`search-extraction-gap`。
- Candidate 晋级仍需要 adapter package、metadata validation 和 online smoke 三段真实证据。
- 本版不生成 candidate adapter package，也不把任何 candidate 标记为 active。

## 风险与兼容性

- 这是生成代码保存边界的安全加固，不改变 CLI 参数、adapter metadata schema、action runtime、AXTree 语义匹配、session 存储或 marketplace package 格式。
- 之前能够写出的危险生成代码现在会被阻断；这是预期行为。正常模板和现有合法生成代码已由 codegen / security / full test suite 覆盖。
- 审计只阻断 critical finding；warning 级 import finding 仍由现有审计报告路径展示，不在本版改变策略。

## Release note draft

v0.16.243 makes generated adapter security audit part of the default save path. `save_adapter()` now blocks critical AST audit findings before `commands.py` is written, and release readiness fixture repos now pin their branch name so tests are stable across Git default-branch settings.
