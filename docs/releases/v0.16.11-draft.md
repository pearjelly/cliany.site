# v0.16.11 发布草案

**目标版本：** `0.16.11`
**提交范围：** `v0.16.10..HEAD`
**当前阻塞：** 需要提交 publication publish script artifact、release cadence 文档入口和测试；最新本地 release 仍需维护者确认后 push 到远端。

## 用户价值

`v0.16.10` 已经把 publication audit 的 `publish_commands` 带入 next iteration plan，维护者能在周初计划里看到 `git push origin master`、`git push origin v0.16.10` 和 `python scripts/check_release_publication.py --remote --json`。但发布交接时，维护者仍需要从 JSON 或 Markdown 里复制多行命令到终端。

`v0.16.11` 给 `scripts/check_release_publication.py` 增加 `--publish-script`，把同一组 `publish_commands` 写成一个可审阅的 shell 脚本，例如 `/tmp/cliany-publish-release.sh`。脚本包含 `set -euo pipefail`、审阅提示、branch push、tag push 和远端复核命令，并设置可执行权限。生成脚本不会自动运行，真实发布仍由维护者显式确认。

成功标准是：`python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh` 写出可执行脚本，脚本包含 `git push origin master`、`git push origin v0.16.11` 和 `python scripts/check_release_publication.py --remote --json`。

## 变更分组

### Publication publish script

- `scripts/check_release_publication.py` 新增 `--publish-script` 参数。
- 生成脚本复用 `publish_commands`，避免 JSON/report/script 三套命令漂移。
- 脚本写入调用者指定路径，并通过 `chmod 755` 设置可执行权限。
- 没有待发布命令时，脚本只输出 `No publish commands are needed.`。

### 文档与门禁

- [docs/release-cadence.md](../release-cadence.md) 说明 `--publish-script /tmp/cliany-publish-release.sh` 的用途和边界。
- `tests/test_release_publication.py` 固定脚本内容、权限和 CLI 入口。
- `tests/test_release_cadence_docs.py` 固定 release cadence 文档入口。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例数量和状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本聚焦 release publication handoff，不访问第三方业务站点，不生成 adapter 包，也不会修改案例 manifest 或离线样例。

## 风险与兼容性

- `--publish-script` 是新增 opt-in 参数；默认 publication audit 行为不变。
- 生成脚本不会自动执行 `git push`，维护者必须显式运行脚本。
- 默认模式仍不访问网络；脚本里的远端复核命令只有在维护者运行脚本时才执行。
- 脚本复用 shell-quoted `publish_commands`，避免 branch、tag 或 remote 名称导致命令不可复制。

## 发版前验证

```bash
pytest tests/test_release_publication.py tests/test_release_cadence_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/check_release_publication.py tests/test_release_publication.py tests/test_release_cadence_docs.py tests/test_release_draft_docs.py
python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh
python scripts/release_readiness.py --target-version 0.16.11 --json
```

## 发版步骤

1. 确认 `python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh` 写出可执行脚本。
2. 确认脚本包含 branch push、tag push 和远端复核命令。
3. 运行 publication、release cadence 文档和 release draft 相关测试。
4. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.11`。
5. 将 `pyproject.toml` 版本更新为 `0.16.11`。
6. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.11`。
7. 创建 tag：`git tag v0.16.11`。

## Release Notes 摘要

`v0.16.11` adds `--publish-script` to the release publication audit. Maintainers can now generate a reviewable shell script from the same `publish_commands` used by JSON and Markdown reports, then explicitly run it when they are ready to publish branch and tag refs.
