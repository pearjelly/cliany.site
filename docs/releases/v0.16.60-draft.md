# v0.16.60 发布草案

**目标版本：** `0.16.60`
**提交范围：** `v0.16.59..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue artifacts 用结构化 `create_issues_safety` 描述创建脚本的安全行为。

## 用户价值

`v0.16.59` 已经让生成的 `create-issues.sh` 支持 `CLIANY_CREATE_ISSUES_DRY_RUN=1` dry-run mode。维护者可以安全预览 `gh issue create` 命令，但自动化如果要判断脚本的安全属性，仍只能解析 README 或脚本文本。

`v0.16.60` 在 `artifact-manifest.json` 中新增 `create_issues_safety`，结构化记录 `dry_run_supported`、`dry_run_env`、`dry_run_command`、`preflight_required`、`preflight_command` 和 `preflight_json`。Artifacts `README.md` 也新增 `Create Issues Safety` 小节，向人类维护者展示同源字段。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.60 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json` 包含 `create_issues_safety`；README 包含 `Create Issues Safety`、`dry_run_supported`、`preflight_required` 和 `preflight_json`；`create_issues_dry_run_command` 继续保留，兼容已有工具。

## 变更分组

### Candidate issue create script safety metadata

- `artifact-manifest.json` 新增 `create_issues_safety`。
- `create_issues_safety` 记录 dry-run、preflight 和 preflight JSON 路径。
- Artifacts `README.md` 新增 `Create Issues Safety` 小节。
- `create_issues_dry_run_command` 继续保留，避免破坏已有读取路径。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `create_issues_safety` 与 README 安全字段。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让候选 issue artifacts 的安全语义可由工具读取。

## 风险与兼容性

- `create_issues_safety` 是新增 manifest 字段，不改变 `create_issues_dry_run_command` 或现有文件名。
- README 新增安全字段只用于人工审阅，不自动执行命令。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
