# v0.15.2 发布草案

**目标版本：** `0.15.2`
**提交范围：** `v0.15.1..HEAD`
**当前阻塞：** 需要提交发布可见性检查脚本、文档入口和 release readiness 元数据 gate。

## 用户价值

`v0.15.2` 聚焦发布可见性：维护者可以在本地完成版本号、CHANGELOG、tag、release preflight 和构建检查，但如果 branch 或 tag 没有推送，外部用户仍然看不到 GitHub Release 或 PyPI 版本。这个 patch 增加一个独立的 publication audit，让维护者在触发真实发布前后都能看到“本地 release 是否已经公开可见”。

成功标准是：运行 `python scripts/check_release_publication.py --json` 时，报告应展示当前分支相对 upstream 的 ahead/behind、最新 tag、tag 是否指向 HEAD、tag commit 是否进入 upstream，以及可执行的 `next_actions`；运行 `python scripts/check_release_publication.py --remote --json` 时，应进一步检查实时远端 branch/tag refs。

## 变更分组

### 发布可见性检查

- 新增 `scripts/check_release_publication.py`，默认只读取本地 git/upstream 状态，不访问网络。
- `--remote` 模式使用 `git ls-remote` 检查远端 branch 和 tag refs，适合发布前后确认真实可见性。
- JSON 和默认文本输出都包含 `next_actions`，例如提示 push branch、push tag、设置 upstream 或处理 behind 状态。
- `--strict` 可在 publication check 未通过时返回非零状态，便于维护者在手动发布 checklist 中使用。

### 文档与门禁

- [docs/release-cadence.md](../release-cadence.md) 增加 publication audit 命令和使用场景。
- [docs/contributor-starter.md](../contributor-starter.md) 把 release publication 脚本纳入发布脚本贡献路径。
- `release_readiness.py` 的项目元数据 gate 应检查 release cadence 文档保留 publication audit 入口，避免下一轮维护时丢失。

## 案例库映射

本版本不改变案例库内容，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。`suitecrm-accounts`、`apache-jira-issues`、`apache-confluence-search` 和 `apache-jenkins-jobs` 仍是 active；`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是维护者脚本和文档层面的 patch，不改变 CLI runtime、adapter schema 或 JSON envelope。
- 默认模式不访问网络，只依赖本地 upstream tracking refs；远端实时检查必须显式传入 `--remote`。
- `--remote` 只读取 git refs，不发布、不推送、不创建 GitHub Release，也不访问 PyPI API。

## 发版前验证

```bash
python scripts/check_release_publication.py --json
pytest tests/test_release_publication.py tests/test_release_readiness.py tests/test_contributor_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/check_release_publication.py scripts/release_readiness.py tests/test_release_publication.py tests/test_release_readiness.py tests/test_contributor_docs.py tests/test_release_draft_docs.py
python scripts/release_readiness.py --target-version 0.15.2 --json
```

如需要发布前实时确认远端 refs：

```bash
python scripts/check_release_publication.py --remote --json
```

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.15.2 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行 `python scripts/check_release_publication.py --json`，确认本地最新 tag 和 upstream 状态。
3. 运行相关 pytest 与 ruff 检查。
4. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.15.2`。
5. 将 `pyproject.toml` 版本更新为 `0.15.2`。
6. 运行 `python scripts/release_readiness.py --strict --target-version 0.15.2`。
7. 创建 tag：`git tag v0.15.2`。

## Release Notes 摘要

`v0.15.2` adds a release publication audit. Maintainers can now check whether the latest local release commit and tag are visible from upstream, optionally verify live remote refs with `--remote`, and get concrete `next_actions` before triggering the real GitHub Release/PyPI flow.
