# v0.16.210 发布草案

**目标版本：** `0.16.210`
**提交范围：** `v0.16.209..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 review checklist preview 与 tail 显式化到整包摘要。

## 用户价值

`v0.16.209` 已经让 `artifact_bundle_summary` 可以展示 validation commands 的前后窗口。维护者在审阅 candidate issue artifacts 时，还需要快速确认 review checklist 的前后步骤是否符合预期；目前 summary 已有数量、完整 hash 和首尾边界，但只读 summary 的工具仍无法在审阅清单变长时看到前后窗口。

`v0.16.210` 在 `artifact_bundle_summary` 中新增 review checklist preview/tail 字段。这样维护者和自动化可以只读整包摘要，先检查审阅清单的前后窗口、数量、完整 hash 和首尾边界，再决定是否展开 artifacts README 的完整审阅清单。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.210 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 review checklist preview/tail fields，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且 preview/tail 等于 review checklist 的前 8 项和后 8 项。

## 变更分组

### Review checklist window summary

- `artifact_bundle_summary` 新增 `review_checklist_preview_count`。
- `artifact_bundle_summary` 新增 `review_checklist_preview`。
- `artifact_bundle_summary` 新增 `review_checklist_preview_sha256`。
- `artifact_bundle_summary` 新增 `review_checklist_tail_count`。
- `artifact_bundle_summary` 新增 `review_checklist_tail`。
- `artifact_bundle_summary` 新增 `review_checklist_tail_sha256`。
- Preview 从 review checklist 的前 8 项派生；tail 从后 8 项派生。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示 review checklist preview/tail fields。
- 现有 `review_checklist_count`、`review_checklist_sha256`、`review_checklist_first_item`、`review_checklist_last_item`、`review_checklist_boundary_sha256`、validation command windows、release draft issue/action windows、publication next action boundary 和 publication handoff boundary 字段保持兼容。
- 不改变 review checklist 内容、candidate issue gate、release readiness、publication audit、publish commands 内容或 candidate case 状态。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示 review checklist preview/tail fields。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的 review checklist preview/tail 字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的整包摘要可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有 review checklist，不新增网络访问或外部状态读取。
- 旧工具忽略未知 summary 字段即可继续读取现有 manifest。
- 新工具可以只读 `review_checklist_preview_count`、`review_checklist_preview`、`review_checklist_preview_sha256`、`review_checklist_tail_count`、`review_checklist_tail` 和 `review_checklist_tail_sha256` 快速确认 review checklist 窗口是否漂移。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.210..HEAD`。最终带 `--release-tag v0.16.210` 的 strict gate 仍以本草案顶部的 `v0.16.209..HEAD` 作为实际 release diff 范围。

```bash
python scripts/plan_next_iteration.py --target-version 0.16.210 --json
python scripts/plan_next_iteration.py --target-version 0.16.210 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.210 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.210 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 review checklist preview/tail fields。
3. 确认 artifacts README 的 `Artifact Bundle Summary` 展示这些字段。
4. 确认 preview/tail 等于 review checklist 的前 8 项和后 8 项。
5. 确认 `review_checklist_sha256` 和完整 review checklist 仍保持同源。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.210`。
8. 将 `pyproject.toml` 版本更新为 `0.16.210`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.210`。
10. 创建 tag：`git tag v0.16.210`。

## Release Notes 摘要

`v0.16.210` adds preview and tail windows for the candidate artifact review checklist to the artifact bundle summary, helping tools inspect review drift before expanding the full README.
