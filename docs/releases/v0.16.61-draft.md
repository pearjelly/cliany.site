# v0.16.61 发布草案

**目标版本：** `0.16.61`
**提交范围：** `v0.16.60..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue artifacts 提供 `issue_body_inventory`，方便审计 body 文件完整性。

## 用户价值

`v0.16.60` 已经让 candidate issue artifacts 的创建脚本安全语义可由工具读取。维护者现在能安全预览命令，也能判断 preflight 是否存在；但 artifacts 包中仍缺少一个轻量的 body 文件清单，工具如果想确认 `issue-metadata.json`、README 和实际 body 文件一致，需要重新扫描目录并自己计算 hash。

`v0.16.61` 在 `artifact-manifest.json` 中新增 `issue_body_inventory`。它为每个 candidate body 记录 `case_id`、`issue_body_name`、`byte_count` 和 `sha256`。Artifacts `README.md` 也新增 `Issue Body Inventory` 表，让维护者能直接看到每个 body 的字节数和 SHA-256。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.61 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json` 包含 `issue_body_inventory`；README 包含 `Issue Body Inventory`、`byte_count` 和 `SHA-256`；生成的 hash 与实际 body 文件内容一致。

## 变更分组

### Candidate issue body inventory

- `artifact-manifest.json` 新增 `issue_body_inventory`。
- 每条 inventory 记录包含 `case_id`、`issue_body_name`、`byte_count` 和 `sha256`。
- Artifacts `README.md` 新增 `Issue Body Inventory` 表。
- Inventory 不包含本机绝对路径，避免泄漏维护者环境。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `issue_body_inventory` 与 README 表。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README，并用实际 body 文件计算 SHA-256。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让候选 issue artifacts 的 body 文件完整性可审计。

## 风险与兼容性

- `issue_body_inventory` 是新增 manifest 字段，不改变 `issue-metadata.json` 或 body 文件内容。
- SHA-256 基于生成的 UTF-8 body 文件内容，便于本地复核。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.61 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json` 包含 `issue_body_inventory`。
3. 确认 `issue_body_inventory` 中的 `sha256` 与实际 body 文件一致。
4. 确认 artifacts `README.md` 包含 `Issue Body Inventory`。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.61`。
7. 将 `pyproject.toml` 版本更新为 `0.16.61`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.61`。
9. 创建 tag：`git tag v0.16.61`。

## Release Notes 摘要

`v0.16.61` adds issue body inventory metadata to candidate issue artifacts, recording body file sizes and SHA-256 hashes for easier review and automation.
