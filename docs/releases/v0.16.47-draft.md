# v0.16.47 发布草案

**目标版本：** `0.16.47`
**提交范围：** `v0.16.46..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue artifacts manifest 直接声明自身 schema version。

## 用户价值

`v0.16.46` 已经让 `artifact-manifest.json` 包含复现命令。维护者或工具可以从一个机器可读入口审阅 candidate issue artifacts，也可以重新生成同一批 artifacts。但随着 manifest 持续承载 publication、release draft、复现和验证信息，自动化脚本需要一个稳定字段先判断 manifest 格式版本。

`v0.16.47` 让 `artifact-manifest.json` 直接包含 `schema_version`。这让维护脚本在读取候选 issue artifacts 前可以先判断字段语义是否兼容，再决定是否继续解析 candidate cases、publication handoff、release draft handoff 和 validation commands。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.47 --issues-dir /tmp/cliany-candidate-issues --json` 写出的 `artifact-manifest.json` 包含 `schema_version: 1`；artifacts `README.md` 的 Files 小节说明 manifest 包含 schema version。

## 变更分组

### Candidate artifacts manifest schema version

- `artifact-manifest.json` 新增 `schema_version`，当前值为 `1`。
- Artifacts `README.md` 的 Files 小节说明 manifest 包含 schema version。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `schema_version` 是 artifacts manifest 自身的格式版本。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 内容和 README 文件列表。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强候选 issue artifacts manifest 的格式版本声明。

## 风险与兼容性

- `schema_version` 是 `artifact-manifest.json` 的新增字段，不改变 plan JSON schema。
- 既有 publication、release draft、candidate、files、review order、reproduction command 和 validation commands 字段保持兼容。
- 不自动执行命令，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- 当前 manifest schema version 固定为 `1`；后续只有破坏性改变 manifest 字段语义时才应提升。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.47 --issues-dir /tmp/cliany-candidate-issues --json` 写出 `artifact-manifest.json`。
2. 确认 manifest 包含 `schema_version: 1`。
3. 确认 artifacts `README.md` 的 Files 小节说明 manifest 包含 schema version。
4. 运行计划器、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.47`。
6. 将 `pyproject.toml` 版本更新为 `0.16.47`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.47`。
8. 创建 tag：`git tag v0.16.47`。

## Release Notes 摘要

`v0.16.47` adds a schema version to generated `artifact-manifest.json` files, giving maintainer tooling a stable way to identify candidate issue artifact manifest semantics before parsing.
