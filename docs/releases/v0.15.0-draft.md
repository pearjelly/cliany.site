# v0.15.0 发布草案

**状态：** draft  
**目标版本：** `0.15.0`  
**预计类型：** minor  
**提交范围：** `v0.14.4..HEAD`  
**当前阻塞：** 需要完成 10 分钟成功路径的最短闭环验证，并在发版前运行 `release_readiness.py --target-version 0.15.0 --strict`。

## 用户价值

`v0.15.0` 聚焦新用户首次成功路径：让用户不需要先理解 provider、CDP、LLM、adapter 生命周期和案例库细节，就能在 10 分钟内完成环境检查、安装一个真实 demo adapter、验证 adapter、执行只读命令，并知道下一步是继续生成自己的命令还是贡献新的公开只读案例。

成功标准不是“文档更完整”，而是首次用户可以按一条最短路径得到可解释的 JSON envelope，并且失败时能从 `doctor`、`verify`、`release_readiness` 或案例库报告中看到明确下一步。

## 变更分组

### 10 分钟成功路径

- README、README.zh 和官网首屏继续优先指向 [docs/quickstart-10min.md](../quickstart-10min.md)。
- quickstart 要把“安装 -> doctor -> demo adapter install -> list -> verify -> 只读命令”压成一条主路径。
- quickstart 中的分叉只保留必要场景：无 LLM key、无 Chrome/CDP、第三方 demo 站点不可用、命令成功但业务数据为空。
- 首次成功后的贡献路径继续指向 `Real Demo Case Proposal`、`cases/manifest.json`、`cases/examples/` 和 `python scripts/validate_cases.py --strict`。

### doctor 与可执行下一步

- `doctor` human 输出继续展示 `下一步`，JSON 输出继续提供 `data.summary.recommended_next_step`。
- `data.summary.capabilities` 必须清楚区分 `manage_adapters`、`run_browser_workflows` 和 `generate_adapters` 三条路径。
- 无 LLM key 时不应阻断 demo adapter 路径；无 Chrome/CDP 时应提示哪些命令仍可先跑。

### demo adapter 验证

- 至少一个 active demo adapter 必须能按文档完成 install/list/verify 的静态路径。
- 如果本地有 release asset 包，发版前运行 `python scripts/release_readiness.py --target-version 0.15.0 --packages-dir ~/.cliany-site/packages --require-packages --strict`。
- 第三方 demo 站点在线 smoke 仍是手动或受保护 workflow，不进入默认 PR 门禁。

## 案例库映射

本版本继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。`suitecrm-accounts`、`apache-jira-issues`、`apache-confluence-search` 和 `apache-jenkins-jobs` 是 active demo adapter 场景；`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 是 candidate 工作流；`search-extraction-gap` 继续记录搜索/列表抽取质量边界。

## 风险与兼容性

- 本版本目标是首次成功路径和诊断体验，不应引入 adapter schema、JSON envelope 或 CLI 参数的破坏性变更。
- 真实 demo 站点可能临时不可用；默认验证优先使用离线案例库、adapter package metadata 和 `verify`。
- 不把真实 LLM key 或第三方在线 smoke 放进 fork PR 默认路径。
- 不新增脆弱 CSS selector 兜底；仍以 AXTree 语义匹配和现有 selector map 语义为边界。

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.15.0 --json` 只剩可解释的节奏或包资产阻塞。
2. 完成 quickstart/doctor/官网的最短路径改动与测试。
3. 将 `CHANGELOG.md` 的 Unreleased 内容移动到 `## [0.15.0] - <date>`。
4. 将 `pyproject.toml` 版本号改为 `0.15.0`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.15.0`。
6. 创建 tag：`git tag v0.15.0`。
7. 推送 `master` 和 tag，触发 release workflow。

## Release Notes 摘要

v0.15.0 should make the first successful run shorter and more deterministic: users start with `doctor`, install and verify a real demo adapter, execute a read-only command, then decide whether to configure an LLM for custom adapter generation or contribute another public read-only case.
