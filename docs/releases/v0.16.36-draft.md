# v0.16.36 发布草案

**目标版本：** `0.16.36`
**提交范围：** `v0.16.35..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 next iteration plan 和 candidate issue handoff 直接给出 publication visibility 结论。

## 用户价值

`v0.16.35` 已经把 publication audit 的 ref context 带到 `scripts/plan_next_iteration.py`、Markdown plan、`publication-handoff.json` 和 candidate artifacts `README.md`。但维护者仍需要同时阅读 `publication_ok`、worktree、ahead/tag 和 ref context，才能判断当前状态到底是已发布、只在本地、tag 不可见，还是需要远端复核。

`v0.16.36` 新增 `publication_visibility`，把这些 publication audit 信号归纳成一个稳定结论字段。计划 JSON、文本输出、Markdown report、`publication-handoff.json` 和 artifacts `README.md` 会展示 `published`、`local_only`、`dirty_worktree`、`tag_not_visible` 或 `needs_remote_check`，并附带 summary，帮助维护者在周初计划和候选 issue handoff 中直接判断是否应该先发布最新本地 release。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.36 --json` 输出 `publication_visibility`；Markdown plan 包含 `Publication Visibility`；`--issues-dir` 生成的 `publication-handoff.json` 包含 `visibility`，artifacts `README.md` 的 `Publication Handoff` 展示 visibility。

## 变更分组

### Next iteration publication visibility

- `scripts/plan_next_iteration.py` 的 `IterationPlan` 新增 `publication_visibility`。
- JSON、文本和 Markdown plan 展示 publication visibility status 与 summary。
- `--issues-dir` 生成的 `publication-handoff.json` 新增 `visibility`，artifacts `README.md` 在 `Publication Handoff` 中展示 visibility。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明计划器会展示 publication visibility。
- `tests/test_plan_next_iteration.py` 覆盖 JSON、Markdown、publication handoff 和 artifacts README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强 next iteration plan 和候选 issue artifacts 的发布可见性结论。

## 风险与兼容性

- `publication_visibility` 是 additive plan JSON 字段，不移除现有 key。
- `publication-handoff.json` 新增 `visibility`，已有 publication/worktree/ref 字段保持兼容。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- visibility summary 只归纳 `scripts/check_release_publication.py` 的本地 audit 结果；实时远端判断仍由维护者显式运行 `--remote`。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.36 --json` 输出 `publication_visibility`。
2. 确认 Markdown plan 包含 `Publication Visibility`。
3. 确认 `publication-handoff.json` 包含 `visibility`。
4. 运行计划器、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.36`。
6. 将 `pyproject.toml` 版本更新为 `0.16.36`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.36`。
8. 创建 tag：`git tag v0.16.36`。

## Release Notes 摘要

`v0.16.36` adds a publication visibility summary to next-iteration plans and candidate issue handoff artifacts, so maintainers can see whether the latest local release is published, local-only, blocked by dirty worktree state, missing a visible tag, or waiting for a remote check.
