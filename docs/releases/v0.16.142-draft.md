# v0.16.142 发布草案

**目标版本：** `0.16.142`
**提交范围：** `v0.16.141..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 release draft required actions 的摘要补进 artifact bundle summary。

## 用户价值

`v0.16.141` 已经让 `release-draft-handoff.json` 和 artifacts `README.md` 的 `Release Draft Handoff` 展示完整 release draft 修复动作列表。维护者和自动化如果只想快速判断动作列表是否变化，仍需要展开完整 handoff。

`v0.16.142` 在 `artifact_bundle_summary` 中新增 `release_draft_required_action_count` 和 `release_draft_required_actions_sha256`，并在 artifacts `README.md` 的 `Artifact Bundle Summary` 段落展示同源字段。这样工具只读 manifest summary 或 README summary，就能判断 release draft required actions 是否需要展开。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.142 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 required actions 数量和 hash，artifacts README 的 `Artifact Bundle Summary` 展示同一组字段，且这些值与 `release-draft-handoff.json` 的 `release_draft_required_actions` 同源。

## 变更分组

### Artifact bundle release draft required actions summary

- `artifact_bundle_summary` 新增 `release_draft_required_action_count`。
- `artifact_bundle_summary` 新增 `release_draft_required_actions_sha256`。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示 required actions 数量和 hash。
- 摘要值完全派生自 `release_draft_required_actions`，不重复推断 release draft issue 语义。
- 现有 `release_draft_handoff_key_count`、`release_draft_handoff_sha256`、`release_draft_path_sha256`、`release_draft_issues_sha256` 和 `Release Draft Handoff` 保持兼容。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示 release draft required actions 的数量和 hash。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的新字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 整包摘要的 release draft required actions 可见性。

## 风险与兼容性

- 新字段只派生自既有 `release_draft_required_actions`，不新增网络访问或外部状态读取。
- 无 release draft issue 时，count 为 `0`，hash 基于空列表稳定计算。
- Summary 只用于快速判断是否展开完整 handoff，不替代 `release-draft-handoff.json` 审阅。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.142 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `release_draft_required_action_count`。
3. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `release_draft_required_actions_sha256`。
4. 确认 artifacts README 的 `Artifact Bundle Summary` 展示 required actions 数量和 hash。
5. 确认 summary hash 与 `release-draft-handoff.json.release_draft_required_actions` 稳定序列化结果一致。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.142`。
8. 将 `pyproject.toml` 版本更新为 `0.16.142`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.142`。
10. 创建 tag：`git tag v0.16.142`。

## Release Notes 摘要

`v0.16.142` adds release draft required action count and hash to artifact bundle summaries, helping tools detect draft action drift before expanding the full release draft handoff.
