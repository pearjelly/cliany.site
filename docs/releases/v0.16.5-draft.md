# v0.16.5 发布草案

**目标版本：** `0.16.5`
**提交范围：** `v0.16.4..HEAD`
**当前阻塞：** 需要提交计划器 issue metadata、维护者循环入口和测试；最新本地 release 仍需维护者确认后 push 到远端。

## 用户价值

`v0.16.4` 已经让 `scripts/plan_next_iteration.py` 为 candidate cases 输出可复制 `issue_body`。维护者创建 GitHub issue 时仍需要手动补标题和标签，容易造成同一批候选案例 issue 命名不一致，或者遗漏 `case-proposal` / `good first issue` 这样的入口标签。

`v0.16.5` 把 issue title 和 labels 也纳入计划器输出。周初运行 `python scripts/plan_next_iteration.py --report /tmp/cliany-next-iteration.md` 后，维护者可以从 `Candidate Issue Metadata` 表复制标题和标签，再从 `Candidate Issue Body Templates` 复制正文，让 candidate 晋级任务更容易被拆成一致的 GitHub issue。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.5 --json` 的 `candidate_promotions` 包含 `issue_title` 和 `issue_labels`；Markdown report 输出 `Candidate Issue Metadata` 表，列出每个 candidate 的 title 和 labels。

## 变更分组

### Candidate issue metadata

- `CandidatePromotion` 新增 `issue_title` 字段。
- `CandidatePromotion` 新增 `issue_labels` 字段，默认包含 `case-proposal` 和 `good first issue`。
- JSON 输出保留 issue body，并同时输出 title / labels。
- Markdown report 新增 `Candidate Issue Metadata` 表。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明计划器报告会生成 issue title、labels 和 body。
- `tests/test_plan_next_iteration.py` 固定 JSON 和 Markdown report 中的 issue metadata。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环里的新说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例数量和状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。计划器会为 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 三个 candidate cases 输出 issue title、labels 和 body。

`search-extraction-gap` 继续作为抽取质量 known-gap。本版本只读取 manifest 和 readiness 里的现有证据，不访问第三方站点，不执行真实 adapter 包构建。

## 风险与兼容性

- 这是维护脚本和报告输出 patch，不改变 CLI runtime、adapter schema、JSON envelope 或生成代码。
- `issue_title` 和 `issue_labels` 是新增 JSON 字段；已有 `candidate_cases`、`candidate_promotions`、`issue_body`、`next_actions` 和 `validation_commands` 保持兼容。
- 默认验证继续离线运行，不需要真实 LLM key，也不会写入 `~/.cliany-site/`。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.5 --json` 输出 `candidate_promotions[].issue_title` 和 `candidate_promotions[].issue_labels`。
2. 运行计划脚本、维护者循环文档和 release draft 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.5`。
4. 将 `pyproject.toml` 版本更新为 `0.16.5`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.5`。
6. 创建 tag：`git tag v0.16.5`。

## Release Notes 摘要

`v0.16.5` adds candidate issue titles and labels to the next-iteration planner. Maintainers can now create consistently named and labeled candidate promotion issues from the JSON or Markdown planning report.
