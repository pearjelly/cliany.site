# v0.16.276

Candidate promotion issues now make the required live-LLM preflight the first executable step.

## What Changed

- Generated candidate issue templates now show `Next executable step` and `Next executable command` in their Primary Evidence Task.
- For `adapter_package`, that command is `cliany-site doctor --llm-live --json`; the subsequent explore command is clearly labeled as the task command after preflight.
- Issue-template JSON now preserves both `next_*` and `task_*` fields, including command source, missing state, and handoff.
- Metadata validation and online smoke retain their direct commands after an adapter package is complete.

## Compatibility

Consumers should run `next_command` first. Use `task_command` only after any declared prerequisite has passed.

## Trust Boundaries

This release does not run a live LLM request, create a candidate adapter archive, or claim a third-party online result. Candidate promotion remains blocked until real evidence is attached.

## Verification

Regression coverage verifies the Primary Evidence Task ordering, JSON handoff fields, direct metadata-validation behavior, and the candidate catalog. No live LLM, adapter package, or online smoke success is claimed.
