# v0.16.139 发布草案

**目标版本：** `0.16.139`
**提交范围：** `v0.16.138..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 schema version 补进 release draft handoff。

## 用户价值

`v0.16.138` 已经让 `release-draft-handoff.json` 能直接展示第一条 release draft 阻塞原因。维护者和自动化只读 handoff 时，仍缺少一个明确的格式版本字段来判断当前 handoff 的字段语义是否符合解析预期。

`v0.16.139` 在 `release-draft-handoff.json` 中新增 `schema_version`，并在 artifacts `README.md` 的 `Release Draft Handoff` 段落展示同一字段。维护者和自动化只读 handoff 文件或 README handoff 小节，就能先确认 release draft handoff 格式版本，再解析后续状态和摘要字段。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.139 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `release-draft-handoff.json` 包含 `schema_version`，artifacts README 的 `Release Draft Handoff` 展示同一字段，`artifact_bundle_summary` 中的 `release_draft_handoff_key_count` 和 `release_draft_handoff_sha256` 随新增字段稳定更新。

## 变更分组

### Release draft handoff schema version

- `release-draft-handoff.json` 新增 `schema_version`，当前值为 `1`。
- Artifacts `README.md` 的 `Release Draft Handoff` 段落展示 `schema_version`。
- 现有 `release_draft_ok`、`release_draft_issue_count`、`release_draft_primary_issue`、`release_draft_path_sha256`、`release_draft_issues_sha256` 和 `artifact_bundle_summary` 保持兼容。
- `release_draft_handoff_key_count` 和 `release_draft_handoff_sha256` 会随 handoff 结构变化稳定更新。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `release-draft-handoff.json` 和 README handoff 会展示 `schema_version`。
- `tests/test_plan_next_iteration.py` 覆盖 handoff JSON、README handoff 和 bundle summary 的新字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 release draft handoff 的格式版本可见性。

## 风险与兼容性

- 新字段固定为整数 `1`，用于标识 release draft handoff 当前字段语义。
- 现有 release draft 状态、路径、issue 摘要和 `artifact_bundle_summary` 保持兼容。
- schema version 不代表 release draft 内容版本，也不替代完整草案审阅。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.139 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `release-draft-handoff.json` 包含 `schema_version`。
3. 确认 artifacts README 的 `Release Draft Handoff` 展示 `schema_version`。
4. 确认 `schema_version` 为整数 `1`。
5. 确认 `artifact_bundle_summary` 的 `release_draft_handoff_key_count` 和 `release_draft_handoff_sha256` 已随新字段更新。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.139`。
8. 将 `pyproject.toml` 版本更新为 `0.16.139`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.139`。
10. 创建 tag：`git tag v0.16.139`。
