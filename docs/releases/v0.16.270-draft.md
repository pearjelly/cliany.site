# v0.16.270 发布草案

**目标版本：** `0.16.270`
**提交范围：** `v0.16.269..HEAD`
**当前状态：** 2026-07-22 release slice；为合法零匹配补齐生成 adapter 的质量语义。

完成 version bump 后，target-mode readiness 还会检查占位锚点 **提交范围：** `v0.16.270..HEAD`；该锚点不改变实际 release diff。

## 用户价值

生成的 `list-` 和 `search-` adapter 命令现在可以声明 `expects_nonempty=false`。当工作流的业务目标是确认某项是否不存在时，零匹配会保留 `data.quality` 的空结果证据，同时返回 `ok=true`；默认仍为 `true`，因此已有自动化不会改变对空结果的失败判断。

## 案例库映射

本版不改变案例状态或 adapter 资产。`cases/README.md` 和 `cases/manifest.json` 仍是案例目录的唯一来源；`search-extraction-gap` 与 `llm_live_preflight_not_ready` 继续约束 candidate 晋级。2026-07-22 的真实 LLM preflight 仍返回可重试的 `E_LLM_UNAVAILABLE`，因此没有生成或声明任何 candidate adapter package、online smoke 或 live explore 成功。

## 风险与兼容性

- `expects_nonempty` 是生成 command metadata 和 JSON envelope 的新增布尔字段，默认 `true`。
- `expects_nonempty=false` 只将 `quality.status=empty` 视为合法业务结果；部分字段缺失、混合行形状或其他 `partial` quality 仍返回 `E_EMPTY_RESULT`。
- 已生成 adapter 不会被静默改写；需要新的期望时，重新运行通过 live preflight 的 `explore` 生成新 adapter。
