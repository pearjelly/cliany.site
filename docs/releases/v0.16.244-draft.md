# v0.16.244 发布草案

**目标版本：** `0.16.244`
**提交范围：** `v0.16.243..HEAD`
**当前阻塞：** 本版需要完成 release readiness、完整测试、CHANGELOG / 版本号同步，并在最终 readiness 通过后创建并推送 `v0.16.244` tag。

## 用户价值

`v0.16.243` 发布后，主分支上的 CI 已经暴露出三类发布链路问题：`ruff check src/`、`mypy src/cliany_site/` 以及普通 pytest job 都会失败。前两类是历史 lint/type debt 被现有门禁重新捕获；第三类是 embodied 测试在 module import 阶段直接导入 Playwright，使没有安装 Playwright 的普通 pytest matrix 在 collection 阶段失败。

`v0.16.244` 把这次 CI 修复收敛成一个 patch：源码通过当前 ruff / mypy gate，普通 pytest 环境会在 Playwright 不存在时跳过 embodied 测试模块，而 dedicated embodied workflow 会显式安装 Playwright Python package 再安装 Chromium。这样主 CI 和 embodied CI 的职责重新分开：普通 matrix 验证离线核心能力，embodied workflow 验证 headless Chromium + CDP + AXTree 集成。

## 变更分组

- CI quality gate：`src/` 通过当前 ruff 规则，主要是导入排序、PEP 585/604 类型写法、长行拆分和局部简化。
- Typecheck gate：补齐 envelope、JSON、cache、browser command、release/case command 等路径的类型收窄，使 `mypy src/cliany_site/` 通过。
- Test collection：`tests/embodied/test_explore_embodied.py` 使用 `pytest.importorskip("playwright.async_api")`，避免普通 pytest matrix 因可选依赖缺失在 collection 阶段失败。
- Embodied workflow：`.github/workflows/embodied-ci.yml` 显式安装 `playwright`，并使用 setup-python 提供的 `python -m playwright install --with-deps chromium`。

## 案例库映射

- 本次不修改 [cases/manifest.json](../../cases/manifest.json) 的案例状态；相关说明仍落在 [cases/README.md](../../cases/README.md)。
- 当前案例库仍保持 active 4 个、candidate 3 个、known gap 1 个。
- Known gap 保持 1 个：`search-extraction-gap`。
- Candidate 晋级仍需要 adapter package、metadata validation 和 online smoke 三段真实证据。
- 本版不生成 candidate adapter package，也不把任何 candidate 标记为 active。

## 风险与兼容性

- 这是 CI / type hygiene / optional test dependency 的修复，不改变 CLI 参数、adapter metadata schema、action runtime 语义、AXTree 语义匹配、session 存储或 marketplace package 格式。
- ruff 自动修复主要是语法等价整理；少量手工类型修复通过 full test suite 覆盖。
- 普通 pytest job 会跳过未安装 Playwright 的 embodied 测试；专用 embodied workflow 仍会安装 Playwright 并真实运行 headless Chromium 集成测试。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.244..HEAD`。最终带 `--release-tag v0.16.244` 的 strict gate 仍以本草案顶部的 `v0.16.243..HEAD` 作为实际 release diff 范围。

```bash
ruff check src/
mypy src/cliany_site/
pytest tests/embodied/ -m embodied -v --no-cov
python scripts/validate_cases.py --strict
pytest tests/ -q
git diff --check
```

正式 tag 前再运行：

```bash
pytest tests/ -q
python scripts/release_readiness.py --strict --release-tag v0.16.244
```
