# v0.14.4 发布草案

**状态：** draft  
**目标版本：** `0.14.4`  
**预计类型：** patch  
**提交范围：** `v0.14.3..HEAD`  
**当前阻塞：** 本周提交日尚未达到 3 天；满足节奏后再移动 CHANGELOG、bump version、打 tag。

## 用户价值

`v0.14.4` 聚焦 adapter 分发可靠性、真实案例库可验证性和抽取结果的任务级反馈。它不改变用户的核心探索流程，但让团队在共享、安装、维护 demo adapter 和判断数据命令是否真的有业务结果时更容易发现问题、解释错误并阻止案例索引漂移。

## 变更分组

### Adapter 生命周期

- 新增 `docs/adapter-lifecycle.md`，说明 adapter 从生成、验证、打包、安装到回滚的完整生命周期。
- `market install` 收紧包内容校验，拒绝缺失声明文件、缺失哈希和未声明额外文件。
- `market install --json` 对安装失败返回 `INSTALL_FAILED` 与针对性 `error.fix`，覆盖重复安装、哈希不匹配、manifest 缺失字段、未声明文件和不安全路径。
- `verify --json` 在已安装 adapter 存在 `manifest.json` 时诊断 manifest v1、domain、`files` / `file_hashes` 与已安装文件哈希。

### 真实案例库

- 官网 Quick Start 同步 v0.14.3 首次成功路径，优先引导用户先跑真实 demo adapter，再配置 LLM 生成自定义命令。
- 新增 `scripts/validate_cases.py`，为 `cases/manifest.json` 生成离线验收报告。
- `validate_cases.py --packages-dir` 可离线检查 demo adapter 包的 tarball 安全路径、manifest、声明文件哈希和 metadata schema v3。
- CI 新增 `Case Catalog Validation` job，PR 和主分支默认离线运行案例库验收。
- 保存的抽取 Markdown 报告、内置 `browser extract --mode ... --json` 和生成 adapter 的 JSON 输出新增质量状态，标记空结果、全空字段和部分缺字段；生成的 `list-` / `search-` 数据命令在质量为空时返回 `E_EMPTY_RESULT`，作为 `search-extraction-gap` 后续修复的第一层离线验收。

## 风险与兼容性

- `market install` 现在更严格，手工拼接或被篡改的 adapter 包会失败；使用 `cliany-site market publish <domain>` 重新打包即可修复。
- `verify --json` 会新增 `manifest` 字段；无 market manifest 的本机 `explore` adapter 显示 `manifest.status = "missing"`，但不判定失败。
- `validate_cases.py --packages-dir` 只读取本地包文件，不访问第三方站点，也不写入 `~/.cliany-site/`。
- 生成 adapter 的 JSON 输出新增 `data.quality` 字段；这是向后兼容的附加字段，但 `list-` / `search-` 命令的全空抽取现在会变为 `E_EMPTY_RESULT`，下游如果曾把全空字段当成功处理，需要同步调整。

## Release Notes 摘要

```markdown
v0.14.4 strengthens adapter package distribution and real-demo validation.

- `market install` now rejects incomplete or undeclared package contents before replacing an existing adapter.
- `market install --json` returns actionable `INSTALL_FAILED` fixes for common package failures.
- `verify --json` checks installed adapter manifests and file hashes.
- `scripts/validate_cases.py` validates the real-demo case catalog offline, including optional release asset package checks.
- CI now runs a dedicated Case Catalog Validation job.
- Saved extraction Markdown reports, built-in structured extract JSON, and generated adapter JSON now flag empty results, all-blank fields, and partially missing fields; list-/search- commands return `E_EMPTY_RESULT` when extraction quality is empty.
```
