# v0.16.180 发布草案

**目标版本：** `0.16.180`
**提交范围：** `v0.16.179..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 issue metadata 入口显式化到整包摘要。

## 用户价值

当前 `artifact_bundle_summary` 已经输出 `issue_metadata_count` 和 `issue_metadata_sha256`，维护工具可以判断候选 issue metadata 是否漂移。但如果工具想先看到 metadata 入口，仍需要展开完整 `artifact-manifest.json.issue_metadata_summary` 或 `issue-metadata.json`。

`v0.16.180` 在 `artifact_bundle_summary` 中新增 `issue_metadata_preview_count`、`issue_metadata_preview` 和 `issue_metadata_preview_sha256`，并在 artifacts `README.md` 的 `Artifact Bundle Summary` 段落展示同源字段。这样维护者和自动化可以只读整包摘要，先看到候选 issue metadata 前缀，再决定是否展开完整 manifest 或 metadata 文件。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.180 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 issue metadata preview fields，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且 `issue_metadata_preview` 等于 `artifact-manifest.json.issue_metadata_summary.metadata_preview`。

## 变更分组

### Issue metadata preview

- `issue_metadata_summary` 新增 `metadata_preview_count`。
- `issue_metadata_summary` 新增 `metadata_preview`。
- `issue_metadata_summary` 新增 `metadata_preview_sha256`。
- `artifact_bundle_summary` 新增 `issue_metadata_preview_count`。
- `artifact_bundle_summary` 新增 `issue_metadata_preview`。
- `artifact_bundle_summary` 新增 `issue_metadata_preview_sha256`。
- Preview 从稳定 issue metadata 的前 8 个条目派生。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示 issue metadata preview count/list/hash。
- 现有 issue metadata count/hash、review order preview/tail/boundary/hash、issue body summary 与 inventory 摘要、artifact manifest schema/key/payload 摘要、handoff 摘要和 gate 摘要保持兼容。
- 不改变 release draft validation、publication audit 或 candidate issue gate 的判断逻辑。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示 issue metadata preview fields。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的 issue metadata preview 字段，并断言 preview 与 `issue_metadata_summary` 同源。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的整包摘要可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有稳定 issue metadata，不新增网络访问或外部状态读取。
- 旧工具忽略未知 summary 字段即可继续读取现有 manifest。
- 新工具可以只读 `issue_metadata_count`、`issue_metadata_sha256`、`issue_metadata_preview_count`、`issue_metadata_preview` 和 `issue_metadata_preview_sha256` 快速确认 metadata 入口是否漂移。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
