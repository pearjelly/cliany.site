# v0.16.57 发布草案

**目标版本：** `0.16.57`
**提交范围：** `v0.16.56..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue gate reason codes 带上同源可读说明。

## 用户价值

`v0.16.56` 已经让 `candidate_issue_gate.reason_codes` 给出稳定原因码，例如 `publication_not_published`、`dirty_worktree` 和 `release_draft_issues`。这对脚本足够直接，但维护者在阅读 Markdown report、artifacts `README.md` 或 JSON handoff 时，仍需要回到文档或代码里查每个原因码的含义。

`v0.16.57` 给 `candidate_issue_gate` 增加 `reason_descriptions`。它和 `reason_codes` 同源生成，为每个原因码提供 human-readable 说明，例如 `publication_not_published` 对应 `The latest local release branch or tag is not visible upstream.`。这样自动化可以继续依赖 reason codes，维护者则能在同一个对象里看到可读解释。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.57 --json` 输出 `candidate_issue_gate.reason_descriptions`；Markdown report 展示 `Candidate Issue Gate Reason Descriptions`；candidate issue artifacts README 展示 gate reason descriptions；`artifact-manifest.json` 和 `publication-handoff.json` 保留同源 gate 对象。

## 变更分组

### Candidate issue gate reason descriptions

- `candidate_issue_gate` 新增 `reason_descriptions`。
- `reason_descriptions` 只解释当前存在于 `reason_codes` 的原因码。
- Markdown report 新增 `Candidate Issue Gate Reason Descriptions` 小节。
- Artifacts `README.md` 在 `Publication Handoff` 下展示 gate reason descriptions。
- `artifact-manifest.json` 和 `publication-handoff.json` 通过同源 `candidate_issue_gate` 保留这些说明。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `reason_descriptions` 与 `reason_codes` 的分工。
- `tests/test_plan_next_iteration.py` 覆盖 JSON、文本输出、Markdown report 和 artifacts README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强 candidate issue gate handoff 的自解释能力。

## 风险与兼容性

- `candidate_issue_gate.reason_descriptions` 是新增字段，不改变 `status`、`reason_codes`、`required_actions` 或 `evidence` 的语义。
- 描述文本和 reason codes 同源生成，不新增网络访问。
- 不自动执行命令，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
