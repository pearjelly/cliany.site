# v0.16.108 发布草案

**目标版本：** `0.16.108`
**提交范围：** `v0.16.107..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 candidate issue gate 的首要原因说明补进 candidate issue artifacts 的整包摘要。

## 用户价值

`v0.16.107` 已经让 `artifact_bundle_summary` 能直接展示 `candidate_issue_gate_primary_reason_code` 和 `candidate_issue_gate_primary_required_action`。维护者或自动化工具继续审阅候选 issue artifacts 时，还需要不展开完整 `candidate_issue_gate.reason_descriptions` 就能展示首要原因的可读说明。

`v0.16.108` 在 `artifact_bundle_summary` 中新增 `candidate_issue_gate_primary_reason_description`。当 `candidate_issue_gate.reason_codes` 非空且 `reason_descriptions` 中存在对应说明时，该字段等于首个 reason code 的说明；否则保持 `null`/`None`，避免工具把缺失说明伪装成真实结论。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.108 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 `candidate_issue_gate_primary_reason_description`，且该值等于 manifest 中 `candidate_issue_gate.reason_descriptions[candidate_issue_gate.reason_codes[0]]`；README 的 `Artifact Bundle Summary` 也展示该字段。

## 变更分组

### Artifact bundle gate primary reason description

- `artifact_bundle_summary` 新增 `candidate_issue_gate_primary_reason_description`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示首要 gate reason description。
- 现有 `candidate_issue_gate_primary_reason_code` 和 `candidate_issue_gate_primary_required_action` 保持不变。
- 现有 `candidate_issue_gate_reason_description_count` 和 `candidate_issue_gate_reason_descriptions_sha256` 保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 gate primary reason description。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让候选 issue artifacts 的整包摘要更容易被维护者和自动化工具读取。

## 风险与兼容性

- 新字段只来自已有 `candidate_issue_gate.reason_codes` 和 `candidate_issue_gate.reason_descriptions`。
- 当 gate 允许创建 issue 且 reason code 列表为空时，新字段为 `null`/`None`。
- 当 reason code 存在但 reason description map 缺少对应键时，新字段为 `null`/`None`。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## Release Notes 摘要

`v0.16.108` adds a primary candidate issue gate reason description to artifact bundle summaries, helping tools display the first blocker with human-readable context without expanding the full gate object.
