# v0.16.272 发布草案

**目标版本：** `0.16.272`
**提交范围：** `v0.16.271..HEAD`
**当前状态：** 2026-07-23 release slice；修复已维护 demo 的实际安装入口。

完成 version bump 后，target-mode readiness 还会检查占位锚点 **提交范围：** `v0.16.272..HEAD`；该锚点不改变实际 release diff。

## 用户价值

SuiteCRM、ASF Jira、ASF Confluence 和 ASF Jenkins 四个 `active` demo 不再要求用户先找到并下载一个不存在的本地 `0.14.0` 文件。`cliany-site cases`、README 和官网文档现在给出 GitHub Release v0.14.1 的直接 HTTPS archive URL 与固定 SHA-256；用户可先追加 `--dry-run --json` 无写入地确认包的 domain、version、`commands.py` 与 `metadata.json`。

2026-07-23 在隔离 HOME 下对四个 archive 执行了实际 `market install --sha256 --dry-run --json`：每个都返回 `success=true`、正确 domain、`version=0.14.1` 和两个必需文件。该证据只证明当前 CLI 能下载、校验和读取 release asset；不把第三方网站的在线 replay 或业务结果描述成新鲜成功。

## 变更分组

### Active demo 安装路径

- `cases/manifest.json` 为四个 active case 将首条安装命令改为固定 GitHub Release v0.14.1 URL 和 release digest。
- README、README.zh 和官网文档提供同一条可复制安装路径，并说明先用 `--dry-run --json` 预检。
- `cases/README.md` 明确 active demo asset 可由当前 `market install` 读取，但第三方在线可用性仍需在运行时检查。

### Catalog 校验

- `scripts/validate_cases.py` 继续接受历史 `<domain>.cliany-adapter-...` 本地包名，同时识别标准 `<domain>-<version>.cliany-adapter.tar.gz` release asset 文件名。
- 离线回归固定四个 active URL/digest，并覆盖 release asset filename 的 domain 校验。

## 案例库映射

`suitecrm-accounts`、`apache-jira-issues`、`apache-confluence-search` 和 `apache-jenkins-jobs` 仍为 active；本版只修正其安装可达性。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍为 candidate，`search-extraction-gap` 仍为 known-gap。

本轮真实 `cliany-site doctor --llm-live --json` 处于 `llm_live_preflight_not_ready`：仍返回 `E_LLM_UNAVAILABLE`，`summary.capabilities.generate_adapters.ready=false`。因此没有运行 candidate 的真实 `explore`，也没有生成 adapter package、metadata validation 或 online smoke 成功证据。

## 风险与兼容性

- 这是对历史 release asset 的安装说明和 filename 校验兼容性修复，不会重写、升级或覆盖用户已有 adapter；实际安装时仍保留 `market install` 的冲突、`--force` 和 backup 语义。
- 资产 SHA-256 固定 v0.14.1 archive；下载内容改变会被拒绝，而不是静默安装。
- 第三方 demo 的登录、数据和在线 replay 会随目标站状态变化；本版不承诺其持续可用性。

## 发版步骤

1. 在 `[Unreleased]` 保留本版条目、compare link 为 `v0.16.271...HEAD` 时运行 target-mode readiness。
2. Finalize Changelog：清空 `[Unreleased]`，新增 `## [0.16.272] - 2026-07-23`，并更新 compare links。
3. 提交并推送 `master`，等待 GitHub CI 与 Embodied CI 成功。
4. 创建 `git tag v0.16.272`，然后运行 `uv run --extra dev --frozen python scripts/release_readiness.py --strict --release-tag v0.16.272 --remote --remote-name origin`。
5. 仅在该 gate 通过后推送 tag，等待 GitHub Release 与 PyPI 发布。
6. 在 `site/` 运行 `vercel link --yes --project cliany.site` 和 `vercel --prod --yes`；随后检查 `https://www.cliany.site/` 的 `v0.16.272` 内容。
7. 运行 `uv run --extra dev --frozen python scripts/check_release_publication.py --strict --remote --distribution --json`。

## Release Notes 摘要

`v0.16.272` makes the four maintained historical demos installable from their real, digest-pinned GitHub Release assets and keeps the case catalog honest about what has, and has not, been verified online.
