# v0.16.239 发布草案

**目标版本：** `0.16.239`
**提交范围：** `v0.16.238..HEAD`
**当前阻塞：** 真实 LLM provider 预检仍返回 `502 Bad Gateway` / `E_LLM_UNAVAILABLE`，因此本版不生成 candidate adapter package，不运行真实 `explore`，也不把任何 candidate 标记为 active。本版改进的是候选案例交接材料，让维护者在遇到同类阻塞时记录真实 blocker，而不是伪造晋级证据。

## 用户价值

`v0.16.237` 已经把 candidate promotion command plan 统一为 `llm_live_preflight`、`adapter_package`、`metadata_validation`、`online_smoke` 四步。但当 live preflight 失败时，issue 模板和 evidence bundle 仍更强调“下一步运行 explore”，维护者需要凭经验判断是否应该停止。

`v0.16.239` 把这个判断写进可复制的 handoff：所有 candidate issue body 都新增 `LLM Preflight Gate`，明确要求先运行 `cliany-site doctor --llm-live --json`；如果 `generate_adapters.ready=false` 或 `llm_live` 返回 `E_LLM_UNAVAILABLE`，就停止本轮晋级，贴回 doctor JSON / 错误摘要，并保持 `adapter_package` 为 pending 或 blocked。这样候选真实案例库可以继续推进协作，但不会把上游 502 当作 adapter package 证据污染案例状态。

## 变更分组

- Cases CLI：`cliany-site cases --case-id <id> --issue-template` 现在输出 `LLM Preflight Gate` 小节。
- Evidence bundle：JSON 顶层新增 `llm_live_preflight_blocker_note`，人类 Markdown 输出在 `LLM live preflight` 小节展示同一 blocker handling。
- Next iteration planner：`scripts/plan_next_iteration.py` 生成的 candidate issue body 同步包含 preflight gate，确保 `/tmp/cliany-candidate-issues` artifacts 和 CLI 模板一致。
- Case validation report：`scripts/validate_cases.py --report` 的 Issue Body Template 同步包含 preflight gate。
- 文档：[cases/README.md](../../cases/README.md) 说明 live preflight 失败时应停止 `explore` 并贴回 blocker evidence。
- Tests：`tests/test_cases_command.py`、`tests/test_plan_next_iteration.py` 和 `tests/test_validate_cases.py` 覆盖 JSON、Markdown、planner artifact 和 report 模板中的 preflight gate。

## 案例库映射

- 本次不修改 [cases/manifest.json](../../cases/manifest.json) 的案例状态。
- 当前案例库仍保持 active 4 个、candidate 3 个、known gap 1 个。
- Known gap 保持 1 个：`search-extraction-gap`。
- Candidate 晋级仍需要三段真实证据：adapter package、metadata validation 和 online smoke。
- `pypi-project-search` 仍是首要 candidate，但当前 live LLM preflight 返回 `E_LLM_UNAVAILABLE` / `502 Bad Gateway`；本版只让 handoff 更清楚地记录这个 blocker，不生成或激活任何 candidate adapter。

## 风险与兼容性

- 这是 CLI 输出、planner/report 模板和文档的交接语义增强，不改变 adapter schema、AXTree 语义匹配、action runtime、codegen、session 存储或 marketplace package 格式。
- 新增 JSON 字段是 additive：现有读取 `evidence_bundle` 的自动化可以忽略 `llm_live_preflight_blocker_note`。
- Issue body 多一个 `LLM Preflight Gate` 小节；复制到 GitHub issue 的内容更长，但能减少失败上游调用被误当作晋级证据的风险。

## Release note draft

v0.16.239 adds an explicit LLM preflight failure contract to candidate promotion issue templates, evidence bundles, planner artifacts, validation reports, and the case catalog docs. Maintainers now get clear instructions to stop candidate promotion and attach blocker evidence when `doctor --llm-live` reports `E_LLM_UNAVAILABLE`, instead of treating an upstream outage as adapter package progress.
