# v0.16.32 发布草案

**目标版本：** `0.16.32`
**提交范围：** `v0.16.31..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 publish script 在 push 前拒绝 dirty worktree，避免发布脚本和本地未提交改动混在一起。

## 用户价值

`v0.16.31` 已经让 `scripts/check_release_publication.py --publish-script` 生成的脚本固定 repo root，并检查 HEAD、latest tag 和 tag commit 是否仍然匹配。但维护者可能在生成脚本后改了文件、留下临时调试内容或新增未跟踪文件；这些改动不会改变 HEAD，却会让“当前本地目录是否可发布”变得含糊。

`v0.16.32` 在 generated publish script 的 push 前加入 `git status --porcelain` 检查。如果 worktree 有未提交改动，脚本会打印 `Publish script is stale: worktree has uncommitted changes.` 和具体 status 行，然后退出，不执行 `git push origin master`、`git push origin v0.16.32` 或远端复核命令。

成功标准是：`python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh` 写出的脚本包含 `git status --porcelain` dirty worktree preflight；如果脚本生成后仓库出现未提交文件，直接运行脚本会失败，并且不会推进远端分支。

## 变更分组

### Publication publish script dirty worktree preflight

- `scripts/check_release_publication.py --publish-script` 写出的脚本会在 push 前读取 `git status --porcelain`。
- worktree 不干净时输出 `Publish script is stale: worktree has uncommitted changes.` 和具体 status。
- 检测到 dirty worktree 时后续 publish commands 不会执行。

### 文档与门禁

- [docs/release-cadence.md](../release-cadence.md) 说明 publish script 会检查 worktree 状态。
- `tests/test_release_publication.py` 覆盖脚本字段和 dirty worktree 拒绝执行行为。
- `tests/test_release_cadence_docs.py` 固定发布节奏文档说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强 release publication handoff 脚本的执行前保护。

## 风险与兼容性

- 这是 generated shell script 的本地 preflight 增强，不改变 `publish_commands` 的 JSON schema 或命令顺序。
- 如果维护者在生成脚本后留下未提交改动，旧脚本会失败；这是预期保护。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- `--remote` 行为保持不变；dirty worktree preflight 只读取本地 git 状态。

```bash
python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh
python scripts/release_readiness.py --target-version 0.16.32 --json
```

## 发版步骤

1. 确认 `python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh` 写出的脚本包含 `git status --porcelain`。
2. 确认脚本包含 `worktree has uncommitted changes` 失败提示。
3. 确认 dirty worktree 测试会在 push 前退出。
4. 运行 publication、release cadence 文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.32`。
6. 将 `pyproject.toml` 版本更新为 `0.16.32`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.32`。
8. 创建 tag：`git tag v0.16.32`。

## Release Notes 摘要

`v0.16.32` adds a dirty-worktree preflight to generated publish scripts, refusing to run branch/tag pushes when local files have uncommitted changes.
