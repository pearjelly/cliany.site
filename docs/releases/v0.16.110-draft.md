# v0.16.110 发布草案

**目标版本：** `0.16.110`
**提交范围：** `v0.16.109..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 candidate issue gate 的 reason/action 数量补进 README 快速摘要。

## 用户价值

`v0.16.109` 已经让 candidate issue artifacts `README.md` 在 `Artifact Bundle Summary` 之前展示 `Candidate Issue Gate Quick Summary`。维护者可以先看到首要 reason 和首个 required action，但还不能在这个快速摘要里判断后面是否还有更多 reason/action 需要展开。

`v0.16.110` 在 `Candidate Issue Gate Quick Summary` 中新增 `reason_code_count` 和 `required_action_count`。维护者打开 README 时，可以先判断 gate 是单一阻塞还是多项待办，再决定是否继续展开 `Publication Handoff`、`artifact-manifest.json` 或完整 `Artifact Bundle Summary`。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.110 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 出现在 `Artifact Bundle Summary` 之前，并展示 `reason_code_count`、`required_action_count`、`primary_reason_code` 和 `primary_required_action`。

## 变更分组

### Candidate issue gate quick summary counts

- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `reason_code_count`。
- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `required_action_count`。
- 现有 quick summary 的 status、can_create_issues、requires_maintainer_review、primary reason/action、latest tag 和 visibility 保持不变。
- 现有 `artifact_bundle_summary` 字段保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 quick summary 会展示 reason/action 数量。
- `tests/test_plan_next_iteration.py` 覆盖 README quick summary 的 count 行。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新增的两个 README 行只读取已有 `candidate_issue_gate.reason_code_count` 和 `candidate_issue_gate.required_action_count`。
- `Artifact Bundle Summary` 字段不删除、不改名，机器读取路径保持兼容。
- 如果 gate 对象缺少 count 字段，README 会按现有格式显示 `(missing)`。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.110..HEAD`。最终带 `--release-tag v0.16.110` 的 strict gate 仍以本草案顶部的 `v0.16.109..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.110 --json
python scripts/plan_next_iteration.py --target-version 0.16.110 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.110 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.110 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 `reason_code_count` 和 `required_action_count`。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.110`。
7. 将 `pyproject.toml` 版本更新为 `0.16.110`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.110`。
9. 创建 tag：`git tag v0.16.110`。

## Release Notes 摘要

`v0.16.110` adds reason/action counts to the Candidate Issue Gate Quick Summary in generated candidate issue artifacts README files, helping maintainers see whether more gate details need review before expanding the full artifact bundle summary.
