# v0.16.211 发布草案

**目标版本：** `0.16.211`
**提交范围：** `v0.16.210..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 create issues safety contract key preview 与 tail 显式化到整包摘要。

## 用户价值

`v0.16.210` 已经让 `artifact_bundle_summary` 可以展示 review checklist 的前后窗口。维护者在审阅 candidate issue artifacts 时，还需要快速确认 create issues safety contract 的 key 结构是否仍覆盖 dry-run 与 preflight；目前 summary 已有 key 数量、完整 hash 和首尾边界，但只读 summary 的工具仍无法在契约扩展时看到 key 前后窗口。

`v0.16.211` 在 `artifact_bundle_summary` 中新增 create issues safety contract key preview/tail 字段。这样维护者和自动化可以只读整包摘要，先检查安全契约 key 的前后窗口、数量、完整 hash 和首尾边界，再决定是否展开完整 `artifact-manifest.json.create_issues_safety` 或 artifacts README 的 `Create Issues Safety` 段落。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.211 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 create issues safety contract key preview/tail fields，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且 preview/tail 等于 create issues safety contract keys 的前 8 项和后 8 项。

## 变更分组

### Create issues safety contract key window summary

- `artifact_bundle_summary` 新增 `create_issues_safety_contract_key_preview_count`。
- `artifact_bundle_summary` 新增 `create_issues_safety_contract_key_preview`。
- `artifact_bundle_summary` 新增 `create_issues_safety_contract_key_preview_sha256`。
- `artifact_bundle_summary` 新增 `create_issues_safety_contract_key_tail_count`。
- `artifact_bundle_summary` 新增 `create_issues_safety_contract_key_tail`。
- `artifact_bundle_summary` 新增 `create_issues_safety_contract_key_tail_sha256`。
- Preview 从 create issues safety contract keys 的前 8 项派生；tail 从后 8 项派生。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示 create issues safety contract key preview/tail fields。
- 现有 `create_issues_safety_contract_key_count`、`create_issues_safety_contract_sha256`、`create_issues_safety_contract_first_key`、`create_issues_safety_contract_last_key`、`create_issues_safety_contract_key_boundary_sha256`、review checklist windows、validation command windows 和 release draft windows 字段保持兼容。
- 不改变 create issues safety contract 内容、dry-run/preflight 行为、candidate issue gate、publication audit、publish commands 内容或 candidate case 状态。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示 create issues safety contract key preview/tail fields。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的 create issues safety contract key preview/tail 字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的整包摘要可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有 create issues safety contract keys，不新增网络访问或外部状态读取。
- 旧工具忽略未知 summary 字段即可继续读取现有 manifest。
- 新工具可以只读 `create_issues_safety_contract_key_preview_count`、`create_issues_safety_contract_key_preview`、`create_issues_safety_contract_key_preview_sha256`、`create_issues_safety_contract_key_tail_count`、`create_issues_safety_contract_key_tail` 和 `create_issues_safety_contract_key_tail_sha256` 快速确认 create issues safety contract key 窗口是否漂移。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
