# v0.16.263 发布草案

**目标版本：** `0.16.263`
**提交范围：** `v0.16.262..HEAD`
**当前状态：** 2026-07-15 release execution；本版在安装前提供本地 adapter 包的只读校验与覆盖计划。

## 用户价值

`cliany-site market install <package> --dry-run --json` 复用真实安装的 tar 安全、manifest 与文件哈希校验，并返回 `package_sha256`、`would_replace` 与 `would_create_backup`。预检仅在临时目录解压，不会写入 `~/.cliany-site/adapters/` 或 `~/.cliany-site/backups/`。

当包无效、缺少文件或已安装 adapter 未使用 `--force` 时，命令仍返回既有 `INSTALL_FAILED` 错误 envelope；因此调用方可以先在没有运行时副作用的条件下确认包完整性与实际安装计划。

## 变更范围

- `market install` 新增 `--dry-run`，验证本地 adapter 包而不安装。
- JSON 预检结果包含包 SHA-256、域名、版本、文件清单、替换意图和备份意图。
- 覆盖安装的预检只报告 `would_replace=true` 与 `would_create_backup=true`，不会修改已有 adapter，也不会创建 backup。

## 风险与兼容性

- `--dry-run` 只增加安装前验证路径；默认安装、`--force` 覆盖和备份语义不变。
- 预检沿用真实安装的 tar、manifest 和文件哈希校验，因此不接受安装流程会拒绝的包。
- 本地预检对 `~/.cliany-site/adapters/` 和 `~/.cliany-site/backups/` 保持零写入；运行时状态仍不进入仓库。
- 本草案只准备本地发布材料，不表示 GitHub Release、PyPI 发布或官网生产部署已经完成。

## 案例库映射

- 案例入口继续是 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。
- `pypi-project-search` 保持 candidate：`llm_live_preflight_not_ready` 仍阻止真实 exploration，不能伪造 adapter package、metadata validation 或 online smoke 证据。
- 当前 known gap 仍为 `search-extraction-gap`；本版不改变案例状态或数量。

## 发版步骤

1. 复核 `market install <package> --dry-run` 对有效、新安装与 `--force` 覆盖计划的 JSON，并确认 adapters/backups 均为零写入。
2. 提交并推送 `master`，等待 CI 成功。
3. 创建本地标准 tag：`git tag v0.16.263`；在推送 tag 前运行严格远端 release-tag check：

   ```bash
   uv run --extra dev --frozen python scripts/release_readiness.py --strict --release-tag v0.16.263 --remote --remote-name origin
   ```

4. 仅在 release-tag check 通过后运行 `git push origin v0.16.263`。等待 tag 对应的 release workflow 创建 GitHub Release 并发布 PyPI `cliany-site==0.16.263`；发布前不声称这些外部动作已经发生。
5. 在 `site/` 运行 `vercel link --yes --project cliany.site` 和 `vercel --prod --yes`，再运行 `vercel inspect www.cliany.site --wait --timeout 90s` 进行生产部署检查。The preceding `vercel inspect www.cliany.site --wait --timeout 90s` is the website verification.
6. 运行最终发布审计：

   ```bash
   uv run --extra dev --frozen python scripts/check_release_publication.py --strict --remote --distribution --json
   ```

   该审计用于核对远端 Git、GitHub Release 和 PyPI 分发，不审计官网状态；the final audit checks remote Git, GitHub Release, and the PyPI distribution, not website status。在审计通过前不声称这些外部动作已经完成。

## Release Notes 摘要

v0.16.263 adds a local-only dry run for adapter package installation. It validates the same package safety and integrity rules as installation, reports package identity plus replacement/backup intent, and leaves adapter and backup runtime directories untouched.
