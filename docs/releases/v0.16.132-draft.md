# v0.16.132 发布草案

**目标版本：** `0.16.132`
**提交范围：** `v0.16.131..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把发布脚本 path/command hash 补进 publication handoff。

## 用户价值

`v0.16.131` 已经把 `publication_publish_script_command_sha256` 提升到 plan 级别，工具可以在不生成完整 candidate issue artifacts 时检测发布脚本命令是否漂移。但只读取 `publication-handoff.json` 的工具仍只能看到 `publish_script_path` 和 `publish_script_command`，不能直接拿到对应 hash。

`v0.16.132` 在 `publication-handoff.json` 中新增 `publish_script_path_sha256` 和 `publish_script_command_sha256`，并在 artifacts `README.md` 的 `Publication Handoff` 段落展示同一组摘要。维护者和自动化只读 handoff 文件或 README handoff 小节，就能判断发布脚本路径和生成命令是否漂移。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.132 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `publication-handoff.json` 包含 `publish_script_path_sha256` 与 `publish_script_command_sha256`，artifacts README 的 `Publication Handoff` 展示同一组字段，`artifact_bundle_summary` 中的 publication handoff hash 随新增字段稳定更新。

## 变更分组

### Publication handoff publish script hashes

- `publication-handoff.json` 新增 `publish_script_path_sha256`。
- `publication-handoff.json` 新增 `publish_script_command_sha256`。
- Artifacts `README.md` 的 `Publication Handoff` 段落展示 `publish_script_path_sha256` 和 `publish_script_command_sha256`。
- 现有 `publication_publish_script_path_sha256`、`publication_publish_script_command_sha256` 和 `artifact_bundle_summary` 保持兼容。
- `publication_handoff_key_count` 和 `publication_handoff_sha256` 会随 handoff 结构变化稳定更新。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `publication-handoff.json` 和 README handoff 会展示发布脚本 path/command hash。
- `tests/test_plan_next_iteration.py` 覆盖 handoff JSON、README handoff 和 bundle summary 的新字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 publication handoff 的发布脚本漂移检测能力。

## 风险与兼容性

- 新增字段只派生自既有发布脚本 path 和 command。
- 现有 `publish_script_path`、`publish_script_command`、plan 级 hash 和 `artifact_bundle_summary` 保持兼容。
- 新增 hash 只用于漂移检测，不替代脚本内容审阅。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
