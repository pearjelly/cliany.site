# v0.16.274

Generated data commands now fail honestly when browser actions finish without returning usable extracted data.

## What Changed

- Generated `list-`, `search-`, `read-`, and `extract-` commands now enforce `data.quality`.
- Commands with an `extract` action receive the same guard even when their name does not follow those prefixes.
- A named data command without an `extract` action now returns `E_EMPTY_RESULT`; a browser click or navigation alone cannot count as a zero-match read.
- With the default `expects_nonempty=true`, empty data, missing required fields, and partial rows return `E_EMPTY_RESULT` instead of reporting a successful read.
- `expects_nonempty=false` remains a narrow opt-in for workflows where a genuine zero-match result is valid; it does not accept missing or partial data.

## Compatibility

The new guard applies to newly generated code and commands that are re-explored. Existing installed adapters are not silently rewritten.

## Verification

Regression coverage includes the documented search-extraction gap and generated commands with both conventional and non-conventional names. No live LLM, third-party adapter, or online result is claimed by this release.
