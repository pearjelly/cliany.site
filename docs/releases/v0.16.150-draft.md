# v0.16.150 发布草案

**目标版本：** `0.16.150`
**提交范围：** `v0.16.149..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本给 publication handoff 增加 schema version。

## 用户价值

`release-draft-handoff.json` 和 `artifact-manifest.json` 已经带有 `schema_version`，维护工具可以先判断字段语义版本，再解析后续字段。`publication-handoff.json` 目前已经包含 publication status、gate、visibility、primary next action、primary publish command 和发布脚本摘要，但缺少同样的 schema 入口。

`v0.16.150` 在 `publication-handoff.json` 中新增 `schema_version: 1`，并在 artifacts `README.md` 的 `Publication Handoff` 段落展示同源字段。这样只读取 publication handoff 的工具可以和 release draft handoff 一样，先做字段语义版本判断，再解析 publication gate、发布待办和发布命令。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.150 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `publication-handoff.json` 包含 `schema_version`，artifacts README 的 `Publication Handoff` 展示 `schema_version: 1`，并且 `artifact_bundle_summary` 中的 `publication_handoff_key_count` 与 `publication_handoff_sha256` 反映该 handoff 字段变化。

## 变更分组

### Publication handoff schema version

- `publication-handoff.json` 新增 `schema_version: 1`。
- Artifacts `README.md` 的 `Publication Handoff` 展示 `schema_version`。
- 现有 `publication_handoff_sha256`、`publication_handoff_key_count` 和 `artifact_bundle_summary` 保持兼容并反映 handoff 字段变化。
- `release-draft-handoff.json` 继续保留既有 `schema_version`，两类 handoff 的字段语义版本入口保持一致。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 publication handoff 会展示 `schema_version`。
- `tests/test_plan_next_iteration.py` 覆盖 handoff JSON、README 展示和 handoff hash。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的 publication handoff 解析契约。

## 风险与兼容性

- 新字段为常量 `1`，不新增网络访问或外部状态读取。
- 旧工具忽略未知字段即可继续读取现有 publication handoff。
- 新工具可以先读取 `schema_version`，再决定是否支持当前 handoff 字段语义。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.150..HEAD`。最终带 `--release-tag v0.16.150` 的 strict gate 仍以本草案顶部的 `v0.16.149..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.150 --json
python scripts/plan_next_iteration.py --target-version 0.16.150 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.150 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.150 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `publication-handoff.json` 包含 `schema_version: 1`。
3. 确认 artifacts README 的 `Publication Handoff` 展示 `schema_version: 1`。
4. 确认 `artifact_bundle_summary.publication_handoff_key_count` 已反映新增字段。
5. 确认 `artifact_bundle_summary.publication_handoff_sha256` 与当前 `publication-handoff.json` 同源。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.150`。
8. 将 `pyproject.toml` 版本更新为 `0.16.150`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.150`。
10. 创建 tag：`git tag v0.16.150`。

## Release Notes 摘要

`v0.16.150` adds a schema version to publication handoff artifacts, giving tools a stable way to identify publication handoff field semantics before parsing the rest of the handoff payload.
