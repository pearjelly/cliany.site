# v0.16.224 发布草案

**目标版本：** `0.16.224`
**提交范围：** `v0.16.223..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 `artifact_bundle_summary` 能定位 candidate 晋级首要证据任务。

## 用户价值

`v0.16.223` 已经让只读 `artifact_bundle_summary` 的维护工具看到 candidate 晋级证据摘要的字段 hash、summary hash、计数和 `case_promotion_evidence_primary_next_action`。这能展示下一步动作，但工具仍要展开完整 `case_promotion_evidence_summary` 才知道这个动作对应哪个 case、哪个 promotion task，以及当前任务状态和 evidence 是否漂移。

`v0.16.224` 为首要 candidate 晋级任务补上 bundle-level 定位字段：`case_promotion_evidence_primary_case_id`、`case_promotion_evidence_primary_task`、`case_promotion_evidence_primary_status` 和 `case_promotion_evidence_primary_evidence_sha256`。维护工具只读整包摘要时，也能直接显示“先推进 `pypi-project-search` 的 `adapter_package`，当前仍是 `pending`，当前 evidence hash 为某值”。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.224 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含这些 `case_promotion_evidence_primary_*` 字段；artifacts `README.md` 的 `Artifact Bundle Summary` 展示同源字段；当前真实案例库仍显示 3 个 candidate、9 个 pending 子任务，并把 PyPI adapter package 作为 primary task。

## 变更分组

### Bundle-level primary evidence task locator

- `ARTIFACT_BUNDLE_SUMMARY_KEYS` 新增 `case_promotion_evidence_primary_case_id`、`case_promotion_evidence_primary_task`、`case_promotion_evidence_primary_status` 和 `case_promotion_evidence_primary_evidence_sha256`。
- `artifact_bundle_summary` 从现有 `case_promotion_evidence_summary.primary_task` 派生这些字段。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 输出这些字段，供人工审阅和机器读取保持一致。
- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明只读整包摘要的维护工具可以展示首要 case/task/status 和当前 evidence hash。

### 文档与门禁

- `tests/test_plan_next_iteration.py` 固定 manifest summary 字段、evidence hash 和 README 渲染。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护循环文档中的字段说明。
- `tests/test_release_draft_docs.py` 固定本草案。
- 本版本不改变 `case_promotion_evidence_summary.primary_task` 的完整结构，也不改变 candidate issue body 内容。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让 candidate issue artifacts 的整包摘要能直接定位下一项晋级证据工作。

## 风险与兼容性

- 这是 `artifact_bundle_summary` 的新增字段，不移除既有字段。
- 字段从已有 `primary_task` 派生，不改变 primary task 的选择规则。
- Evidence hash 只反映当前 evidence 字段内容，不代表 evidence 已完成。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.224..HEAD`。最终带 `--release-tag v0.16.224` 的 strict gate 仍以本草案顶部的 `v0.16.223..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.224 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/plan_next_iteration.py --target-version 0.16.224 --report /tmp/cliany-next-iteration.md
python scripts/release_readiness.py --target-version 0.16.224 --json
python scripts/release_readiness.py --strict --target-version 0.16.224 --report /tmp/cliany-release-readiness.md
```

## 发版步骤

1. 确认 focused pytest 和 ruff 通过。
2. 确认 `plan_next_iteration.py --issues-dir` 生成的 `artifact_bundle_summary` 包含 `case_promotion_evidence_primary_case_id`。
3. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 包含 `case_promotion_evidence_primary_evidence_sha256`。
4. 确认 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.224`。
5. 确认 `pyproject.toml` 版本更新为 `0.16.224`。
6. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.224`。
7. 创建 tag：`git tag v0.16.224`。

## Release Notes 摘要

`v0.16.224` adds bundle-level primary candidate promotion task locator fields, so maintenance tools can show which case and task need evidence next without expanding the full artifacts manifest.
