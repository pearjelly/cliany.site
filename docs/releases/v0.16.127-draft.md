# v0.16.127 发布草案

**目标版本：** `0.16.127`
**提交范围：** `v0.16.126..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 publication publish command 数量补进 candidate issue gate quick summary。

## 用户价值

`v0.16.126` 已经修正了 quick summary 中含反引号文本的 inline code 渲染。维护者现在能稳定阅读 `visibility_summary`，但 Quick Summary 仍只展示 publication next action 数量，不展示实际 publication publish commands 的数量。

`v0.16.127` 在 `Candidate Issue Gate Quick Summary` 中新增 `publication_publish_command_count`。维护者打开 README 时，可以先判断发布脚本需要审阅和执行几条命令，再决定是否展开 `Publication Handoff` 或 `artifact-manifest.json` 查看完整命令列表。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.127 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 出现在 `Artifact Bundle Summary` 之前，并展示 `publication_next_action_count` 和 `publication_publish_command_count`。

## 变更分组

### Candidate issue gate quick summary publish command count

- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_publish_command_count`。
- 该字段来自已有 `IterationPlan.publication_publish_command_count`。
- 现有 `publication_publish_commands` 列表和 `artifact_bundle_summary` 字段不改名、不删除。
- 现有 quick summary 的 ok、blocker/action count、visibility summary 和 publication refs 保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 quick summary 会展示 publication publish command 数量。
- `tests/test_plan_next_iteration.py` 覆盖 README quick summary 的发布命令数量行。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新增 README 行只读取已有 plan 状态。
- 现有 JSON schema 和 publication publish command 列表保持兼容。
- 计数只用于快速判断命令规模，不替代完整命令列表审阅。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.127..HEAD`。最终带 `--release-tag v0.16.127` 的 strict gate 仍以本草案顶部的 `v0.16.126..HEAD` 作为实际 release diff 范围。

```bash
python scripts/plan_next_iteration.py --target-version 0.16.127 --json
python scripts/plan_next_iteration.py --target-version 0.16.127 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.127 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.127 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 `publication_publish_command_count`。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.127`。
7. 将 `pyproject.toml` 版本更新为 `0.16.127`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.127`。
9. 创建 tag：`git tag v0.16.127`。

## Release Notes 摘要

`v0.16.127` adds publication publish command count to the Candidate Issue Gate Quick Summary in generated candidate issue artifacts README files, helping maintainers estimate release command review scope before expanding the full artifact bundle summary.
