# v0.15.8 发布草案

**目标版本：** `0.15.8`
**提交范围：** `v0.15.7..HEAD`
**当前阻塞：** 需要提交 case report 的 Issue Body Template、文档入口和验证测试。

## 用户价值

`v0.15.8` 聚焦把真实案例库的 candidate 晋级任务真正接到贡献者协作流程。`v0.15.7` 已经让 `scripts/validate_cases.py --report` 输出 `Candidate Promotion Tasks`，但维护者仍要手工补 issue 的 scope、验收证据和非目标边界，容易遗漏“不提前标记 active”“不依赖真实 LLM key”“不写 repo runtime 状态”等关键约束。

成功标准是：`python scripts/validate_cases.py --report /tmp/cliany-case-catalog-report.md` 生成的 Markdown 报告为每个 candidate 输出 `Issue Body Template`；模板包含 Scope、Tasks、Validation Evidence 和 Non-goals，且保留 `adapter_package`、`metadata_validation`、`online_smoke` 三类任务。

## 变更分组

### Candidate issue body template

- `scripts/validate_cases.py --report` 在每个 candidate 的任务清单后输出 `Issue Body Template`。
- 模板包含 `## Scope`、`## Tasks`、`## Validation Evidence` 和 `## Non-goals`。
- Non-goals 明确不要提前把案例改成 `active`，也不要要求真实 LLM key 或把 runtime 状态写入仓库。

### 文档与门禁

- [cases/README.md](../../cases/README.md) 说明 `Candidate Promotion Tasks` 已包含可复制 issue body。
- [docs/good-first-issues.md](../good-first-issues.md) 的 case report 任务指向 `Issue Body Template`。
- `tests/test_validate_cases.py` 应检查 report 中的 issue body 模板、验收证据和非目标边界。
- `tests/test_good_first_issues_docs.py` / `tests/test_contributor_docs.py` 应保留模板入口。

## 案例库映射

本版本不新增案例数量，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。Active 案例保持 4 个，candidate 案例保持 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 三个；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是 report artifact 和文档入口的 patch，不改变 CLI runtime、adapter schema、JSON envelope 或 `validate_cases.py --json` 输出结构。
- `Issue Body Template` 只是协作模板，不会自动访问第三方站点、生成 adapter 包或创建 GitHub issue。
- 默认验证继续离线运行。

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.15.8 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行 case report、Good First Issues 和 release draft 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.15.8`。
4. 将 `pyproject.toml` 版本更新为 `0.15.8`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.15.8`。
6. 创建 tag：`git tag v0.15.8`。

## Release Notes 摘要

`v0.15.8` adds copy-ready issue body templates to candidate promotion reports. Maintainers can now turn each candidate case into a scoped GitHub issue with tasks, validation evidence, and non-goals preserved from the report.
