# v0.16.248 发布草案

**目标版本：** `0.16.248`
**提交范围：** `v0.16.247..HEAD`
**当前阻塞：** 本周 release cadence 仍未达到 `3/3` commit days；本草案继续准备下一版，不在 readiness 通过前创建或推送 `v0.16.248` tag。`pypi-project-search` candidate 晋级仍必须先通过 live LLM gate，不能用 preflight 失败替代 adapter package、metadata validation 或 online smoke 证据。

## 用户价值

`v0.16.247` 已把 live LLM gate 连续失败的证据沉淀进发布流程，并在不伪造 candidate adapter package 的前提下完成公开发布。`v0.16.248` 应继续沿着真实案例库主线推进，把下一步工作收敛到一个可验证 slice：优先重试 `pypi-project-search`，只有 live LLM preflight 明确允许生成 adapter 时，才进入真实 `explore`、candidate package validation 和 online smoke。

本版准备工作的价值是减少下一次发版的判断成本：先把 release cadence、candidate gate、验证命令和回退边界写清楚，并把发布后公开渠道审计从“tag 可见”扩展到 GitHub Release 与 PyPI latest version。若 LLM 上游仍不可用，本版继续记录阻塞证据并保持 candidate 状态；若恢复可用，则补齐 `pypi-project-search` 的 adapter package、metadata validation 和 online smoke 三段证据。

## 变更分组

- Release draft：新增本草案，承接 `v0.16.247` 后的下一版准备。
- Candidate gate：继续把 `cliany-site doctor --llm-live --json` 作为 `pypi-project-search` 真实 `explore` 的前置门禁。
- Publication audit：`scripts/check_release_publication.py --distribution` 显式检查 GitHub Release latest tag 与 PyPI latest version，发布后可用 `distribution_ok` 证明标准流程已经到达公开渠道。
- Website/docs：README、README.zh、发布节奏文档、周维护手册和官网 quickstart 同步说明 `--remote --distribution --json` 发布后审计命令。
- Changelog：`CHANGELOG.md` 的 Unreleased 记录 `v0.16.248` 下一版准备；正式发布时再移动到 `0.16.248` 小节。

## Candidate preflight evidence

`pypi-project-search` 晋级命令计划：

```bash
python -m cliany_site doctor --llm-live --json
cliany-site explore "https://pypi.org" "search Python packages for cliany-site and list project names" --json
python scripts/validate_cases.py --packages-dir ~/.cliany-site/packages --include-candidate-packages --strict
cliany-site pypi.org search-projects --query cliany-site --limit 5 --json
```

执行真实 `explore` 前必须先运行 live LLM preflight。如果 `llm_live` 返回 `E_LLM_UNAVAILABLE`、`E_UNKNOWN` connection error，或 `ready_for_explore=false` / `generate_adapters.ready=false`，本轮停止真实 `explore`，不生成或伪造 `pypi.org-<version>.cliany-adapter.tar.gz`，并将结果记录为阻塞证据。

## 风险与兼容性

- 本草案包含发布后审计 CLI 增强：`check_release_publication.py` 新增 opt-in `--distribution`、`--github-repo` 和 `--pypi-project` 参数；默认不联网的本地 publication audit 行为保持兼容。
- 本草案不改变 adapter metadata schema、AXTree 语义匹配、action runtime、session 存储或 marketplace package 格式。
- Candidate 晋级证据不得用 preflight 失败代替 adapter package；失败只能解释为什么保持 pending。
- 禁止引入脆弱 CSS selector 兜底，后续 adapter 生成仍必须依赖 AXTree 语义匹配。
- 运行时状态仍必须写入 `~/.cliany-site/`，不得把 adapters、sessions、snapshots 或 packages 写进 repo。


正式 tag 前再运行：

```bash

python scripts/release_readiness.py --strict --release-tag v0.16.248 --remote --remote-name origin
```
