# v0.16.3 发布草案

**目标版本：** `0.16.3`
**提交范围：** `v0.16.2..HEAD`
**当前阻塞：** 需要提交计划器 candidate 晋级任务表、维护者循环入口和测试；最新本地 release 仍需维护者确认后 push 到远端。

## 用户价值

`v0.16.2` 新增了 `scripts/plan_next_iteration.py`，能把 release readiness、publication audit、commit cadence 和 candidate cases 聚合成下一步建议。但它只列出了 candidate case ID，维护者还要再打开 case catalog report 才能看到 `adapter_package`、`metadata_validation` 和 `online_smoke` 三项晋级证据。

`v0.16.3` 把这些证据项直接带进计划器输出。周初运行 `python scripts/plan_next_iteration.py --report /tmp/cliany-next-iteration.md` 后，维护者可以在同一份 Markdown artifact 里看到推荐切片、发布阻塞和 `Candidate Promotion Tasks`，更容易把候选案例拆成可执行贡献任务。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.3 --json` 输出 `candidate_promotions`；Markdown report 输出 `Candidate Promotion Tasks` 表，包含每个 candidate 的 `adapter_package`、`metadata_validation` 和 `online_smoke`。

## 变更分组

### Candidate promotion planning

- `IterationPlan` 新增 `candidate_promotions` 结构化字段。
- JSON 输出列出每个 candidate case 的 `case_id`、`adapter_package`、`metadata_validation` 和 `online_smoke`。
- 默认文本输出增加 `candidate_promotions` 小节。
- Markdown report 增加 `Candidate Promotion Tasks` 表。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明计划器报告会输出 candidate 晋级证据任务。
- `tests/test_plan_next_iteration.py` 固定 JSON 和 Markdown report 中的 candidate promotion 输出。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环里的新说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例数量和状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。计划器会把 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 三个 candidate cases 的晋级证据直接带入输出。

`search-extraction-gap` 继续作为抽取质量 known-gap。本版本只读取 manifest 和 readiness 里的现有证据，不访问 PyPI、npm、crates.io 或任何第三方站点。

## 风险与兼容性

- 这是维护脚本和报告输出 patch，不改变 CLI runtime、adapter schema、JSON envelope 或生成代码。
- `candidate_promotions` 是新增 JSON 字段；已有 `candidate_cases`、`next_actions` 和 `validation_commands` 保持不变。
- 默认验证继续离线运行，不需要真实 LLM key，也不会写入 `~/.cliany-site/`。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.3 --json` 输出 `candidate_promotions`。
2. 运行计划脚本、维护者循环文档和 release draft 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.3`。
4. 将 `pyproject.toml` 版本更新为 `0.16.3`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.3`。
6. 创建 tag：`git tag v0.16.3`。
