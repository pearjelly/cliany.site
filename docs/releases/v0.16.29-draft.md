# v0.16.29 发布草案

**目标版本：** `0.16.29`
**提交范围：** `v0.16.28..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 publish script 自带发布上下文，降低审阅脚本时拿错 tag 或 commit 的风险。

## 用户价值

`scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh` 会生成可审阅的发布脚本。此前脚本只包含 push 命令和远端复核命令；维护者如果想确认这份脚本对应哪个 branch、latest tag 或 HEAD commit，需要再打开 JSON/Markdown 报告。

`v0.16.29` 在脚本顶部增加 `Publication context` 注释，列出 branch、upstream、remote、latest_tag、local_head、tag_commit、ahead_count、behind_count 和 remote_checked。这样维护者审阅脚本时，可以先确认它对应当前本地 release，再决定是否执行 `git push origin master`、`git push origin v0.16.29` 和远端复核命令。

成功标准是：`python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh` 写出的脚本包含 `Publication context`，并展示 `latest_tag`、`local_head`、`tag_commit`、`ahead_count` 和 `remote_checked`，同时原有 publish commands 保持不变。

## 变更分组

### Publication publish script context

- `scripts/check_release_publication.py --publish-script` 写出的 shell 脚本新增只读上下文注释。
- 注释复用 publication report 的本地证据，不新增远端访问，不改变实际执行命令。
- 当没有需要 push 的命令时，脚本仍保留 `echo "No publish commands are needed."` 行。

### 文档与门禁

- [docs/release-cadence.md](../release-cadence.md) 说明 publish script 顶部会带 `Publication context`。
- `tests/test_release_publication.py` 固定脚本上下文字段。
- `tests/test_release_cadence_docs.py` 固定发布节奏文档说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强 release publication handoff 脚本的审阅信息。

## 风险与兼容性

- 这是 shell script 注释增强，不改变 `publish_commands` 的 JSON schema 或命令顺序。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- `--remote` 行为保持不变；默认生成脚本仍只依赖本地 git/upstream 状态。

```bash
pytest tests/test_release_publication.py tests/test_release_cadence_docs.py tests/test_release_draft_docs.py -q --no-cov

python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh
python scripts/release_readiness.py --target-version 0.16.29 --json
```

## 发版步骤

1. 确认 `python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh` 写出的脚本包含 `Publication context`。
2. 确认脚本注释展示 `latest_tag`、`local_head`、`tag_commit`、`ahead_count` 和 `remote_checked`。
3. 确认脚本仍包含原有 branch/tag push 和 `python scripts/check_release_publication.py --remote --json` 命令。
4. 运行 publication、release cadence 文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.29`。
6. 将 `pyproject.toml` 版本更新为 `0.16.29`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.29`。
8. 创建 tag：`git tag v0.16.29`。

## Release Notes 摘要

`v0.16.29` adds publication context comments to generated publish scripts, helping maintainers verify the branch, tag, and commit before manually publishing release refs.
