# v0.16.38 发布草案

**目标版本：** `0.16.38`
**提交范围：** `v0.16.37..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue artifacts README 同时展示 release draft handoff。

## 用户价值

`v0.16.37` 已经让 candidate issue artifacts 首页展示 publication visibility summary，维护者能直接看到为什么最新本地 release 还不适合继续派发候选任务。但 readiness 里另一个常见 blocker 是下一版 release draft 缺失或缺少必备片段；这些信息只出现在 plan JSON、Markdown report 或 release readiness 输出里，打开 artifacts `README.md` 时还不够直观。

`v0.16.38` 在 `scripts/plan_next_iteration.py --issues-dir` 生成的 artifacts `README.md` 中新增 `Release Draft Handoff` 小节，展示 `release_draft_path` 和 `release_draft_issues`。维护者在审阅候选 issue artifacts 时，可以同时看到发布可见性和下一版草案状态，再决定是否先补草案、发布远端，或延后派发。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.38 --issues-dir /tmp/cliany-candidate-issues --json` 写出的 artifacts `README.md` 包含 `Release Draft Handoff`、`release_draft_path` 和 release draft issues；当目标草案通过 gate 时，该小节显示没有 release draft issues。

## 变更分组

### Candidate artifacts release draft handoff

- `scripts/plan_next_iteration.py --issues-dir` 生成的 artifacts `README.md` 新增 `Release Draft Handoff`。
- 该小节展示 `release_draft_path` 和 `release_draft_issues`。
- Review checklist 会要求在 tag 目标版本前确认 release draft issues 已处理或明确延后。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifacts README 会展示 release draft handoff。
- `tests/test_plan_next_iteration.py` 覆盖 artifacts README 的 release draft handoff。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强候选 issue artifacts README 的 release draft 交接信息。

## 风险与兼容性

- `Release Draft Handoff` 是 artifacts README 的新增展示小节，不改变 plan JSON schema。
- `publication-handoff.json` 结构保持兼容。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- release draft issues 只复用 plan 中已有的 `release_draft_issues`，不重新推断 readiness 结果。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.38 --issues-dir /tmp/cliany-candidate-issues --json` 写出的 README 包含 `Release Draft Handoff`。
2. 确认 `Release Draft Handoff` 展示 `release_draft_path` 和 release draft issues。
3. 运行计划器、维护者循环文档和 release draft 相关测试。
4. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.38`。
5. 将 `pyproject.toml` 版本更新为 `0.16.38`。
6. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.38`。
7. 创建 tag：`git tag v0.16.38`。

## Release Notes 摘要

`v0.16.38` adds release draft handoff details to generated candidate issue artifacts README files, so maintainers can see the target draft path and draft issues before creating candidate promotion issues.
