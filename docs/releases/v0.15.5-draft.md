# v0.15.5 发布草案

**目标版本：** `0.15.5`
**提交范围：** `v0.15.4..HEAD`
**当前阻塞：** 需要提交模块 ownership 地图中的 publication audit 入口、文档测试和 release readiness 元数据 gate。

## 用户价值

`v0.15.5` 聚焦贡献者维护入口一致性。发布可见性检查已经进入 release cadence、README 和 contributor starter，但模块 ownership 地图的 Release operations 行仍只列出 `release_readiness`、`check_release_cadence` 和 GitHub workflow。贡献者查看模块地图时，容易把 publication audit 当成旁支脚本，而不是发版操作的一部分。

成功标准是：`docs/module-ownership.md` 的 Release operations 行显式列出 `scripts/check_release_publication.py` 和 `tests/test_release_publication.py`；贡献者文档测试和 `release_readiness.py` 项目元数据 gate 应固定这个入口，避免发布可见性检查从模块地图中漂移。

## 变更分组

### Release operations ownership

- [docs/module-ownership.md](../module-ownership.md) 的 Release operations 主要路径加入 `scripts/check_release_publication.py`。
- Release operations 典型改动加入“发布可见性”。
- Release operations 最小验证加入 `tests/test_release_publication.py`。

### 文档与门禁

- `tests/test_contributor_docs.py` 应检查模块 ownership 地图包含 publication audit 入口。
- `release_readiness.py` 的项目元数据 gate 应检查模块 ownership 地图保留 publication audit 脚本和测试入口。

## 案例库映射

本版本不改变案例库内容，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。Active、candidate 和 known-gap 状态保持不变；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是贡献者文档和 release metadata gate 层面的 patch，不改变 CLI runtime、adapter schema 或 JSON envelope。
- 新增的是模块 ownership 入口，不会自动发布、不推送 tag、不访问 PyPI。

## 发版前验证

```bash
pytest tests/test_contributor_docs.py tests/test_release_readiness.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/release_readiness.py tests/test_contributor_docs.py tests/test_release_readiness.py tests/test_release_draft_docs.py
python scripts/release_readiness.py --target-version 0.15.5 --json
```

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.15.5 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行贡献者文档和 release readiness 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.15.5`。
4. 将 `pyproject.toml` 版本更新为 `0.15.5`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.15.5`。
6. 创建 tag：`git tag v0.15.5`。

## Release Notes 摘要

`v0.15.5` adds the release publication audit to the module ownership map, so contributors can route publication visibility changes through the Release operations owner area and run the right tests.
