# v0.16.71 发布草案

**目标版本：** `0.16.71`
**提交范围：** `v0.16.70..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 `artifact_bundle_summary` 直接携带 blockers 和 action lists 的稳定 hash。

## 用户价值

`v0.16.70` 已经让 `artifact_bundle_summary` 能判断 blockers、next actions 和 publication next actions 的数量。但数量不变时，工具仍然无法知道列表内容是否漂移。

`v0.16.71` 在 `artifact_bundle_summary` 中新增 `blockers_sha256`、`next_actions_sha256` 和 `publication_next_actions_sha256`。Artifacts `README.md` 的 `Artifact Bundle Summary` 也展示这些字段，让维护者和自动化能在展开完整 manifest 前判断待办内容是否变化。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.72 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 blockers/action lists hash，并能从 manifest 中的 `blockers`、`next_actions` 和 `publication_next_actions` 重新计算得到；README 的 `Artifact Bundle Summary` 也展示这些字段。

## 变更分组

### Artifact bundle action drift summary

- `artifact_bundle_summary` 新增 `blockers_sha256`。
- `artifact_bundle_summary` 新增 `next_actions_sha256`。
- `artifact_bundle_summary` 新增 `publication_next_actions_sha256`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示这些待办漂移摘要字段。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 blockers/action lists hash。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让 candidate issue artifacts 的整包摘要更适合自动化判断待办内容是否变化。

## 风险与兼容性

- 新字段只出现在 `artifact_bundle_summary`，不改变 blockers 或 actions 列表的既有字段语义。
- Hash 基于现有 plan 列表稳定 JSON 序列化计算，不新增网络访问。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.71..HEAD`。最终带 `--release-tag v0.16.71` 的 strict gate 仍以本草案顶部的 `v0.16.70..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.71 --json
python scripts/plan_next_iteration.py --target-version 0.16.71 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.71 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.71 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `blockers_sha256`、`next_actions_sha256` 和 `publication_next_actions_sha256`。
3. 确认这些 hash 能从 manifest 中对应列表重新计算得到。
4. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 展示这些字段。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.71`。
7. 将 `pyproject.toml` 版本更新为 `0.16.71`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.71`。
9. 创建 tag：`git tag v0.16.71`。

## Release Notes 摘要

`v0.16.71` adds blockers and action-list hashes to artifact bundle summaries, helping tools detect maintenance-task drift before reading full issue artifact manifests.
