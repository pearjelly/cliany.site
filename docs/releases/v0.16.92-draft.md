# v0.16.92 发布草案

**目标版本：** `0.16.92`
**提交范围：** `v0.16.91..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 publication audit 的待办数量贯通到 next iteration plan。

## 用户价值

`v0.16.91` 已经让 `scripts/check_release_publication.py` 直接展示 `next_action_count` 与 `publish_command_count`。但维护者每周首先看的通常是 `scripts/plan_next_iteration.py --json`、默认文本输出或 Markdown report；如果计划器只展示 action/command 列表，仍需要人工数一遍发布待办规模。

`v0.16.92` 在 next iteration plan 中新增 `publication_next_action_count` 与 `publication_publish_command_count`。维护者可以在周初计划里先判断发布同步工作量，再展开 `publication_next_actions` 和 `publication_publish_commands`。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.92 --json` 输出 `publication_next_action_count` 与 `publication_publish_command_count`；默认文本输出展示这两个字段；Markdown report 的 Metric 表也展示这两个字段。

## 变更分组

### Plan publication action and command counts

- `IterationPlan` 新增 `publication_next_action_count`。
- `IterationPlan` 新增 `publication_publish_command_count`。
- JSON 输出、默认文本输出和 Markdown report 都展示这两个数量字段。
- 现有 `publication_next_actions` 和 `publication_publish_commands` 列表保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 next iteration plan 会展示 publication action/command 数量。
- `tests/test_plan_next_iteration.py` 覆盖 JSON、文本输出和 Markdown report。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让周初计划更容易快速判断发布待办规模。

## 风险与兼容性

- 新字段只新增在 next iteration plan 输出中，不改变 publication action/command 列表内容。
- 数量从同一组 publication next actions 和 publish commands 计算，不新增网络访问。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.92..HEAD`。最终带 `--release-tag v0.16.92` 的 strict gate 仍以本草案顶部的 `v0.16.91..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.92 --json
python scripts/plan_next_iteration.py --target-version 0.16.92 --report /tmp/cliany-next-iteration.md
python scripts/release_readiness.py --target-version 0.16.92 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.92 --json` 输出 `publication_next_action_count` 与 `publication_publish_command_count`。
2. 确认默认文本输出展示 `publication_next_action_count` 与 `publication_publish_command_count`。
3. 确认 Markdown report 的 Metric 表展示这两个字段。
4. 运行计划器、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.92`。
6. 将 `pyproject.toml` 版本更新为 `0.16.92`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.92`。
8. 创建 tag：`git tag v0.16.92`。

## Release Notes 摘要

`v0.16.92` adds publication action and publish command counts to next iteration plans, helping maintainers size release synchronization work before opening the full lists.
