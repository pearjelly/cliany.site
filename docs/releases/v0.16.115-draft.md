# v0.16.115 发布草案

**目标版本：** `0.16.115`
**提交范围：** `v0.16.114..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 publication worktree clean 状态补进 candidate issue gate quick summary。

## 用户价值

`v0.16.114` 已经让 candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 展示 publication ahead/behind 数。维护者可以先判断本地分支与远端的同步规模，但如果工作区有未提交改动，仍需要展开 handoff 或 publication audit 才能确认是否可以直接发布。

`v0.16.115` 在 `Candidate Issue Gate Quick Summary` 中新增 `publication_worktree_clean`。维护者打开 README 时，可以先看到工作区是否干净，再决定是清理本地改动、提交当前切片，还是继续执行发布脚本。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.115 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 出现在 `Artifact Bundle Summary` 之前，并展示 `publication_worktree_clean`、`publication_ahead_count` 和 `publication_behind_count`。

## 变更分组

### Candidate issue gate quick summary worktree state

- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_worktree_clean`。
- 该字段来自已有 candidate issue gate evidence。
- 现有 quick summary 的 status、can_create_issues、requires_maintainer_review、reason/action count、primary reason/action、latest tag、publication ahead/behind count、publication remote checked、release draft issue count、release draft path 和 visibility 保持不变。
- 现有 `artifact_bundle_summary` 字段保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 quick summary 会展示 publication worktree clean。
- `tests/test_plan_next_iteration.py` 覆盖 README quick summary 的 publication worktree clean 行。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新增的 README 行只读取已有 candidate issue gate evidence。
- `Artifact Bundle Summary` 字段不删除、不改名，机器读取路径保持兼容。
- `publication_worktree_clean` 为 `false` 时提醒维护者先提交、stash 或丢弃本地改动，再发布分支和 tag。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.115..HEAD`。最终带 `--release-tag v0.16.115` 的 strict gate 仍以本草案顶部的 `v0.16.114..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.115 --json
python scripts/plan_next_iteration.py --target-version 0.16.115 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.115 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.115 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 `publication_worktree_clean`。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.115`。
7. 将 `pyproject.toml` 版本更新为 `0.16.115`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.115`。
9. 创建 tag：`git tag v0.16.115`。

## Release Notes 摘要

`v0.16.115` adds publication worktree clean status to the Candidate Issue Gate Quick Summary in generated candidate issue artifacts README files, helping maintainers see whether local changes must be resolved before publishing.
