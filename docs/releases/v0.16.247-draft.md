# v0.16.247 发布草案

**目标版本：** `0.16.247`
**提交范围：** `v0.16.246..HEAD`
**当前阻塞：** 本草案只准备下一版；在 candidate adapter package、metadata validation、online smoke 或明确延后记录完成前，不创建或推送 `v0.16.247` tag。

## 用户价值

`v0.16.246` 已完成 Q3 路线图校准并公开发布，当前 release publication audit 为绿，`master` 与 `v0.16.246` tag 对齐。下一版应回到真实案例库主线，把 `pypi-project-search` 作为首要 candidate 晋级对象，继续补齐 adapter package、metadata validation 和 online smoke 三段证据。

`v0.16.247` 的目标不是扩大功能面，而是把 candidate 晋级流程推进到可验证状态：先运行 live LLM preflight，只有 `generate_adapters.ready=true` 时才允许执行真实 `explore`；如果 LLM 上游仍不可用，则将预检结果作为阻塞证据沉淀在本草案中，保持 candidate 状态不变，避免伪造 adapter package 或 online smoke。

## 变更分组

- Release draft：新增本草案，明确 `v0.16.247` 聚焦真实案例库 candidate evidence，而不是路线图或基础设施扩展。
- Candidate gate：继续使用 `cliany-site doctor --llm-live --json` 作为 `pypi-project-search` explore 前置门禁。
- Changelog：`CHANGELOG.md` 的 Unreleased 记录本次 `v0.16.247` 准备，正式发布时再移动到 `0.16.247` 小节。

## Candidate preflight evidence

`pypi-project-search` 晋级命令计划：

```bash
python -m cliany_site doctor --llm-live --json
cliany-site explore "https://pypi.org" "search Python packages for cliany-site and list project names" --json
python scripts/validate_cases.py --packages-dir ~/.cliany-site/packages --include-candidate-packages --strict
cliany-site pypi.org search-projects --query cliany-site --limit 5 --json
```

执行真实 `explore` 前必须先运行 live LLM preflight。如果 `llm_live` 返回 `E_LLM_UNAVAILABLE`，或 `ready_for_explore=false` / `generate_adapters.ready=false`，本轮停止真实 `explore`，不生成或伪造 `pypi.org-<version>.cliany-adapter.tar.gz`，并将结果记录为阻塞证据。

2026-06-27 已按 candidate promotion contract 重试 live LLM preflight。Chrome/CDP、LLM key、provider、runtime dirs、registry 和 browser provider 可用，但 `llm_live` 返回 `E_LLM_UNAVAILABLE`，`status_code=504`，`phase=llm_preflight`，`ready_for_explore=false`，且 `generate_adapters.ready=false`。因此本轮停止 `pypi-project-search` 的真实 `explore`，不生成或伪造 candidate adapter package，并保持三段晋级证据为 pending。

2026-06-27 10:53 +08:00 再次重试 live LLM preflight，结果仍为 `E_LLM_UNAVAILABLE`，`status_code=504`，`phase=llm_preflight`，`ready_for_explore=false`，且 `generate_adapters.ready=false`。Chrome/CDP、LLM key、provider、runtime dirs、registry 和 browser provider 仍可用。由于生成 adapter 的 live LLM gate 未通过，本轮继续停止 `pypi-project-search` 的真实 `explore`，不生成或伪造 candidate adapter package。

## 风险与兼容性

- 本草案本身是文档与发布规划 patch，不改变 CLI 参数、JSON envelope、adapter metadata schema、AXTree 语义匹配、action runtime、session 存储或 marketplace package 格式。
- Candidate 晋级证据不得用 preflight 失败代替 adapter package；失败只能解释为什么保持 pending。
- 禁止引入脆弱 CSS selector 兜底，后续 adapter 生成仍必须依赖 AXTree 语义匹配。
- 运行时状态仍必须写入 `~/.cliany-site/`，不得把 adapters、sessions、snapshots 或 packages 写进 repo。

```bash
python scripts/release_readiness.py --strict --release-tag v0.16.247 --remote --remote-name origin
```
