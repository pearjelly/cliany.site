# v0.15.4 发布草案

**目标版本：** `0.15.4`
**提交范围：** `v0.15.3..HEAD`
**当前阻塞：** 需要提交 README 双语 publication audit 入口、文档测试和 release readiness 元数据 gate。

## 用户价值

`v0.15.4` 聚焦维护者默认入口的一致性。`scripts/check_release_publication.py` 已经能检查本地 release tag 是否公开可见，也能生成 Markdown report；但 README 的路线图入口仍只提示 `release_readiness` 和 `check_release_cadence`，维护者容易继续忽略“本地已 tag 但远端未发布”的状态。

成功标准是：README 和 README.zh 的路线图段落同时指向 `scripts/check_release_publication.py --json`，并说明它用于确认最新本地 tag 是否已经公开可见；`release_readiness.py` 的项目元数据 gate 和 README 测试应固定这个入口，避免后续维护中漂移。

## 变更分组

### README publication audit 入口

- [README.md](../../README.md) 的 Roadmap 段落加入 `scripts/check_release_publication.py --json`。
- [README.zh.md](../../README.zh.md) 的路线图段落加入 `scripts/check_release_publication.py --json`。
- 文案明确 publication audit 用于确认最新本地 tag 是否公开可见，而不是替代 release readiness 或 cadence 检查。

### 文档与门禁

- `tests/test_readme_current_features.py` 应检查 README 双语入口包含 publication audit 命令。
- `release_readiness.py` 的项目元数据 gate 应检查 README 双语入口保留 publication audit 命令。

## 案例库映射

本版本不改变案例库内容，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。Active、candidate 和 known-gap 状态保持不变；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是 README 和 release metadata gate 层面的 patch，不改变 CLI runtime、adapter schema 或 JSON envelope。
- publication audit 入口只是提示维护者读取状态，不会自动 push、创建 release 或访问 PyPI。

## 发版前验证

```bash
pytest tests/test_readme_current_features.py tests/test_release_readiness.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/release_readiness.py tests/test_readme_current_features.py tests/test_release_readiness.py tests/test_release_draft_docs.py
python scripts/release_readiness.py --target-version 0.15.4 --json
```

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.15.4 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行 README 和 release readiness 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.15.4`。
4. 将 `pyproject.toml` 版本更新为 `0.15.4`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.15.4`。
6. 创建 tag：`git tag v0.15.4`。

## Release Notes 摘要

`v0.15.4` adds the release publication audit to the bilingual README roadmap entry, so maintainers see readiness, cadence, and publication visibility checks from the default project entrypoint.
