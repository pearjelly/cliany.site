# v0.16.128 发布草案

**目标版本：** `0.16.128`
**提交范围：** `v0.16.127..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 publication publish script command 补进 candidate issue gate quick summary。

## 用户价值

`v0.16.127` 已经让维护者能在 `Candidate Issue Gate Quick Summary` 中先看到 publication publish command 数量。这个数量能判断发布命令规模，但维护者仍需要继续展开 `Publication Handoff` 或 `Publication Publish Script` 才能看到生成可审阅发布脚本的命令。

`v0.16.128` 在 `Candidate Issue Gate Quick Summary` 中新增 `publication_publish_script_command`。维护者打开 candidate issue artifacts 的 `README.md` 时，可以第一屏直接看到如何生成 `/tmp/cliany-publish-release.sh`，先审阅脚本，再决定是否执行 branch/tag push 和远端复核。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.128 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 出现在 `Artifact Bundle Summary` 之前，并展示 `publication_publish_command_count` 与 `publication_publish_script_command`。

## 变更分组

### Candidate issue gate quick summary publish script command

- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_publish_script_command`。
- 该字段来自已有 `IterationPlan.publication_publish_script_command`。
- 现有 `Publication Publish Script` 小节、`publication-handoff.json` 和 `artifact-manifest.json` 字段保持兼容。
- 现有 quick summary 的 ok、blocker/action count、publication publish command 数量、visibility summary 和 publication refs 保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 quick summary 会展示 publication publish script command。
- `tests/test_plan_next_iteration.py` 覆盖 README quick summary 的发布脚本生成命令行。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新增 README 行只读取已有 plan 状态。
- 现有 JSON schema、publication publish command 列表和 `Publication Publish Script` 小节保持兼容。
- Quick Summary 中的命令只用于提示如何生成可审阅脚本，不替代完整脚本内容审阅。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.128..HEAD`。最终带 `--release-tag v0.16.128` 的 strict gate 仍以本草案顶部的 `v0.16.127..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.128 --json
python scripts/plan_next_iteration.py --target-version 0.16.128 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.128 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.128 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 `publication_publish_script_command`。
5. 确认 `Publication Publish Script` 小节仍展示同一条脚本生成命令。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.128`。
8. 将 `pyproject.toml` 版本更新为 `0.16.128`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.128`。
10. 创建 tag：`git tag v0.16.128`。

## Release Notes 摘要

`v0.16.128` adds the publication publish script command to the Candidate Issue Gate Quick Summary in generated candidate issue artifacts README files, helping maintainers generate and review the release script before expanding the full publication handoff.
