# v0.15.9 发布草案

**目标版本：** `0.15.9`
**提交范围：** `v0.15.8..HEAD`
**当前阻塞：** 需要提交 case proposal 模板中的 candidate promotion 入口、release readiness 元数据 gate 和测试。

## 用户价值

`v0.15.9` 聚焦外部贡献入口的一致性。`v0.15.8` 已经让 `scripts/validate_cases.py --report` 输出可复制的 `Issue Body Template`，但 GitHub 的 `Real Demo Case Proposal` 仍只收集目标 URL、只读工作流、期望命令、离线样例和验证方式。外部用户提交候选案例时，如果没有同步说明 `adapter_package`、`metadata_validation`、`online_smoke` 证据，维护者后续还要重新整理晋级 active 的 issue。

成功标准是：`.github/ISSUE_TEMPLATE/case_proposal.yml` 新增可选 `promotion` 字段，并在描述中指向 `Candidate Promotion Tasks` / `Issue Body Template`；`docs/contributor-starter.md` 说明真实 demo 候选可沿用这三个晋级证据字段；`release_readiness.py` 的项目元数据 gate 固定这些入口，避免模板漂移。

## 变更分组

### Case proposal promotion evidence

- `.github/ISSUE_TEMPLATE/case_proposal.yml` 新增 `promotion` textarea，收集 `adapter_package`、`metadata_validation`、`online_smoke`。
- 字段描述指向 `Candidate Promotion Tasks` / `Issue Body Template`，让提案和 case report 的后续晋级任务使用同一套结构。
- placeholder 明确包名、`python scripts/validate_cases.py --packages-dir ~/.cliany-site/packages --strict` 和在线只读 smoke 的质量证据。

### 文档与门禁

- [docs/contributor-starter.md](../contributor-starter.md) 的真实 demo 候选行补充 candidate promotion 证据入口。
- `scripts/release_readiness.py` 的项目元数据 gate 现在要求 case proposal 模板和 contributor starter 保留 `Candidate Promotion Tasks` / `Issue Body Template`。
- `tests/test_issue_templates.py`、`tests/test_contributor_docs.py` 和 `tests/test_release_readiness.py` 固定这些入口。

## 案例库映射

本版本不新增案例数量，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。Active 案例保持 4 个，candidate 案例保持 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 三个；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是 GitHub issue 模板、贡献者文档和 release readiness gate 的 patch，不改变 CLI runtime、adapter schema、JSON envelope 或 `validate_cases.py --json` 输出结构。
- `promotion` 字段是可选字段，不会阻止用户提交早期真实案例提案。
- 默认验证继续离线运行，不需要真实 LLM key，也不会访问第三方站点。

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.15.9 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行 issue template、贡献者文档和 release readiness 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.15.9`。
4. 将 `pyproject.toml` 版本更新为 `0.15.9`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.15.9`。
6. 创建 tag：`git tag v0.15.9`。

## Release Notes 摘要

`v0.15.9` connects real demo case proposals to the candidate promotion workflow. The GitHub case proposal template now captures optional package, metadata validation, and online smoke evidence using the same structure as case catalog report issue body templates.
