# v0.16.121 发布草案

**目标版本：** `0.16.121`
**提交范围：** `v0.16.120..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 candidate issue gate quick summary 的维护者文档契约补完整。

## 用户价值

`v0.16.120` 已经让 candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 展示 publication tag commit。当前 quick summary 实际还展示 publication upstream HEAD、tag state、published state 和 remote refs，但维护者循环文档仍只用较粗的 local HEAD/tag commit 描述这些发布状态字段。

`v0.16.121` 明确记录 `Candidate Issue Gate Quick Summary` 会展示 `publication_upstream_head`、`publication_tag_points_at_head`、`publication_tag_commit_in_upstream`、`publication_branch_published`、`publication_tag_published`、`publication_remote_branch_head` 和 `publication_remote_tag_commit`。维护者打开周循环说明时，可以直接知道 quick summary 第一屏覆盖哪些发布判断，不必再回读 generated README 或完整 `artifact_bundle_summary`。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.121 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 仍包含 `Candidate Issue Gate Quick Summary`，维护者循环文档明确说明 publication upstream HEAD、tag state、published state 和 remote refs，且 release draft 测试固定这些字段。

## 变更分组

### Candidate issue gate quick summary documentation contract

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 明确 quick summary 会展示 publication upstream HEAD。
- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 明确 quick summary 会展示 publication tag state。
- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 明确 quick summary 会展示 publication published state。
- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 明确 quick summary 会展示 publication remote refs。
- 现有 `Candidate Issue Gate Quick Summary` 输出字段不改名、不删除。

### 文档与门禁

- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明中的 quick summary 字段族。
- `tests/test_release_draft_docs.py` 固定本草案。
- 本版本不修改 `scripts/plan_next_iteration.py` 的生成逻辑。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 文档只描述现有 quick summary 字段，不改变 artifacts schema。
- 现有 `artifact_bundle_summary` 字段保持不变。
- 维护者文档新增的是字段族说明，不影响 CLI 输出和发布审计行为。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.121..HEAD`。最终带 `--release-tag v0.16.121` 的 strict gate 仍以本草案顶部的 `v0.16.120..HEAD` 作为实际 release diff 范围。

```bash
python scripts/plan_next_iteration.py --target-version 0.16.121 --json
python scripts/plan_next_iteration.py --target-version 0.16.121 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.121 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.121 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认维护者循环文档说明 publication upstream HEAD、tag state、published state 和 remote refs。
4. 运行维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.121`。
6. 将 `pyproject.toml` 版本更新为 `0.16.121`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.121`。
8. 创建 tag：`git tag v0.16.121`。

## Release Notes 摘要

`v0.16.121` documents the full Candidate Issue Gate Quick Summary publication field families in the weekly maintainer loop, helping maintainers understand which HEAD, tag, published-state, and remote-ref checks are visible before expanding generated artifacts.
