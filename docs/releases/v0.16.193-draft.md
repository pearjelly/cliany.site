# v0.16.193 发布草案

**目标版本：** `0.16.193`
**提交范围：** `v0.16.192..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 next action tail 显式化到整包摘要。

## 用户价值

`v0.16.192` 已经让 `artifact_bundle_summary` 输出 next action preview，维护者和自动化可以只读 summary 看到下一步行动列表入口。但当 action list 增长时，只看前缀仍可能漏掉列表末尾新增的发布、草案或候选案例待办。

`v0.16.193` 在 `artifact_bundle_summary` 中新增 `next_action_tail_count`、`next_action_tail` 和 `next_action_tail_sha256`。这样维护者和自动化可以只读整包摘要，同时检查 next actions 的前缀、末尾和完整 hash，再决定是否展开完整 `artifact-manifest.json.next_actions` 或计划 JSON。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.193 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 next action tail fields，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且 `next_action_tail` 等于 `artifact-manifest.json.next_actions` 的末尾片段。

## 变更分组

### Next action tail

- `artifact_bundle_summary` 新增 `next_action_tail_count`。
- `artifact_bundle_summary` 新增 `next_action_tail`。
- `artifact_bundle_summary` 新增 `next_action_tail_sha256`。
- Tail 从 `artifact-manifest.json.next_actions` 的顺序派生。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示 next action tail count/list/hash。
- 现有 `next_action_count`、`next_actions_sha256`、next action preview、blocker boundary、blocker preview、blocker tail、publication visibility 摘要、artifact files 摘要、issue metadata 摘要、review order 摘要、issue body summary 与 inventory 摘要、artifact manifest schema/key/payload 摘要、handoff 摘要和 gate 摘要保持兼容。
- 不改变 release draft validation、publication audit 或 candidate issue gate 的判断逻辑。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示 next action tail fields。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的 next action tail 字段，并断言 tail 与 `artifact-manifest.json.next_actions` 同源。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的整包摘要可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有 `next_actions` 列表，不新增网络访问或外部状态读取。
- 旧工具忽略未知 summary 字段即可继续读取现有 manifest。
- 新工具可以只读 `next_action_preview_count`、`next_action_preview`、`next_action_preview_sha256`、`next_action_tail_count`、`next_action_tail`、`next_action_tail_sha256`、`next_action_count` 和 `next_actions_sha256` 快速确认 next actions 是否漂移。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.193 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `next_action_tail_count`、`next_action_tail` 和 `next_action_tail_sha256`。
3. 确认 artifacts README 的 `Artifact Bundle Summary` 展示这三个字段。
4. 确认 `next_action_tail` 等于 `artifact-manifest.json.next_actions` 的末尾片段。
5. 确认 `next_action_preview`、`next_action_count`、`next_actions_sha256` 和完整 `next_actions` 列表仍保持同源。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.193`。
8. 将 `pyproject.toml` 版本更新为 `0.16.193`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.193`。
10. 创建 tag：`git tag v0.16.193`。

## Release Notes 摘要

`v0.16.193` adds next action tail count/list/hash fields to the artifact bundle summary, helping tools inspect release plan action suffixes before expanding the full manifest.
