# v0.16.141 发布草案

**目标版本：** `0.16.141`
**提交范围：** `v0.16.140..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把完整 release draft 修复动作列表补进 release draft handoff。

## 用户价值

`v0.16.140` 已经让 `release-draft-handoff.json` 能直接展示第一条草案阻塞的处理动作。维护者和自动化只读 handoff 时，如果 release draft 同时有多条问题，仍需要自行遍历 `release_draft_issues` 并拼出完整动作列表。

`v0.16.141` 在 `release-draft-handoff.json` 中新增 `release_draft_required_action_count`、`release_draft_required_actions_sha256` 和 `release_draft_required_actions`，并在 artifacts `README.md` 的 `Release Draft Handoff` 段落展示同源字段。这样只读取 handoff 的工具可以一次性拿到完整草案修复动作，并用 hash 判断动作列表是否漂移。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.141 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `release-draft-handoff.json` 包含完整 required actions 列表、数量和 hash，artifacts README 的 `Release Draft Handoff` 展示同一组字段，`artifact_bundle_summary` 中的 `release_draft_handoff_key_count` 和 `release_draft_handoff_sha256` 随新增字段稳定更新。

## 变更分组

### Release draft handoff required actions

- `release-draft-handoff.json` 新增 `release_draft_required_action_count`。
- `release-draft-handoff.json` 新增 `release_draft_required_actions_sha256`。
- `release-draft-handoff.json` 新增 `release_draft_required_actions`。
- 每条动作使用 `Resolve release draft issue: {issue}`，并完全派生自既有 `release_draft_issues`。
- 无 release draft issue 时，动作列表为空，数量为 `0`。
- Artifacts `README.md` 的 `Release Draft Handoff` 段落展示 required actions 数量、hash 和列表。
- 现有 `schema_version`、`release_draft_ok`、`release_draft_issue_count`、`release_draft_primary_issue`、`release_draft_primary_required_action`、`release_draft_path_sha256`、`release_draft_issues_sha256` 和 `artifact_bundle_summary` 保持兼容。
- `release_draft_handoff_key_count` 和 `release_draft_handoff_sha256` 会随 handoff 结构变化稳定更新。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `release-draft-handoff.json` 和 README handoff 会展示完整 `release_draft_required_actions`。
- `tests/test_plan_next_iteration.py` 覆盖 handoff JSON、README handoff 和 bundle summary 的新字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 release draft handoff 的完整草案处理动作可见性。

## 风险与兼容性

- 新字段只派生自既有 `release_draft_issues`，不新增网络访问或外部状态读取。
- 无 release draft issue 时字段保持空列表语义，方便机器读取。
- required actions 用于快速执行草案修复，不替代完整 release draft issue 审阅。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.141 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `release-draft-handoff.json` 包含 `release_draft_required_action_count`。
3. 确认 `release-draft-handoff.json` 包含 `release_draft_required_actions_sha256`。
4. 确认 `release-draft-handoff.json` 包含 `release_draft_required_actions`。
5. 确认 artifacts README 的 `Release Draft Handoff` 展示 required actions 数量、hash 和列表。
6. 确认 required actions 与 `release_draft_issues` 一一对应。
7. 确认 `artifact_bundle_summary` 的 `release_draft_handoff_key_count` 和 `release_draft_handoff_sha256` 已随新字段更新。
8. 运行计划器、维护者循环文档和 release draft 相关测试。
9. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.141`。
10. 将 `pyproject.toml` 版本更新为 `0.16.141`。
11. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.141`。
12. 创建 tag：`git tag v0.16.141`。

## Release Notes 摘要

`v0.16.141` adds complete release draft required actions to handoff artifacts, helping maintainers and automation act on every draft blocker without rebuilding the action list themselves.
