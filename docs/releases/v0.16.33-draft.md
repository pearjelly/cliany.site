# v0.16.33 发布草案

**目标版本：** `0.16.33`
**提交范围：** `v0.16.32..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 publication audit 直接展示 worktree 状态，避免维护者只在运行 publish script 时才发现本地未提交改动。

## 用户价值

`v0.16.32` 已经让 generated publish script 在 push 前运行 `git status --porcelain`，dirty worktree 会拒绝执行。但维护者通常先看 `python scripts/check_release_publication.py --json`、文本输出或 Markdown report；如果这些入口仍不展示 worktree 状态，就会出现“报告给了 push 命令，脚本执行才失败”的体验落差。

`v0.16.33` 把 worktree 状态提升到 publication report 本身。JSON 输出新增 `worktree_clean` 和 `worktree_status`；文本输出会显示 `worktree_clean` 并在需要时列出 status；Markdown report 新增 `Worktree Status` 小节。dirty worktree 时，`next_actions` 会先提示提交、stash 或丢弃本地改动，`publish_commands` 只保留 `python scripts/check_release_publication.py --json`，避免维护者直接复制 push 命令。

成功标准是：`python scripts/check_release_publication.py --json` 在 dirty worktree 下输出 `worktree_clean: false` 和具体 `worktree_status`；同一状态下 `publish_commands` 不包含 `git push origin master` 或 `git push origin v0.16.33`。

## 变更分组

### Publication worktree visibility

- `scripts/check_release_publication.py` 的 `PublicationReport` 新增 `worktree_clean` 和 `worktree_status`。
- JSON、文本和 Markdown report 都展示 worktree 状态。
- dirty worktree 时，`next_actions` 优先提示清理本地改动，`publish_commands` 收敛为重新运行 publication audit。

### 文档与门禁

- [docs/release-cadence.md](../release-cadence.md) 说明 publication report 会展示 worktree 状态。
- `tests/test_release_publication.py` 覆盖 clean/dirty report 和 dirty 状态下的 publish command 收敛。
- `tests/test_release_cadence_docs.py` 固定发布节奏文档说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强 release publication audit 的本地状态可见性。

## 风险与兼容性

- `worktree_clean` 和 `worktree_status` 是 additive JSON 字段，不移除现有 key。
- dirty worktree 时不再直接输出 push commands；维护者需要先清理本地改动再重新生成 publication audit。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- `--remote` 行为保持不变；worktree 状态只读取本地 git 状态。

## 发版前验证

```bash
pytest tests/test_release_publication.py tests/test_release_cadence_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/check_release_publication.py tests/test_release_publication.py tests/test_release_cadence_docs.py tests/test_release_draft_docs.py
python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh
python scripts/release_readiness.py --target-version 0.16.33 --json
```

## 发版步骤

1. 确认 `python scripts/check_release_publication.py --json` 输出 `worktree_clean` 和 `worktree_status`。
2. 确认 dirty worktree 测试中 `publish_commands` 只保留 `python scripts/check_release_publication.py --json`。
3. 确认 Markdown report 包含 `Worktree Status`。
4. 运行 publication、release cadence 文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.33`。
6. 将 `pyproject.toml` 版本更新为 `0.16.33`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.33`。
8. 创建 tag：`git tag v0.16.33`。

## Release Notes 摘要

`v0.16.33` surfaces worktree status directly in release publication audits and suppresses direct push commands until local changes are committed, stashed, or discarded.
