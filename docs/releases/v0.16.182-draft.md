# v0.16.182 发布草案

**目标版本：** `0.16.182`
**提交范围：** `v0.16.181..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 issue metadata 首尾边界显式化到整包摘要。

## 用户价值

`v0.16.181` 已经让 `artifact_bundle_summary` 输出 issue metadata preview 与 tail，维护工具可以只读 summary 看到候选 issue metadata 前缀和后缀。但如果工具想确认 metadata 首尾边界的稳定摘要，仍需要展开完整 `artifact-manifest.json.issue_metadata_summary` 或 `issue-metadata.json` 后自行计算。

`v0.16.182` 在 `issue_metadata_summary` 中新增 `metadata_first_item`、`metadata_last_item` 和 `metadata_boundary_sha256`，并在 `artifact_bundle_summary` 中新增对应的 `issue_metadata_first_item`、`issue_metadata_last_item` 和 `issue_metadata_boundary_sha256`。这样维护者和自动化可以只读整包摘要，同时检查 candidate issue metadata 首尾、preview、tail 和完整 hash。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.182 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 issue metadata boundary fields，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且 `issue_metadata_first_item` / `issue_metadata_last_item` 等于 `artifact-manifest.json.issue_metadata_summary` 的首尾条目。

## 变更分组

### Issue metadata boundary

- `issue_metadata_summary` 新增 `metadata_first_item`。
- `issue_metadata_summary` 新增 `metadata_last_item`。
- `issue_metadata_summary` 新增 `metadata_boundary_sha256`。
- `artifact_bundle_summary` 新增 `issue_metadata_first_item`。
- `artifact_bundle_summary` 新增 `issue_metadata_last_item`。
- `artifact_bundle_summary` 新增 `issue_metadata_boundary_sha256`。
- Boundary 从稳定 issue metadata 的首尾条目派生。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示 issue metadata first/last item 与 boundary hash。
- 现有 issue metadata preview/tail/count/hash、review order 摘要、issue body summary 与 inventory 摘要、artifact manifest schema/key/payload 摘要、handoff 摘要和 gate 摘要保持兼容。
- 不改变 release draft validation、publication audit 或 candidate issue gate 的判断逻辑。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示 issue metadata boundary fields。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的 issue metadata boundary 字段，并断言 first/last 与 `issue_metadata_summary` 同源。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的整包摘要可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有稳定 issue metadata，不新增网络访问或外部状态读取。
- 旧工具忽略未知 summary 字段即可继续读取现有 manifest。
- 新工具可以只读 `issue_metadata_first_item`、`issue_metadata_last_item`、`issue_metadata_boundary_sha256`、`issue_metadata_preview`、`issue_metadata_tail` 和 `issue_metadata_sha256` 快速确认 metadata 是否漂移。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

python scripts/plan_next_iteration.py --target-version 0.16.182 --issues-dir /tmp/cliany-candidate-issues --json

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.182 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `issue_metadata_first_item`、`issue_metadata_last_item` 和 `issue_metadata_boundary_sha256`。
3. 确认 artifacts README 的 `Artifact Bundle Summary` 展示这三个字段。
4. 确认 `issue_metadata_first_item` 和 `issue_metadata_last_item` 等于 `artifact-manifest.json.issue_metadata_summary` 的首尾条目。
5. 确认 `issue_metadata_preview`、`issue_metadata_tail`、`issue_metadata_count`、`issue_metadata_sha256` 和完整 `issue_metadata_summary` 仍保持同源。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.182`。
8. 将 `pyproject.toml` 版本更新为 `0.16.182`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.182`。
10. 创建 tag：`git tag v0.16.182`。

## Release Notes 摘要

`v0.16.182` adds issue metadata first/last item and boundary hash fields to the artifact bundle summary, helping tools inspect candidate issue metadata boundaries before expanding the full manifest.
