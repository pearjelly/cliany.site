# v0.16.56 发布草案

**目标版本：** `0.16.56`
**提交范围：** `v0.16.55..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue gate 直接给出机器可读 reason codes。

## 用户价值

`v0.16.55` 已经让 `candidate_issue_gate.evidence` 展示 worktree 与 release draft 的布尔门禁状态。维护者可以审计 gate 输入，但脚本如果只想快速判断“为什么不能创建 candidate issue”，仍需要组合 `status`、`summary`、`evidence` 和 publication visibility。

`v0.16.56` 给 `candidate_issue_gate` 增加 `reason_codes`。这些原因码使用稳定字符串，例如 `publication_not_published`、`dirty_worktree`、`local_release_only`、`tag_not_visible`、`needs_remote_check` 和 `release_draft_issues`。自动化可以直接根据 reason codes 决定是先发布本地 release、清理 worktree、远端复核，还是处理 release draft。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.56 --json` 输出的 `candidate_issue_gate.reason_codes` 包含当前 gate 原因；Markdown report 展示 `Candidate Issue Gate Reason Codes`；candidate issue artifacts README 展示 gate reason codes；`artifact-manifest.json` 和 `publication-handoff.json` 保留同源 gate 对象。

## 变更分组

### Candidate issue gate reason codes

- `candidate_issue_gate` 新增 `reason_codes`。
- Publication 未公开时包含 `publication_not_published`。
- Publication visibility 为 `local_only` 时包含 `local_release_only`。
- Release draft 仍有 issues 时包含 `release_draft_issues`。
- Markdown report 新增 `Candidate Issue Gate Reason Codes` 小节。
- Artifacts `README.md` 在 `Publication Handoff` 下展示 gate reason codes。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `reason_codes` 的用途和常见值。
- `tests/test_plan_next_iteration.py` 覆盖 JSON、文本输出、Markdown report 和 artifacts README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强 candidate issue gate 的机器可读原因。

## 风险与兼容性

- `candidate_issue_gate.reason_codes` 是新增字段，不改变 `status`、`required_actions` 或 `evidence` 的语义。
- Reason codes 复用已有 publication audit 与 release readiness 数据，不新增网络访问。
- 不自动执行命令，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
