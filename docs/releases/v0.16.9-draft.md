# v0.16.9 发布草案

**目标版本：** `0.16.9`
**提交范围：** `v0.16.8..HEAD`
**当前阻塞：** 需要提交 publication audit `publish_commands`、release cadence 文档入口和测试；最新本地 release 仍需维护者确认后 push 到远端。

## 用户价值

`v0.16.8` 已经完成本地 tag，但 publication audit 会继续提示分支和 tag 尚未公开。维护者虽然能从 `next_actions` 看出要 push branch/tag，仍需要手工拼出命令，并在 push 后记得重新跑远端校验。

`v0.16.9` 给 `scripts/check_release_publication.py` 增加 `publish_commands`：JSON、默认文本和 Markdown report 都会列出维护者可审阅执行的命令，例如 `git push origin master`、`git push origin v0.16.9` 和 `python scripts/check_release_publication.py --remote --json`。这不自动发布远端，只把“本地 release 已 tag 但未公开”的交接动作变成可复制 artifact。

成功标准是：本地存在未公开 release commit/tag 时，`python scripts/check_release_publication.py --json --report /tmp/cliany-release-publication.md` 输出 `publish_commands`，报告包含 `## Publish Commands`，并且命令块包含 branch push、tag push 和远端复核命令。

## 变更分组

### Publication handoff commands

- `scripts/check_release_publication.py` 的 JSON 输出新增 `publish_commands`。
- 默认文本输出在存在可执行发布交接命令时打印 `publish_commands:`。
- Markdown report 新增 `## Publish Commands`，把命令放进 bash block。
- 命令只用于交接，不由脚本执行；真实 `git push` 仍必须由维护者确认。

### 文档与门禁

- [docs/release-cadence.md](../release-cadence.md) 说明 `publish_commands` 的用途和边界。
- `tests/test_release_publication.py` 固定 JSON、文本和 Markdown report 输出。
- `tests/test_release_cadence_docs.py` 固定 release cadence 文档入口。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例数量和状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本聚焦 release publication handoff，不访问第三方业务站点，不生成 adapter 包，也不会修改案例 manifest 或离线样例。

## 风险与兼容性

- `publish_commands` 是新增字段；已有 publication report 字段、`next_actions` 和 exit code 语义保持兼容。
- 脚本不会执行 `git push`，只把命令写到 stdout、JSON 或 Markdown report。
- 默认模式仍不访问网络；只有维护者显式执行 `python scripts/check_release_publication.py --remote --json` 才读取实时远端 refs。
- 命令使用 shell quoting 生成，避免 remote、branch 或 tag 名称包含特殊字符时报告不可复制。

```bash
pytest tests/test_release_publication.py tests/test_release_cadence_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/check_release_publication.py tests/test_release_publication.py tests/test_release_cadence_docs.py tests/test_release_draft_docs.py
python scripts/check_release_publication.py --json --report /tmp/cliany-release-publication.md
python scripts/release_readiness.py --target-version 0.16.9 --json
```

发布后由维护者显式执行远端复核：

```bash
python scripts/check_release_publication.py --remote --json
```

## 发版步骤

1. 确认 `python scripts/check_release_publication.py --json --report /tmp/cliany-release-publication.md` 输出 `publish_commands`。
2. 确认报告中的 `## Publish Commands` 包含 branch push、tag push 和远端复核命令。
3. 运行 publication、release cadence 文档和 release draft 相关测试。
4. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.9`。
5. 将 `pyproject.toml` 版本更新为 `0.16.9`。
6. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.9`。
7. 创建 tag：`git tag v0.16.9`。

## Release Notes 摘要

`v0.16.9` adds explicit `publish_commands` to the release publication audit. Maintainers now get reviewable branch/tag push commands and a remote verification command in JSON, text, and Markdown reports without the script pushing anything automatically.
