# v0.16.7 发布草案

**目标版本：** `0.16.7`
**提交范围：** `v0.16.6..HEAD`
**当前阻塞：** 需要提交 issue artifact README、维护者循环入口和测试；最新本地 release 仍需维护者确认后 push 到远端。

## 用户价值

`v0.16.6` 让 `scripts/plan_next_iteration.py --issues-dir` 可以离线生成 candidate issue body 文件、`issue-metadata.json` 和 `create-issues.sh`。这些文件已经足够创建 GitHub issue，但目录本身还缺少一个给维护者审阅和交接的入口说明。

`v0.16.7` 为 issue artifacts 目录增加 `README.md`。维护者可以把整个目录作为 CI artifact 或本地交接包，先读 README 了解文件用途、审阅清单、验证命令，以及 `create-issues.sh` 只是生成出来供审阅，不会由计划器自动执行。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.7 --issues-dir /tmp/cliany-candidate-issues` 写出 `README.md`，并且 README 包含 `issue-metadata.json`、`create-issues.sh`、候选 body 文件、审阅清单和离线验证命令。

## 变更分组

### Issue artifact README

- `--issues-dir` 输出新增 `README.md`。
- README 列出 `issue-metadata.json`、`create-issues.sh` 和每个 candidate body 文件。
- README 固定审阅清单：先确认最新 release 已发布、再检查 scope/tasks/evidence/non-goals。
- README 明确 `create-issues.sh` 不会自动执行。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `--issues-dir` 会生成 README。
- `tests/test_plan_next_iteration.py` 固定 README 内容。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环里的新说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例数量和状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。计划器会为 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 三个 candidate cases 写出 issue body 文件，并在 README 中列出它们。

`search-extraction-gap` 继续作为抽取质量 known-gap。本版本只读取 manifest 和 readiness 里的现有证据，不访问第三方站点，不执行真实 adapter 包构建，也不会自动调用 GitHub。

## 风险与兼容性

- 这是维护脚本和 artifact 输出 patch，不改变 CLI runtime、adapter schema、JSON envelope 或生成代码。
- `README.md` 是 `--issues-dir` 的新增输出文件；默认运行不会写这些 artifacts。
- 输出目录由维护者指定，建议使用 `/tmp/...` 或 CI artifact 目录，避免把生成文件提交进仓库。
- 默认验证继续离线运行，不需要真实 LLM key，也不会写入 `~/.cliany-site/`。

```bash
python scripts/plan_next_iteration.py --target-version 0.16.7 --json
python scripts/plan_next_iteration.py --target-version 0.16.7 --report /tmp/cliany-next-iteration.md
python scripts/plan_next_iteration.py --target-version 0.16.7 --issues-dir /tmp/cliany-candidate-issues
python scripts/release_readiness.py --target-version 0.16.7 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.7 --issues-dir /tmp/cliany-candidate-issues` 写出 README、body、metadata 和 create script。
2. 运行计划脚本、维护者循环文档和 release draft 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.7`。
4. 将 `pyproject.toml` 版本更新为 `0.16.7`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.7`。
6. 创建 tag：`git tag v0.16.7`。

## Release Notes 摘要

`v0.16.7` adds a README to the offline candidate issue artifacts generated by the next-iteration planner. Maintainers now get a self-contained review package for candidate promotion issue creation.
