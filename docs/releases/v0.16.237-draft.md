# v0.16.237 发布草案

**目标版本：** `0.16.237`
**提交范围：** `v0.16.236..HEAD`
**当前阻塞：** 需要完成 candidate promotion command plan 的 live preflight 接入，并重新运行 case validation、release readiness、next iteration planning、文档/官网测试、完整 pytest、官网浏览器检查和标准发布流程。真实 LLM provider 预检当前仍可能返回 `502 Bad Gateway`，因此本版不生成或激活任何 candidate adapter。

## 用户价值

v0.16.237 聚焦候选案例晋级 handoff 的可执行性。v0.16.236 已新增 `cliany-site doctor --llm-live --json`，能在 `explore` 前暴露 LLM provider 的真实可用性；但 candidate promotion plan 仍直接从 `adapter_package` 开始，维护者或贡献者容易在 provider 当前 502 时启动长时间真实探索。

本版本把 `llm_live_preflight` 加入 candidate 晋级命令计划的第一步。`cliany-site cases --promotion-plan`、单案例 evidence bundle、`scripts/validate_cases.py` report/JSON 和 `scripts/plan_next_iteration.py` 的 issue artifacts 都会先提示运行 `cliany-site doctor --llm-live --json`。这样 candidate 晋级任务能先判断 provider readiness，再决定继续真实 `explore`、稍后重试或切换 provider。

## 变更分组

- CLI：candidate evidence bundle 现在输出 `llm_live_preflight_command`，Markdown bundle 也新增 LLM live preflight 小节。
- Case validation：candidate `promotion_command_plan` 从三步扩展为四步，新增首项 `llm_live_preflight`；summary 的 expected command count 同步按四步计算。
- Planning：next iteration planner 的 candidate promotion command plan summary、candidate issue body 和 artifact bundle 摘要同步包含 live preflight。
- 文档：README、README.zh、cases README、周维护手册、good-first-issues 和官网 quickstart 同步说明四步命令计划。

## 案例库映射

- 本次不修改 [cases/manifest.json](../../cases/manifest.json) 的任何案例状态。
- 当前真实案例库仍保持 active 4 个、candidate 3 个、known gap 1 个。
- Known gap 保持 1 个：`search-extraction-gap`。
- [cases/README.md](../../cases/README.md) 现在明确：`llm_live_preflight` 是执行前预检，不是 promotion evidence；`adapter_package`、`metadata_validation` 和 `online_smoke` 仍是 candidate 晋级 active 的三段证据。
- `pypi-project-search` 仍是首要 candidate；真实 PyPI adapter 晋级仍等待后续可用 LLM provider、adapter package、metadata validation 和 online smoke 证据。

## 风险与兼容性

- `cliany-site cases` 的 candidate command plan JSON 会多出一个 `llm_live_preflight` item；下游如按 command count 断言三步，需要更新为读取 task 名称或接受四步。
- Candidate evidence status、primary next evidence task 和 status counts 不变，避免把 provider preflight 误当成已完成的 adapter 证据。
- 默认 `doctor` 仍不真实调用 LLM；只有命令计划提示的 `doctor --llm-live --json` 会发起一次 opt-in provider preflight。
- 生成 adapter schema、action runtime、AXTree 语义匹配和 codegen 逻辑不变。
