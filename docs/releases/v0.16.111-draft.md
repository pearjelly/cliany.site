# v0.16.111 发布草案

**目标版本：** `0.16.111`
**提交范围：** `v0.16.110..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 publication ahead 数和 release draft issue 数补进 README 快速摘要。

## 用户价值

`v0.16.110` 已经让 candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 展示 reason/action 数量。维护者可以知道 gate 是否还有更多细节，但仍需要继续读 `Publication Handoff` 才能看到本地 release ahead 规模和 release draft 问题数量。

`v0.16.111` 在 `Candidate Issue Gate Quick Summary` 中新增 `publication_ahead_count` 和 `release_draft_issue_count`。维护者打开 README 时，可以先看到当前本地分支领先远端多少提交，以及下一版 release draft 还有多少个问题，再决定是先发布本地 release、补草案，还是继续展开完整 artifacts。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.111 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 出现在 `Artifact Bundle Summary` 之前，并展示 `publication_ahead_count`、`release_draft_issue_count`、`primary_reason_code` 和 `primary_required_action`。

## 变更分组

### Candidate issue gate quick summary release scale

- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_ahead_count`。
- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `release_draft_issue_count`。
- 现有 quick summary 的 status、can_create_issues、requires_maintainer_review、reason/action count、primary reason/action、latest tag 和 visibility 保持不变。
- 现有 `artifact_bundle_summary` 字段保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 quick summary 会展示 publication ahead 数和 release draft issue 数。
- `tests/test_plan_next_iteration.py` 覆盖 README quick summary 的新增 count 行。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新增的两个 README 行只读取已有 candidate issue gate evidence。
- `Artifact Bundle Summary` 字段不删除、不改名，机器读取路径保持兼容。
- 如果 gate evidence 缺少对应字段，README 会按现有格式显示 `(missing)`。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.111..HEAD`。最终带 `--release-tag v0.16.111` 的 strict gate 仍以本草案顶部的 `v0.16.110..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.111 --json
python scripts/plan_next_iteration.py --target-version 0.16.111 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.111 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.111 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 `publication_ahead_count` 和 `release_draft_issue_count`。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.111`。
7. 将 `pyproject.toml` 版本更新为 `0.16.111`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.111`。
9. 创建 tag：`git tag v0.16.111`。

## Release Notes 摘要

`v0.16.111` adds publication ahead count and release draft issue count to the Candidate Issue Gate Quick Summary in generated candidate issue artifacts README files, helping maintainers estimate release and draft work before expanding the full artifact bundle summary.
