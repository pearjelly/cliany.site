# v0.16.148 发布草案

**目标版本：** `0.16.148`
**提交范围：** `v0.16.147..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把第一条 publication next action 补进 publication handoff。

## 用户价值

`v0.16.147` 已经让 `publication-handoff.json` 能直接展示 `primary_publish_command`。维护工具读取 publication handoff 时，可以显示第一条可执行发布命令；但如果它要展示第一条发布同步待办，仍需要展开完整 `publication_next_actions` 列表。

`v0.16.148` 在 `publication-handoff.json` 中新增 `primary_next_action`，并在 artifacts `README.md` 的 `Publication Handoff` 段落展示同源字段。这样只读取 handoff 的工具可以同时展示第一条发布待办和第一条发布命令，再按需展开完整 `publication_next_actions` 与 `publish_commands`。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.148 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `publication-handoff.json` 包含 `primary_next_action`，artifacts README 的 `Publication Handoff` 展示同一字段，且该值与 `publication_next_actions[0]` 同源；无 publication next action 时字段为 `null` 且 README 展示 `(none)`。

## 变更分组

### Publication handoff primary next action

- `publication-handoff.json` 新增 `primary_next_action`。
- 有 publication next actions 时，该字段取第一条 action。
- 无 publication next action 时，该字段为 `null`；README 用 `(none)` 展示。
- Artifacts `README.md` 的 `Publication Handoff` 展示第一条发布同步待办。
- 现有 `publication_next_actions`、`publication_primary_next_action`、`primary_publish_command`、`artifact_bundle_summary` 和 `publication_handoff_sha256` 保持兼容并反映 handoff 字段变化。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `publication-handoff.json` 与 artifacts README 会展示 `primary_next_action`。
- `tests/test_plan_next_iteration.py` 覆盖 handoff JSON、README 展示和 handoff hash。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的 publication handoff 可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有 `publication_next_actions`，不新增网络访问或外部状态读取。
- 无 publication next action 时字段为 `null`，方便机器读取。
- Primary action 只用于快速展示第一条发布同步待办，不替代完整 `publication_next_actions` 审阅。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.148 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `publication-handoff.json` 包含 `primary_next_action`。
3. 确认 artifacts README 的 `Publication Handoff` 展示 `primary_next_action`。
4. 确认有 publication next action 时，该字段等于 `publication_next_actions[0]`。
5. 确认无 publication next action 时，JSON 字段为 `null` 且 README 展示 `(none)`。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.148`。
8. 将 `pyproject.toml` 版本更新为 `0.16.148`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.148`。
10. 创建 tag：`git tag v0.16.148`。

## Release Notes 摘要

`v0.16.148` adds the primary publication next action to publication handoff artifacts, helping tools show the first release publishing task without expanding the full action list.
