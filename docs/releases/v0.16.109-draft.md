# v0.16.109 发布草案

**目标版本：** `0.16.109`
**提交范围：** `v0.16.108..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 candidate issue gate 的首要摘要提升到 artifacts README 的前半部分。

## 用户价值

`v0.16.108` 已经让 `artifact_bundle_summary` 带上首要 gate reason code、reason description 和 required action。维护者打开 candidate issue artifacts 时，仍然需要先扫过较长的 `Artifact Bundle Summary` 才能看到这些关键信息。

`v0.16.109` 在 candidate issue artifacts `README.md` 中新增 `Candidate Issue Gate Quick Summary`，并放在 `Artifact Bundle Summary` 之前。这个段落直接展示 gate status、`can_create_issues`、`requires_maintainer_review`、primary reason code、primary reason description、primary required action、latest tag 和 visibility，让维护者先看到“为什么不能创建 issue / 下一步做什么”。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.109 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 包含 `Candidate Issue Gate Quick Summary`，该段落出现在 `Artifact Bundle Summary` 之前，并展示 `artifact-manifest.json.artifact_bundle_summary` 中 `candidate_issue_gate_primary_reason_code`、`candidate_issue_gate_primary_reason_description` 和 `candidate_issue_gate_primary_required_action` 对应的人类可读值。

## 变更分组

### Candidate issue gate quick summary

- Candidate issue artifacts `README.md` 新增 `Candidate Issue Gate Quick Summary`。
- 该段落展示 gate status、`can_create_issues`、`requires_maintainer_review`、primary reason/action、latest tag 和 visibility。
- 该段落出现在 `Artifact Bundle Summary` 之前，方便维护者先读结论。
- 现有 `artifact_bundle_summary` 字段保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 README quick summary。
- `tests/test_plan_next_iteration.py` 覆盖 README 段落和顺序。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新段落只读取已有 `candidate_issue_gate`、`publication_visibility` 和 gate evidence。
- `Artifact Bundle Summary` 字段不删除、不改名，机器读取路径保持兼容。
- `primary_required_action` 可能包含反引号；README 使用已有 inline-code 渲染逻辑避免 Markdown 破损。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.109 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 primary reason code、primary reason description 和 primary required action。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.109`。
7. 将 `pyproject.toml` 版本更新为 `0.16.109`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.109`。
9. 创建 tag：`git tag v0.16.109`。
