# v0.16.89 发布草案

**目标版本：** `0.16.89`
**提交范围：** `v0.16.88..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 `artifact_bundle_summary` 直接携带 publication visibility 的稳定摘要。

## 用户价值

`v0.16.88` 已经让 artifacts 复现命令可以被整包摘要快速校验。但维护者在决定是否创建候选 issue 或继续扩大下一版范围前，最常看的阻塞仍是 `publication_visibility`：本地 tag 是否公开、分支是否 ahead、worktree 是否干净，以及 summary 文本是否明确。

`v0.16.89` 在 `artifact_bundle_summary` 中新增 `publication_visibility_sha256` 和 `publication_visibility_summary_sha256`。Artifacts `README.md` 的 `Artifact Bundle Summary` 也展示这两个字段，让工具只读整包摘要就能判断发布可见性对象或 summary 文本是否漂移，再决定是否展开 `publication-handoff.json`。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.89 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 `publication_visibility_sha256` 和 `publication_visibility_summary_sha256`，并能分别从 manifest 中的 `publication_visibility` 和 `publication_visibility.summary` 重新计算得到；README 的 `Artifact Bundle Summary` 也展示这两个字段。

## 变更分组

### Artifact bundle publication visibility hash

- `artifact_bundle_summary` 新增 `publication_visibility_sha256`。
- `artifact_bundle_summary` 新增 `publication_visibility_summary_sha256`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示 publication visibility hash 和 summary hash。
- 现有 `publication_visibility` 字段保持不变；新摘要只用于快速判断发布可见性证据是否需要展开核对。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 publication visibility 摘要。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让 candidate issue artifacts 的整包摘要更适合自动化判断发布可见性是否变化。

## 风险与兼容性

- 新字段只出现在 `artifact_bundle_summary`，不改变 `publication_visibility` 的既有字段语义。
- Hash 基于 publication visibility 对象和 summary 文本的稳定 JSON 序列化计算，不新增网络访问。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.89 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `publication_visibility_sha256` 和 `publication_visibility_summary_sha256`。
3. 确认这两个 hash 能从 `artifact-manifest.json.publication_visibility` 和 `artifact-manifest.json.publication_visibility.summary` 重新计算得到。
4. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 展示这两个字段。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.89`。
7. 将 `pyproject.toml` 版本更新为 `0.16.89`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.89`。
9. 创建 tag：`git tag v0.16.89`。

## Release Notes 摘要

`v0.16.89` adds publication visibility hashes to artifact bundle summaries, helping tools detect release visibility drift before reading full handoff files.
