# v0.16.42 发布草案

**目标版本：** `0.16.42`
**提交范围：** `v0.16.41..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue artifacts manifest 直接展示 blockers 和 next actions。

## 用户价值

`v0.16.41` 已经让 `artifact-manifest.json` 展示 candidate count 和 candidate cases。维护者或工具可以快速判断这批 artifacts 覆盖哪些候选案例，但如果想知道为什么暂时不能派发、下一步要先做什么，仍然要读取 plan JSON、README 或 publication handoff。

`v0.16.42` 让 `artifact-manifest.json` 直接包含 `blockers` 和 `next_actions`。这样一个 JSON 就能概览目标版本、候选案例、阻塞原因、下一步、文件索引和验证命令，更适合作为候选 issue artifacts 的机器可读入口。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.42 --issues-dir /tmp/cliany-candidate-issues --json` 写出的 `artifact-manifest.json` 包含 `blockers` 和 `next_actions`；artifacts `README.md` 的 Files 小节说明 manifest 会包含 blockers 和 next actions。

## 变更分组

### Candidate artifacts manifest handoff summary

- `artifact-manifest.json` 新增 `blockers`。
- `artifact-manifest.json` 新增 `next_actions`。
- Artifacts `README.md` 的 Files 小节说明 manifest 包含 blockers 和 next actions。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 candidate issue artifacts manifest 会包含 blockers 和 next actions。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 内容和 README 文件列表。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强候选 issue artifacts manifest 的交接摘要。

## 风险与兼容性

- `blockers` 和 `next_actions` 是 `artifact-manifest.json` 的新增字段，不改变 plan JSON schema。
- 既有 `candidate_count`、`candidate_cases`、`files`、`review_order` 和 `validation_commands` 字段保持兼容。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- Blockers 和 next actions 只复用当前 plan 的既有字段。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.42 --issues-dir /tmp/cliany-candidate-issues --json` 写出 `artifact-manifest.json`。
2. 确认 manifest 包含 `blockers` 和 `next_actions`。
3. 确认 artifacts `README.md` 的 Files 小节说明 manifest 包含 blockers 和 next actions。
4. 运行计划器、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.42`。
6. 将 `pyproject.toml` 版本更新为 `0.16.42`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.42`。
8. 创建 tag：`git tag v0.16.42`。

## Release Notes 摘要

`v0.16.42` adds blockers and next actions to generated `artifact-manifest.json` files, making the candidate issue artifact bundle easier to inspect from a single machine-readable entrypoint.
