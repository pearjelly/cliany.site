# v0.16.1 发布草案

**目标版本：** `0.16.1`
**提交范围：** `v0.16.0..HEAD`
**当前阻塞：** 需要提交 adapter 包校验 next actions、报告输出和测试。

## 用户价值

`v0.16.1` 继续推进真实案例库一期。`v0.16.0` 已经让每个 case 声明结构化离线验证命令，但当维护者真的把 demo adapter 包下载到 `~/.cliany-site/packages` 后，包校验失败仍只给出 `fail: invalid` 和 issue 列表。这个 patch 把常见失败转换成可执行下一步，减少 release asset 诊断成本。

成功标准是：`python scripts/validate_cases.py --packages-dir ~/.cliany-site/packages --report /tmp/cliany-case-catalog-report.md --strict` 在 domain mismatch、metadata schema、manifest hash、缺失文件或 unsafe archive path 等失败时输出 `next_actions`；Markdown report 的 Package 列应显示 `next:` 建议。

## 变更分组

### Package validation next actions

- `_check_package` 返回 `next_actions` 字段。
- domain mismatch 建议重新生成目标 adapter_domain 的包或修正 case metadata。
- metadata schema 失败建议重新生成 schema v3 metadata。
- hash / manifest 文件列表失败建议重建 adapter 包并同步 `manifest.file_hashes`。
- missing package 和无效 tarball 也返回可执行下一步。

### 文档与门禁

- [cases/README.md](../../cases/README.md) 说明 package validation report 会输出 `next:` 建议。
- `tests/test_validate_cases.py` 固定 package next actions 和 Markdown report 输出。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例数量和状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。Active 案例保持 4 个，candidate 案例保持 3 个；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是 package validation report 的 patch，不改变 CLI runtime、adapter schema 或 JSON envelope。
- `next_actions` 是新增诊断字段；已有 `ok`、`status`、`issues` 语义不变。
- 默认验证继续离线运行，不访问第三方站点，不需要真实 LLM key。

## 发版前验证

```bash
pytest tests/test_validate_cases.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/validate_cases.py tests/test_validate_cases.py tests/test_release_draft_docs.py
python scripts/validate_cases.py --report /tmp/cliany-case-catalog-report.md --strict
python scripts/release_readiness.py --target-version 0.16.1 --json
```

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.16.1 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行案例库和 release draft 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.1`。
4. 将 `pyproject.toml` 版本更新为 `0.16.1`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.1`。
6. 创建 tag：`git tag v0.16.1`。

## Release Notes 摘要

`v0.16.1` adds next actions to adapter package validation reports. Maintainers now get concrete repair suggestions for domain mismatches, metadata schema drift, manifest hash problems, missing files, and invalid tarballs when checking demo adapter release assets.
