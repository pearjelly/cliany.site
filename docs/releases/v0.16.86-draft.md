# v0.16.86 发布草案

**目标版本：** `0.16.86`
**提交范围：** `v0.16.85..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 `artifact_bundle_summary` 直接携带 candidate issue gate 的整体稳定摘要。

## 用户价值

`v0.16.85` 已经让 `artifact_bundle_summary` 能直接展示 `requires_maintainer_review`。但维护者和自动化如果想知道整个 `candidate_issue_gate` 是否变化，仍需要展开完整 manifest 并比对多个子字段。

`v0.16.86` 在 `artifact_bundle_summary` 中新增 `candidate_issue_gate_key_count` 和 `candidate_issue_gate_sha256`。Artifacts `README.md` 的 `Artifact Bundle Summary` 也展示这些字段，让工具只读整包摘要就能判断完整 gate 是否漂移，再决定是否展开 `candidate_issue_gate` 或 Markdown report 的 `Candidate Issue Gate` 小节。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.86 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 `candidate_issue_gate_key_count` 和 `candidate_issue_gate_sha256`，并能从 manifest 中的 `candidate_issue_gate` 重新计算得到；README 的 `Artifact Bundle Summary` 也展示这些字段。

## 变更分组

### Artifact bundle candidate issue gate hash

- `artifact_bundle_summary` 新增 `candidate_issue_gate_key_count` 和 `candidate_issue_gate_sha256`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示 candidate issue gate key count 和 hash。
- 现有 gate 子字段摘要保持不变；这个总 hash 用于快速判断是否需要展开完整 gate。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 candidate issue gate 总摘要。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让 candidate issue artifacts 的整包摘要更适合自动化判断 candidate issue gate 是否变化。

## 风险与兼容性

- 新字段只出现在 `artifact_bundle_summary`，不改变 `candidate_issue_gate` 的既有字段语义。
- Hash 基于 candidate issue gate 稳定 JSON 序列化计算，不新增网络访问。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.86 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `candidate_issue_gate_key_count` 和 `candidate_issue_gate_sha256`。
3. 确认这个 hash 能从 `artifact-manifest.json.candidate_issue_gate` 重新计算得到。
4. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 展示这些字段。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.86`。
7. 将 `pyproject.toml` 版本更新为 `0.16.86`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.86`。
9. 创建 tag：`git tag v0.16.86`。
