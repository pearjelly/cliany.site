# v0.16.246 发布草案

**目标版本：** `0.16.246`
**提交范围：** `v0.16.245..HEAD`
**当前阻塞：** 本周提交日仍未达到 release cadence 的 `3/3` 要求；本草案只准备下一版，不在门禁清除前创建或推送 `v0.16.246` tag。

## 用户价值

`v0.16.245` 之后，项目已经从 CI 稳定化转入 Q3 路线图校准阶段。原 Q3 roadmap 仍以 `v0.14.2` 为基线，并把 `v0.16.0` 放在未来周次；实际发布已经到 `v0.16.245`，且真实案例库、release readiness、candidate evidence、LLM live preflight、Windows/Embodied CI 等基础设施已经完成。

`v0.16.246` 将这次路线图校准沉淀为文档 patch：维护者执行版路线图更新到 `v0.16.245` 后的现实基线，公开版路线图单独新增，README/README.zh 暴露两个入口。这样外部用户可以看到简洁的公开承诺，维护者则继续保留 candidate 晋级、adapter 生命周期、抽取质量、SDK/API/headless 和 1.0 alpha readiness 的可执行计划。

## 变更分组

- Maintainer roadmap：`docs/roadmap-2026-q3.md` 更新为 2026-06-21 校准版，记录已完成基础设施、当前判断、北极星指标、双视图维护规则、产品主线、阶段计划、近期执行队列和发布验证节奏。
- Public roadmap：新增 `docs/public-roadmap.md`，用用户可读语言说明已具备能力、近期真实案例晋级、中期 adapter 生命周期/抽取可靠性、Q3 后段 1.0 alpha readiness，以及不优化方向。
- README entrypoints：README 和 README.zh 在 quickstart 附近增加公开路线图与维护者路线图链接。
- Changelog：`CHANGELOG.md` 的 Unreleased 记录本次路线图更新，下一次真正发布时再移动到 `0.16.246` 小节。

## 案例库映射

- 本次不修改 [cases/manifest.json](../../cases/manifest.json) 的案例状态；相关说明仍落在 [cases/README.md](../../cases/README.md)。
- 当前案例库仍保持 active 4 个、candidate 3 个、known gap 1 个。
- Known gap 保持 1 个：`search-extraction-gap`。
- Candidate 晋级仍需要 adapter package、metadata validation 和 online smoke 三段真实证据。
- 本版不生成 candidate adapter package，也不把任何 candidate 标记为 active；路线图仅把 `pypi-project-search` 作为近期首要晋级对象。

## Candidate preflight evidence

2026-06-23 已按 candidate promotion contract 先运行 live LLM preflight：

```bash
python -m cliany_site doctor --llm-live --json
```

结果显示 Chrome/CDP、LLM key 和 provider 配置可用，但 `llm_live` 返回 `E_LLM_UNAVAILABLE`，`status_code=502`，`phase=llm_preflight`，且 `generate_adapters.ready=false`。因此本轮停止 `pypi-project-search` 的真实 `explore`，不生成或伪造 `pypi.org-<version>.cliany-adapter.tar.gz`，并保持 `adapter_package` 证据为 pending。

## 风险与兼容性

- 这是文档与发布规划 patch，不改变 CLI 参数、JSON envelope、adapter metadata schema、AXTree 语义匹配、action runtime、session 存储或 marketplace package 格式。
- 公开路线图必须保持候选案例边界：缺少 release asset 或 online smoke 时，不把 candidate 描述成 active 能力。
- 维护者路线图继续强调 PR 默认零真实 LLM key、禁止脆弱 CSS selector 兜底、运行时状态不写入 repo。
- 当前 release cadence 仍会阻塞 tag，直到本周至少 3 个独立提交日满足；维护者可以继续准备、验证或推进未发布切片，但不能绕过 cadence gate。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.245..HEAD`。最终带 `--release-tag v0.16.246` 的 strict gate 仍以本草案顶部的 `v0.16.245..HEAD` 作为实际 release diff 范围。

```bash
git diff --check
python scripts/validate_cases.py --strict
pytest tests/test_weekly_maintainer_loop_docs.py tests/test_release_readiness.py -q --no-cov
python scripts/release_readiness.py --json --remote --remote-name origin
```

正式 tag 前再运行：

```bash
CLIANY_QA_OFFLINE=1 pytest tests/ -q
python scripts/release_readiness.py --strict --release-tag v0.16.246 --remote --remote-name origin
```

## 发版步骤

1. 等到本周提交日达到 `3/3`，确认 `release_count_today < max_daily_releases`。
2. 确认维护者路线图、公开路线图、README/README.zh、CHANGELOG 和本草案已同步。
3. 运行文档相关测试、case validation、完整离线 pytest 和 strict release readiness。
4. 将 `CHANGELOG.md` 的 Unreleased 条目移动到 `## [0.16.246] - <date>`，并将 `pyproject.toml` 版本号更新到 `0.16.246`。
5. 提交并推送 `master`。
6. 创建并推送 `v0.16.246` tag。
7. 等待 `.github/workflows/release.yml` 更新 GitHub Release 和 PyPI。
8. 官网源码未改动；本版不需要 Vercel production deploy。
