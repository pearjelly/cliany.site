# v0.16.79 发布草案

**目标版本：** `0.16.79`
**提交范围：** `v0.16.78..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 `artifact_bundle_summary` 直接携带 issue body summary 的稳定 hash。

## 用户价值

`v0.16.78` 已经让 `artifact_bundle_summary` 能检测 candidate case 列表是否漂移。但维护者在展开完整 manifest 前，也需要快速知道 issue body summary 本身是否变化，例如 body 数量、总字节数或 inventory hash 是否漂移。

`v0.16.79` 在 `artifact_bundle_summary` 中新增 `issue_body_summary_sha256`。Artifacts `README.md` 的 `Artifact Bundle Summary` 也展示这个字段，让维护者和自动化能先判断 `issue_body_summary` 是否漂移，再决定是否展开完整 manifest 或 body inventory。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.79 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 `issue_body_summary_sha256`，并能从 manifest 中的 `issue_body_summary` 重新计算得到；README 的 `Artifact Bundle Summary` 也展示这个字段。

## 变更分组

### Artifact bundle issue body summary

- `artifact_bundle_summary` 新增 `issue_body_summary_sha256`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示 issue body summary hash。
- 现有 `issue_body_summary` 和 `Issue Body Summary` 小节保持不变，供需要展开详情的工具继续读取。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 issue body summary hash。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让 candidate issue artifacts 的整包摘要更适合自动化判断 issue body summary 是否变化。

## 风险与兼容性

- 新字段只出现在 `artifact_bundle_summary`，不改变 `issue_body_summary` 的既有字段语义。
- Hash 基于现有 issue body summary 稳定 JSON 序列化计算，不新增网络访问。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.79 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `issue_body_summary_sha256`。
3. 确认这个 hash 能从 manifest 中的 `issue_body_summary` 重新计算得到。
4. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 展示该字段。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.79`。
7. 将 `pyproject.toml` 版本更新为 `0.16.79`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.79`。
9. 创建 tag：`git tag v0.16.79`。

## Release Notes 摘要

`v0.16.79` adds an issue-body-summary hash to artifact bundle summaries, helping tools detect body-summary drift before reading full manifests.
