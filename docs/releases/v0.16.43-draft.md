# v0.16.43 发布草案

**目标版本：** `0.16.43`
**提交范围：** `v0.16.43..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue artifacts manifest 直接展示 publication 摘要和推送命令。

## 用户价值

`v0.16.42` 已经让 `artifact-manifest.json` 展示 blockers 和 next actions。维护者或工具可以从一个入口看到候选任务产物包为什么不能直接派发，但如果要判断发布可见性状态、publication next actions 和应审阅的 push 命令，仍然要继续读取 `publication-handoff.json`。

`v0.16.43` 让 `artifact-manifest.json` 直接包含 `publication_ok`、`publication_visibility`、`publication_next_actions` 和 `publication_publish_commands`。这样工具可以先读 manifest 判断是否要暂停候选 issue 创建，也能把具体 publication 待办和推送命令展示给维护者审阅。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.43 --issues-dir /tmp/cliany-candidate-issues --json` 写出的 `artifact-manifest.json` 包含 publication summary 字段；artifacts `README.md` 的 Files 小节说明 manifest 会包含 publication status、publication next actions 和 publish commands。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强候选 issue artifacts manifest 的 publication 交接摘要。

## 风险与兼容性

- `publication_ok`、`publication_visibility`、`publication_next_actions` 和 `publication_publish_commands` 是 `artifact-manifest.json` 的新增字段，不改变 plan JSON schema。
- 既有 `candidate_count`、`candidate_cases`、`blockers`、`next_actions`、`files`、`review_order` 和 `validation_commands` 字段保持兼容。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- Publication 字段只复用当前 plan 的既有字段和 publication audit 结果。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.43 --issues-dir /tmp/cliany-candidate-issues --json` 写出 `artifact-manifest.json`。
2. 确认 manifest 包含 `publication_ok`、`publication_visibility`、`publication_next_actions` 和 `publication_publish_commands`。
3. 确认 artifacts `README.md` 的 Files 小节说明 manifest 包含 publication status、publication next actions 和 publish commands。
4. 运行计划器、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.43`。
6. 将 `pyproject.toml` 版本更新为 `0.16.43`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.43`。
8. 创建 tag：`git tag v0.16.43`。

## Release Notes 摘要

`v0.16.43` adds publication summary fields to generated `artifact-manifest.json` files, so maintainers and tools can inspect candidate issue bundle publication status and push commands from the same machine-readable entrypoint.
