# v0.16.271 发布草案

**目标版本：** `0.16.271`
**提交范围：** `v0.16.270..HEAD`
**当前状态：** 2026-07-23 release slice；保证合法零匹配的 adapter 合并与分发语义。

完成 version bump 后，target-mode readiness 还会检查占位锚点 **提交范围：** `v0.16.271..HEAD`；该锚点不改变实际 release diff。

## 用户价值

已有 adapter 再次运行 `explore` 合并新命令时，会保留每个 `list-` 或 `search-` 命令的 `expects_nonempty` 语义，并写出完整的 schema-v3 metadata。合法零匹配在打包、安装后仍保持 `ok=true` 与 `data.quality`；默认严格空结果和 partial 质量错误不变。

## 变更分组

### Adapter 合并与 metadata

- 合并命令定义保留 `expects_nonempty` 的实际布尔值。
- 合并写入复用标准 adapter 保存路径，生成 schema-v3 metadata、signature 和现有安全审计字段。
- schema v3 显式把 command-level `expects_nonempty` 定义为 boolean。

### 分发回归

- 合并回归覆盖 `expects_nonempty=false` 和 v3 metadata。
- marketplace pack/install 回归断言安装后的 metadata 仍保留实际布尔值 `false`。

## 案例库映射

本版不改变案例状态或 adapter 资产。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 继续是 candidate；2026-07-23 的真实 `doctor --llm-live` 预检仍返回可重试的 `E_LLM_UNAVAILABLE`，因此没有生成 candidate package、online smoke 或 live explore 成功证据。

## 风险与兼容性

- 缺失 `expects_nonempty` 的历史 metadata 仍按默认严格语义处理。
- `expects_nonempty=false` 只允许 `quality.status=empty` 的合法业务零结果；partial 结果继续是 `E_EMPTY_RESULT`。
- 本版不修改已安装 adapter；只有之后对其运行 `explore` 合并时才采用完整 v3 写入路径。

## 发版步骤

1. 在 `[Unreleased]` 保留本版条目、compare link 为 `v0.16.270...HEAD` 时运行 target-mode readiness。
2. Finalize Changelog：清空 `[Unreleased]`，新增 `## [0.16.271] - 2026-07-23`，并更新 compare links。
3. 提交并推送 `master`，等待 CI 和 Embodied CI 成功。
4. 创建 `git tag v0.16.271`，然后运行 `uv run --extra dev --frozen python scripts/release_readiness.py --strict --release-tag v0.16.271 --remote --remote-name origin`。
5. 仅在该 gate 通过后推送 tag，等待 GitHub Release 与 PyPI 发布。
6. 在 `site/` 运行 `vercel link --yes --project cliany.site` 和 `vercel --prod --yes`；随后核验 `www.cliany.site`。
7. 运行 `uv run --extra dev --frozen python scripts/check_release_publication.py --strict --remote --distribution --json`。
