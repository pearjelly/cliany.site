# v0.16.124 发布草案

**目标版本：** `0.16.124`
**提交范围：** `v0.16.123..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 blocker/next action 数量补进 candidate issue gate quick summary。

## 用户价值

`v0.16.123` 已经让 candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 展示 `publication_ok` 和 `release_draft_ok`。维护者可以先区分发布阻塞和草案阻塞，但仍需要展开完整 `Artifact Bundle Summary` 才能知道本轮到底有多少 blocker、多少 next action，以及 publication audit 自己贡献了多少待办。

`v0.16.124` 在 `Candidate Issue Gate Quick Summary` 中新增 `blocker_count`、`next_action_count` 和 `publication_next_action_count`。维护者打开 README 时，可以先判断这轮是单一发布同步动作，还是有多项阻塞需要拆开处理，再决定是否展开完整 handoff。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.124 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 出现在 `Artifact Bundle Summary` 之前，并展示 `blocker_count`、`next_action_count` 和 `publication_next_action_count`。

## 变更分组

### Candidate issue gate quick summary workload counts

- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `blocker_count`。
- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `next_action_count`。
- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_next_action_count`。
- 这些字段来自已有 plan 状态，不新增 schema，不改变 `artifact_bundle_summary`。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 quick summary 会展示 blocker/next action 数量。
- `tests/test_plan_next_iteration.py` 覆盖 README quick summary 的待办规模字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新增 README 行只读取已有 plan 状态。
- 现有 `Artifact Bundle Summary` 字段不删除、不改名，机器读取路径保持兼容。
- 计数只用于快速判断待办规模，不替代完整 blocker/action 列表。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.124 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 `blocker_count`、`next_action_count` 和 `publication_next_action_count`。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.124`。
7. 将 `pyproject.toml` 版本更新为 `0.16.124`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.124`。
9. 创建 tag：`git tag v0.16.124`。

## Release Notes 摘要

`v0.16.124` adds blocker, next action, and publication next action counts to the Candidate Issue Gate Quick Summary in generated candidate issue artifacts README files, helping maintainers estimate release workload before expanding the full artifact bundle summary.
