# v0.16.208 发布草案

**目标版本：** `0.16.208`
**提交范围：** `v0.16.207..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 release draft issue/action preview 与 tail 显式化到整包摘要。

## 用户价值

`v0.16.207` 已经让 `artifact_bundle_summary` 可以展示 publication next actions 的首尾边界。维护者在审阅 candidate issue artifacts 时，还需要快速判断 release draft issues 与 required actions 的前后窗口是否符合预期；目前 summary 已有数量、完整 hash、首尾边界和 primary action，但只读 summary 的工具无法在问题列表变长时看到前后窗口。

`v0.16.208` 在 `artifact_bundle_summary` 中新增 release draft required actions 与 issues 的 preview/tail 字段。这样维护者和自动化可以只读整包摘要，先检查 release draft blockers 的前后窗口、数量、完整 hash 和首尾边界，再决定是否展开完整 `artifact-manifest.json.release_draft_issues` 或 `release-draft-handoff.json.release_draft_required_actions`。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.208 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 release draft required action preview/tail fields 与 release draft issue preview/tail fields，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且 preview/tail 等于原始列表的前 8 项和后 8 项。

## 变更分组

### Release draft window summary

- `artifact_bundle_summary` 新增 `release_draft_required_action_preview_count`。
- `artifact_bundle_summary` 新增 `release_draft_required_action_preview`。
- `artifact_bundle_summary` 新增 `release_draft_required_action_preview_sha256`。
- `artifact_bundle_summary` 新增 `release_draft_required_action_tail_count`。
- `artifact_bundle_summary` 新增 `release_draft_required_action_tail`。
- `artifact_bundle_summary` 新增 `release_draft_required_action_tail_sha256`。
- `artifact_bundle_summary` 新增 `release_draft_issue_preview_count`。
- `artifact_bundle_summary` 新增 `release_draft_issue_preview`。
- `artifact_bundle_summary` 新增 `release_draft_issue_preview_sha256`。
- `artifact_bundle_summary` 新增 `release_draft_issue_tail_count`。
- `artifact_bundle_summary` 新增 `release_draft_issue_tail`。
- `artifact_bundle_summary` 新增 `release_draft_issue_tail_sha256`。
- Preview 从 release draft required actions 和 issues 的前 8 项派生；tail 从后 8 项派生。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示 release draft required action 与 issue preview/tail fields。
- 现有 `release_draft_required_action_count`、`release_draft_required_actions_sha256`、`release_draft_first_required_action`、`release_draft_last_required_action`、`release_draft_required_action_boundary_sha256`、`release_draft_issues_sha256`、`release_draft_first_issue`、`release_draft_last_issue`、`release_draft_issue_boundary_sha256`、publication next action boundary、publication handoff boundary、validation command boundary 和 review checklist boundary 字段保持兼容。
- 不改变 release draft validation、candidate issue gate、publication audit、publish commands 内容或 candidate case 状态。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的整包摘要可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有 release draft issues 与 required actions，不新增网络访问或外部状态读取。
- 旧工具忽略未知 summary 字段即可继续读取现有 manifest。
- 新工具可以只读 `release_draft_required_action_preview_count`、`release_draft_required_action_preview`、`release_draft_required_action_preview_sha256`、`release_draft_required_action_tail_count`、`release_draft_required_action_tail`、`release_draft_required_action_tail_sha256`、`release_draft_issue_preview_count`、`release_draft_issue_preview`、`release_draft_issue_preview_sha256`、`release_draft_issue_tail_count`、`release_draft_issue_tail` 和 `release_draft_issue_tail_sha256` 快速确认 release draft blocker 窗口是否漂移。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.208 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 release draft required action preview/tail fields。
3. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 release draft issue preview/tail fields。
4. 确认 artifacts README 的 `Artifact Bundle Summary` 展示这些字段。
5. 确认 preview/tail 等于 release draft required actions 与 issues 的前 8 项和后 8 项。
6. 确认 `release_draft_required_actions_sha256`、`release_draft_issues_sha256` 和完整列表仍保持同源。
7. 运行计划器、维护者循环文档和 release draft 相关测试。
8. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.208`。
9. 将 `pyproject.toml` 版本更新为 `0.16.208`。
10. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.208`。
11. 创建 tag：`git tag v0.16.208`。

## Release Notes 摘要

`v0.16.208` adds preview and tail windows for release draft required actions and issues to the artifact bundle summary, helping tools inspect release draft blockers before expanding the full manifest.
