# v0.16.91 发布草案

**目标版本：** `0.16.91`
**提交范围：** `v0.16.90..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 publication audit 直接展示 next actions 和 publish commands 的数量。

## 用户价值

`v0.16.90` 已经让候选 issue artifacts 能校验发布脚本生成命令是否漂移。但维护者直接运行 `scripts/check_release_publication.py` 时，仍需要数一遍 `next_actions` 和 `publish_commands` 才能判断当前发布待办规模。

`v0.16.91` 在 publication audit 的 JSON、默认文本输出和 Markdown report 中新增 `next_action_count` 与 `publish_command_count`。维护者可以先看数量判断发布阻塞复杂度，再展开具体 action 和 command；自动化也能用这两个字段做轻量告警或 dashboard 指标。

成功标准是：本地存在未公开 release commit/tag 时，`python scripts/check_release_publication.py --json` 输出 `next_action_count` 和 `publish_command_count`；默认文本输出展示这两个字段；Markdown report 的 Metric 表也展示这两个字段。

## 变更分组

### Publication audit action and command counts

- `PublicationReport.to_dict()` 新增 `next_action_count`。
- `PublicationReport.to_dict()` 新增 `publish_command_count`。
- 默认文本输出展示 `next_action_count` 和 `publish_command_count`。
- Markdown report 的 Metric 表展示 `next_action_count` 和 `publish_command_count`。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 publication audit 会展示 action/command 数量。
- `tests/test_release_publication.py` 覆盖 JSON、文本输出和 Markdown report。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让 publication audit 更容易被维护者和自动化快速读取。

## 风险与兼容性

- 新字段只新增在 publication audit 输出中，不改变 `next_actions` 和 `publish_commands` 的既有内容。
- 数量从同一组 action/command 列表计算，不新增网络访问。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.91..HEAD`。最终带 `--release-tag v0.16.91` 的 strict gate 仍以本草案顶部的 `v0.16.90..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_release_publication.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/check_release_publication.py tests/test_release_publication.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/check_release_publication.py --json
python scripts/check_release_publication.py --report /tmp/cliany-release-publication.md
python scripts/release_readiness.py --target-version 0.16.91 --json
```

## 发版步骤

1. 确认 `python scripts/check_release_publication.py --json` 输出 `next_action_count` 和 `publish_command_count`。
2. 确认默认文本输出展示 `next_action_count` 和 `publish_command_count`。
3. 确认 Markdown report 的 Metric 表展示这两个字段。
4. 运行 publication audit、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.91`。
6. 将 `pyproject.toml` 版本更新为 `0.16.91`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.91`。
8. 创建 tag：`git tag v0.16.91`。

## Release Notes 摘要

`v0.16.91` adds publication audit action and publish command counts to JSON, text, and Markdown reports so maintainers can size release follow-up work before expanding the full lists.
