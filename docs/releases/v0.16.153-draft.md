# v0.16.153 发布草案

**目标版本：** `0.16.153`
**提交范围：** `v0.16.152..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 release draft handoff 的 primary alias 提升到 artifact bundle summary。

## 用户价值

`v0.16.149` 已经让 `release-draft-handoff.json` 同时输出 `primary_issue` 和 `primary_required_action`，`v0.16.152` 又把 `release_draft_handoff_schema_version` 提升到 `artifact_bundle_summary`。维护工具现在可以只读整包摘要判断 handoff schema，但如果要展示首要 release draft 问题，仍要展开 `release-draft-handoff.json` 或理解 release-draft 专用字段。

`v0.16.153` 在 `artifact_bundle_summary` 中新增 `release_draft_handoff_primary_issue` 和 `release_draft_handoff_primary_required_action`，并在 artifacts `README.md` 的 `Artifact Bundle Summary` 段落展示同源字段。这样工具只读整包摘要时，就能用和 handoff 一致的 primary alias 读取首要草案问题和处理动作。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.153 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 `release_draft_handoff_primary_issue` 和 `release_draft_handoff_primary_required_action`，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且它们分别与 `release-draft-handoff.json.primary_issue` 和 `release-draft-handoff.json.primary_required_action` 同源。

## 变更分组

### Artifact bundle release draft handoff primary aliases

- `artifact_bundle_summary` 新增 `release_draft_handoff_primary_issue`。
- `artifact_bundle_summary` 新增 `release_draft_handoff_primary_required_action`。
- 这两个字段分别从 `release-draft-handoff.json.primary_issue` 和 `release-draft-handoff.json.primary_required_action` 派生。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示这两个字段。
- Artifacts `README.md` 的 `Release Draft Handoff` 段落继续展示同源 `primary_issue` 和 `primary_required_action`。
- 现有 `release_draft_primary_issue`、`release_draft_primary_required_action`、`primary_issue`、`primary_required_action`、`release_draft_handoff_schema_version`、`release_draft_handoff_key_count` 和 `release_draft_handoff_sha256` 保持兼容。
- 不改变 release draft validation、publication audit 或 candidate issue gate 的判断逻辑。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示 release draft handoff primary aliases。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的新字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的整包摘要可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有 `release-draft-handoff.json` primary alias，不新增网络访问或外部状态读取。
- 旧工具忽略未知 summary 字段即可继续读取现有 manifest。
- 新工具可以只读 `artifact_bundle_summary` 展示首要 release draft issue 和 required action，再决定是否展开 handoff。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.153 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `release_draft_handoff_primary_issue` 和 `release_draft_handoff_primary_required_action`。
3. 确认 artifacts README 的 `Artifact Bundle Summary` 展示这两个字段。
4. 确认这两个字段分别等于 `release-draft-handoff.json.primary_issue` 和 `release-draft-handoff.json.primary_required_action`。
5. 确认 `release_draft_handoff_schema_version`、`release_draft_handoff_key_count` 和 `release_draft_handoff_sha256` 仍与当前 handoff 同源。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.153`。
8. 将 `pyproject.toml` 版本更新为 `0.16.153`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.153`。
10. 创建 tag：`git tag v0.16.153`。

## Release Notes 摘要

`v0.16.153` adds release draft handoff primary aliases to artifact bundle summaries, helping tools show the first draft issue and required action without expanding the full handoff payload.
