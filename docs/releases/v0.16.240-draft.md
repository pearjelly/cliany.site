# v0.16.240 发布草案

**目标版本：** `0.16.240`
**提交范围：** `v0.16.239..HEAD`
**当前阻塞：** 真实 LLM provider 预检仍返回 `502 Bad Gateway` / `E_LLM_UNAVAILABLE`，因此本版不生成 candidate adapter package，不运行真实 `explore`，也不把任何 candidate 标记为 active。本版把同一阻断规则提升到 candidate promotion queue，让维护者在安排候选案例工作前就能看到停机条件。

## 用户价值

`v0.16.239` 已经让单个 candidate issue template 和 evidence bundle 明确要求先运行 `cliany-site doctor --llm-live --json`，并在 `generate_adapters.ready=false` 或 `llm_live` 返回 `E_LLM_UNAVAILABLE` 时停止晋级。但维护者日常更可能先打开 `cliany-site cases --promotion-plan` 来看整个候选队列；如果总入口没有同样的阻断说明，就仍可能从队列里的 `explore` 命令直接开始。

`v0.16.240` 把 live LLM preflight gate 加到 promotion plan 顶层、candidate 摘要、primary next item 和每个未完成 task queue item。自动化可以直接读 JSON 中的 `llm_live_preflight_command` / `llm_live_preflight_blocker_note`，人类 Markdown 输出也会在队列顶部、首要任务和候选项里看到同一条规则：live preflight 失败时，贴回 doctor JSON / 错误摘要，保持 `adapter_package` pending 或 blocked，不生成候选 adapter。

## 变更分组

- Cases CLI：`cliany-site cases --promotion-plan --json` 现在在 plan 顶层输出 `llm_live_preflight_command` 和 `llm_live_preflight_blocker_note`。
- Candidate queue：每个 candidate 摘要和 task queue item 都带同一份 LLM blocker handling，方便 issue 机器人、看板同步或维护脚本直接消费。
- Human output：`cliany-site cases --promotion-plan` 的 Markdown 输出展示 LLM live preflight 命令、首要任务的 blocker handling，以及候选队列 / 未完成任务队列里的 `llm_blocker`。
- Tests：`tests/test_cases_command.py` 覆盖 JSON 和人类输出里的 queue-level blocker note，锁住 `E_LLM_UNAVAILABLE` 停机契约。

## 案例库映射

- 本次不修改 [cases/manifest.json](../../cases/manifest.json) 的案例状态；相关说明仍落在 [cases/README.md](../../cases/README.md)。
- `cases/README.md` 已记录 candidate promotion 需要真实 adapter package、metadata validation 和 online smoke 证据。
- 当前案例库仍保持 active 4 个、candidate 3 个、known gap 1 个。
- Known gap 保持 1 个：`search-extraction-gap`。
- Candidate 晋级仍需要三段真实证据：adapter package、metadata validation 和 online smoke。
- `pypi-project-search` 仍是首要 candidate，但当前 live LLM preflight 返回 `E_LLM_UNAVAILABLE` / `502 Bad Gateway`；本版只让 queue-level handoff 更清楚地记录这个 blocker，不生成或激活任何 candidate adapter。

## 风险与兼容性

- 这是 CLI 输出的 additive JSON / Markdown 增强，不改变 adapter schema、AXTree 语义匹配、action runtime、codegen、session 存储或 marketplace package 格式。
- 新增 JSON 字段是 additive：现有读取 `promotion_plan` 的自动化可以忽略 `llm_live_preflight_command` 和 `llm_live_preflight_blocker_note`。
- Markdown 输出会多打印 blocker handling；交接信息更长，但能降低维护者直接从 queue 的 `explore` 命令开始而跳过 live preflight 的风险。

## Release note draft

v0.16.240 raises the live LLM preflight blocker contract from individual candidate evidence bundles into the candidate promotion queue. `cliany-site cases --promotion-plan` now exposes the preflight command and `E_LLM_UNAVAILABLE` handling in JSON and human output, so maintainers and automation can stop promotion before running real `explore` while the upstream LLM provider is unavailable.
