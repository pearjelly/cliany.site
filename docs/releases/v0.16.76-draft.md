# v0.16.76 发布草案

**目标版本：** `0.16.76`
**提交范围：** `v0.16.75..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 `artifact_bundle_summary` 直接携带 create issues safety contract 的 key count 与稳定 hash。

## 用户价值

`v0.16.75` 已经让 `artifact_bundle_summary` 能检测 review checklist 是否漂移。但候选 issue artifacts 的安全契约同样关键：dry-run 是否可用、preflight 是否必须执行、preflight JSON 写到哪里，都会影响维护者是否能安全创建候选任务。

`v0.16.76` 在 `artifact_bundle_summary` 中新增 `create_issues_safety_contract_key_count` 和 `create_issues_safety_contract_sha256`。这个摘要只覆盖稳定的 dry-run/preflight 契约，不包含本机脚本路径，避免不同输出目录产生无意义漂移。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.76 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 create issues safety contract count/hash，并能从 manifest 中的 `create_issues_safety` 提取稳定 contract 后重新计算得到；README 的 `Artifact Bundle Summary` 也展示这些字段。

## 变更分组

### Artifact bundle safety contract summary

- `artifact_bundle_summary` 新增 `create_issues_safety_contract_key_count`。
- `artifact_bundle_summary` 新增 `create_issues_safety_contract_sha256`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示这些 create issues safety contract 漂移摘要字段。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 create issues safety contract count/hash。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让 candidate issue artifacts 的整包摘要更适合自动化判断 create issues safety contract 是否变化。

## 风险与兼容性

- 新字段只出现在 `artifact_bundle_summary`，不改变 `create_issues_safety` 的既有字段语义。
- Hash 基于现有 dry-run/preflight safety contract 稳定 JSON 序列化计算，不包含 `script` 和 `dry_run_command` 里的本机路径。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.76 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `create_issues_safety_contract_key_count` 和 `create_issues_safety_contract_sha256`。
3. 确认这些摘要能从 manifest 中的 `create_issues_safety` 提取稳定 contract 后重新计算得到。
4. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 展示这些字段。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.76`。
7. 将 `pyproject.toml` 版本更新为 `0.16.76`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.76`。
9. 创建 tag：`git tag v0.16.76`。

## Release Notes 摘要

`v0.16.76` adds create-issues safety-contract count and hash fields to artifact bundle summaries, helping tools detect dry-run/preflight drift before reading full issue artifact manifests.
