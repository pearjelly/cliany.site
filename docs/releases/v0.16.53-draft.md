# v0.16.53 发布草案

**目标版本：** `0.16.53`
**提交范围：** `v0.16.52..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 next iteration 默认文本输出展开 candidate issue gate evidence。

## 用户价值

`v0.16.52` 已经给 `candidate_issue_gate` 增加了机器可读 `evidence`，JSON 和 artifacts 都能说明 gate 结论来自哪些 publication/release draft 输入。但维护者周初最常运行的命令之一仍是 `python scripts/plan_next_iteration.py` 的默认文本输出；如果 `evidence` 在终端里只显示为一整段 Python dict，人工审阅仍然费劲。

`v0.16.53` 让默认文本输出把 `candidate_issue_gate.evidence` 展开成缩进列表。维护者无需切到 JSON 或 Markdown report，就能在终端里直接看到 publication visibility status、latest tag、ahead count 和 release draft issue count。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.53` 的文本输出在 `candidate_issue_gate` 下包含 `- evidence:`，并把 `publication_visibility_status`、`publication_latest_tag`、`publication_ahead_count` 和 `release_draft_issue_count` 分行展示；测试固定该输出不会退回单行 dict。

## 变更分组

### Candidate issue gate text output

- `scripts/plan_next_iteration.py` 的文本输出 helper 会展开 dict 值。
- `candidate_issue_gate.evidence` 在默认文本输出中显示为缩进键值列表。
- 列表型 `required_actions` 继续保持逐行输出。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明默认文本输出会展开 gate evidence。
- `tests/test_plan_next_iteration.py` 覆盖文本输出中的 evidence 展开格式。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强 next iteration plan 的终端可读性。

## 风险与兼容性

- JSON schema 不变，`candidate_issue_gate.evidence` 字段不变。
- Markdown report 和 artifacts 输出不变。
- 默认文本输出只改变 nested dict 的展示方式，便于人工审阅。
- 不自动执行命令，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.53..HEAD`。最终带 `--release-tag v0.16.53` 的 strict gate 仍以本草案顶部的 `v0.16.52..HEAD` 作为实际 release diff 范围。

```bash
python scripts/plan_next_iteration.py --target-version 0.16.53
python scripts/release_readiness.py --target-version 0.16.53 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.53` 的文本输出包含 `candidate_issue_gate`。
2. 确认文本输出中的 `evidence` 展开为缩进列表。
3. 确认输出包含 `publication_visibility_status`、`publication_latest_tag`、`publication_ahead_count` 和 `release_draft_issue_count`。
4. 运行计划器、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.53`。
6. 将 `pyproject.toml` 版本更新为 `0.16.53`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.53`。
8. 创建 tag：`git tag v0.16.53`。

## Release Notes 摘要

`v0.16.53` expands `candidate_issue_gate.evidence` in the default next-iteration text output, making gate evidence readable in the terminal without switching to JSON or Markdown reports.
