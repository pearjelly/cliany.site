# v0.16.106 发布草案

**目标版本：** `0.16.106`
**提交范围：** `v0.16.105..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 publication remote ref heads 补进 candidate issue artifacts 的整包摘要。

## 用户价值

`v0.16.105` 已经让 `artifact_bundle_summary` 能展示发布分支和 tag 是否公开可见。维护者继续审阅候选 issue artifacts 时，如果执行过 `python scripts/check_release_publication.py --remote --json`，还需要不展开 `publication_ref_context` 就能看到远端分支和远端 tag 实际指向哪个提交；这能帮助判断“不公开”是缺 tag、分支未同步，还是远端指向了不同 commit。

`v0.16.106` 在 `publication_ref_context` 中保留 `remote_branch_head` 和 `remote_tag_commit`，并在 `artifact_bundle_summary` 中新增 `publication_remote_branch_head` 和 `publication_remote_tag_commit`。Artifacts `README.md` 的 `Artifact Bundle Summary` 也展示这些字段，让工具只读整包摘要就能判断 remote check 的远端 ref 证据。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.106 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 `publication_remote_branch_head` 和 `publication_remote_tag_commit`，且这些值分别等于 manifest 中 `publication_ref_context.remote_branch_head` 和 `publication_ref_context.remote_tag_commit`；README 的 `Artifact Bundle Summary` 也展示这些字段。

## 变更分组

### Artifact bundle publication remote refs

- `publication_ref_context` 新增 `remote_branch_head`。
- `publication_ref_context` 新增 `remote_tag_commit`。
- `artifact_bundle_summary` 新增 `publication_remote_branch_head`。
- `artifact_bundle_summary` 新增 `publication_remote_tag_commit`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示远端分支 HEAD 和远端 tag commit。
- 现有 `publication_branch_published`、`publication_tag_published`、`publication_tag_points_at_head`、`publication_tag_commit_in_upstream` 和 `publication_remote_checked` 保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 publication remote ref heads。
- `tests/test_plan_next_iteration.py` 覆盖 manifest、publication handoff 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让候选 issue artifacts 的整包摘要更容易读取 remote check 证据。

## 风险与兼容性

- 新字段只来自 publication audit 已有的远端 ref 结论，不新增网络访问。
- 未执行 remote check 时，这两个字段可以是 `null`/`None`；执行 remote check 后会反映远端实际 ref。
- `publication_ref_context` 会多出两个键；它们与 `check_release_publication.py` 的 JSON 字段同名。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.106..HEAD`。最终带 `--release-tag v0.16.106` 的 strict gate 仍以本草案顶部的 `v0.16.105..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.106 --json
python scripts/plan_next_iteration.py --target-version 0.16.106 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.106 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.106 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `publication_remote_branch_head` 和 `publication_remote_tag_commit`。
3. 确认这些值分别等于 `artifact-manifest.json.publication_ref_context.remote_branch_head` 和 `artifact-manifest.json.publication_ref_context.remote_tag_commit`。
4. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 展示这些字段。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.106`。
7. 将 `pyproject.toml` 版本更新为 `0.16.106`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.106`。
9. 创建 tag：`git tag v0.16.106`。

## Release Notes 摘要

`v0.16.106` adds remote branch HEAD and remote tag commit fields to candidate issue artifact bundle summaries, helping tools inspect remote publication evidence without expanding the full publication ref context.
