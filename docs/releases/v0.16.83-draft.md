# v0.16.83 发布草案

**目标版本：** `0.16.83`
**提交范围：** `v0.16.82..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 `artifact_bundle_summary` 直接携带 candidate issue gate reason descriptions 的稳定摘要。

## 用户价值

`v0.16.82` 已经让 `artifact_bundle_summary` 能检测 candidate issue gate evidence 是否漂移。但维护者在展开完整 manifest 前，也需要快速知道 `candidate_issue_gate.reason_descriptions` 是否变化，避免原因码文本、人工审阅说明或自动化提示悄悄漂移。

`v0.16.83` 在 `artifact_bundle_summary` 中新增 `candidate_issue_gate_reason_description_count` 和 `candidate_issue_gate_reason_descriptions_sha256`。Artifacts `README.md` 的 `Artifact Bundle Summary` 也展示这些字段，让维护者和自动化能先判断 gate reason descriptions 是否漂移，再决定是否展开完整 `candidate_issue_gate` 或 Markdown report 的 `Candidate Issue Gate Reason Descriptions` 小节。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.83 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 `candidate_issue_gate_reason_description_count` 和 `candidate_issue_gate_reason_descriptions_sha256`，并能从 manifest 中的 `candidate_issue_gate.reason_descriptions` 重新计算得到；README 的 `Artifact Bundle Summary` 也展示这些字段。

## 变更分组

### Artifact bundle gate reason descriptions summary

- `artifact_bundle_summary` 新增 `candidate_issue_gate_reason_description_count` 和 `candidate_issue_gate_reason_descriptions_sha256`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示 gate reason descriptions count 和 hash。
- `Candidate Issue Gate Reason Descriptions` 仍保留完整可审阅说明；新摘要只用于快速判断是否需要展开。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 gate reason descriptions 摘要。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让 candidate issue artifacts 的整包摘要更适合自动化判断 gate reason descriptions 是否变化。

## 风险与兼容性

- 新字段只出现在 `artifact_bundle_summary`，不改变 `candidate_issue_gate.reason_descriptions` 的既有字段语义。
- Hash 基于 gate reason descriptions 稳定 JSON 序列化计算，不新增网络访问。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
