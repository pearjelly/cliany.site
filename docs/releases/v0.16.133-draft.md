# v0.16.133 发布草案

**目标版本：** `0.16.133`
**提交范围：** `v0.16.132..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 publish command count 补进 publication handoff。

## 用户价值

`v0.16.132` 已经让 `publication-handoff.json` 能直接展示发布脚本路径和命令摘要。只读 handoff 的维护工具可以检测脚本漂移，但如果它只想先判断需要审阅几条发布命令，仍要展开 `publish_commands` 数组。

`v0.16.133` 在 `publication-handoff.json` 中新增 `publish_command_count`，并在 artifacts `README.md` 的 `Publication Handoff` 段落展示同一字段。维护者和自动化只读 handoff 文件或 README handoff 小节，就能判断发布命令规模，再决定是否展开完整命令列表。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.133 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `publication-handoff.json` 包含 `publish_command_count`，artifacts README 的 `Publication Handoff` 展示同一字段，`artifact_bundle_summary` 中的 `publication_handoff_key_count` 和 `publication_handoff_sha256` 随新增字段稳定更新。

## 变更分组

### Publication handoff publish command count

- `publication-handoff.json` 新增 `publish_command_count`。
- Artifacts `README.md` 的 `Publication Handoff` 段落展示 `publish_command_count`。
- 字段值来自已有 `IterationPlan.publication_publish_command_count`。
- 现有 `publication_publish_command_count`、`publication_publish_commands` 和 `artifact_bundle_summary` 保持兼容。
- `publication_handoff_key_count` 和 `publication_handoff_sha256` 会随 handoff 结构变化稳定更新。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `publication-handoff.json` 和 README handoff 会展示 `publish_command_count`。
- `tests/test_plan_next_iteration.py` 覆盖 handoff JSON、README handoff 和 bundle summary 的新字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 publication handoff 的发布命令规模可见性。

## 风险与兼容性

- 新增字段只派生自既有 publish commands 列表长度。
- 现有 `publish_commands`、publish script 字段、plan 级 publication publish command count 和 `artifact_bundle_summary` 保持兼容。
- 计数只用于快速判断命令规模，不替代完整命令列表审阅。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.133 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `publication-handoff.json` 包含 `publish_command_count`。
3. 确认 artifacts README 的 `Publication Handoff` 展示 `publish_command_count`。
4. 确认 `publish_command_count` 与 `publication_publish_command_count` 一致。
5. 确认 `artifact_bundle_summary` 的 `publication_handoff_key_count` 和 `publication_handoff_sha256` 已随新字段更新。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.133`。
8. 将 `pyproject.toml` 版本更新为 `0.16.133`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.133`。
10. 创建 tag：`git tag v0.16.133`。

## Release Notes 摘要

`v0.16.133` adds publish command count to publication handoff artifacts, helping maintainers and automation estimate release command review scope from the handoff alone.
