# v0.16.64 发布草案

**目标版本：** `0.16.64`
**提交范围：** `v0.16.63..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue artifacts 提供 `review_order_sha256`，方便工具快速发现审阅顺序漂移。

## 用户价值

`v0.16.63` 已经给 candidate issue artifacts 增加 `artifact_bundle_summary`，让工具能先判断整包状态。但 `review_order` 仍只能通过展开完整 manifest 后逐项比较，自动化很难快速知道审阅顺序有没有变化。

`v0.16.64` 在 `artifact_bundle_summary` 中新增 `review_order_sha256`，对 `review_order` 做稳定 SHA-256 摘要。Artifacts `README.md` 的 `Artifact Bundle Summary` 也展示同一个 hash，让维护者和工具能先检查 review order hash，再决定是否展开完整 `artifact-manifest.json`。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.64 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json` 包含 `artifact_bundle_summary.review_order_sha256`；README 包含 `Artifact Bundle Summary`、`review_order_sha256` 和 `review_item_count`；该 hash 能从 `review_order` 重新计算得到。

## 变更分组

### Candidate issue artifact review order hash

- `artifact_bundle_summary` 新增 `review_order_sha256`。
- Summary 继续保留 target version、candidate/body/review item 数量、inventory hash、gate 状态和脚本安全布尔值。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 新增 `review_order_sha256`。
- Hash 基于 `review_order` 稳定 JSON 序列化计算，不包含本机绝对路径。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `review_order_sha256` 与 review order hash。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让候选 issue artifacts 的审阅顺序更容易被工具和维护者审计。

## 风险与兼容性

- `review_order_sha256` 是新增 summary 字段，不改变现有字段语义。
- Hash 基于已存在的 `review_order` 计算，不新增网络访问。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.64..HEAD`。最终带 `--release-tag v0.16.64` 的 strict gate 仍以本草案顶部的 `v0.16.63..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.64 --json
python scripts/plan_next_iteration.py --target-version 0.16.64 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.64 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.64 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json` 包含 `artifact_bundle_summary.review_order_sha256`。
3. 确认 `artifact_bundle_summary.review_order_sha256` 能从 `review_order` 重新计算得到。
4. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 包含 `review_order_sha256`。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.64`。
7. 将 `pyproject.toml` 版本更新为 `0.16.64`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.64`。
9. 创建 tag：`git tag v0.16.64`。

## Release Notes 摘要

`v0.16.64` adds a review order hash to candidate issue artifact bundle summaries, helping maintainers and tools detect review order changes before expanding the full manifest.
