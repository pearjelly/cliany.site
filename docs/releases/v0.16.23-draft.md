# v0.16.23 发布草案

**目标版本：** `0.16.23`
**提交范围：** `v0.16.22..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让周初计划直接展示 publication audit 的具体 next actions。

## 用户价值

`v0.16.22` 让 next iteration plan 能显示 release draft gate 的具体问题，但 publication gate 仍主要依赖 `publication_publish_commands`。维护者能看到要执行哪些命令，却不一定能在同一份计划里看到为什么要执行：本地分支 ahead 数是多少、tag 尚未发布，还是需要通过 `--remote` 复核 live remote refs。

`v0.16.23` 在 next iteration plan 中新增 `publication_next_actions`。JSON、文本输出和 Markdown report 都会直接透传 publication audit 的具体待办；Markdown report 会新增 `Publication Next Actions` 小节。`scripts/plan_next_iteration.py --issues-dir` 写出的 `publication-handoff.json` 也会包含同一组 publication next actions，让候选 issue 派发前的交接文件同时保留“为什么不能继续”和“应该执行什么命令”。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.23 --json` 输出 `publication_next_actions`；当本地 release 尚未公开可见时，该字段会列出 push branch、push tag 或 remote refs 复核相关待办；Markdown report 和 `publication-handoff.json` 保留同样信息。

## 变更分组

### Publication audit handoff

- `scripts/plan_next_iteration.py` 的 `IterationPlan` 新增 `publication_next_actions` 字段。
- JSON 输出保留 publication audit 的具体 next actions。
- 文本输出在存在 publication 待办时打印 `publication_next_actions` 列表。
- Markdown report 新增 `Publication Next Actions` 小节。
- `--issues-dir` 生成的 `publication-handoff.json` 新增 `publication_next_actions`。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明周初计划会直接暴露 `publication_next_actions`。
- `tests/test_plan_next_iteration.py` 固定 JSON、Markdown report 和 `publication-handoff.json` 输出。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，也不把 candidate 标记为 active；它只提升维护者在 publication gate 前的可见性，避免候选任务派发与 release 发布状态脱节。

## 风险与兼容性

- `publication_next_actions` 是新增字段，不改变既有字段语义。
- 该字段只透传 `scripts/check_release_publication.py` 的 audit 结果，不新增 git remote 访问行为。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.23 --json
python scripts/plan_next_iteration.py --target-version 0.16.23 --report /tmp/cliany-next-iteration.md
python scripts/plan_next_iteration.py --target-version 0.16.23 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.23 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.23 --json` 输出 `publication_next_actions`。
2. 确认 Markdown report 包含 `Publication Next Actions` 小节。
3. 确认 `--issues-dir` 生成的 `publication-handoff.json` 包含 `publication_next_actions`。
4. 运行计划脚本、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.23`。
6. 将 `pyproject.toml` 版本更新为 `0.16.23`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.23`。
8. 创建 tag：`git tag v0.16.23`。

## Release Notes 摘要

`v0.16.23` adds `publication_next_actions` to next iteration plans and candidate issue publication handoffs. Maintainers can now see the exact publication audit tasks alongside publish commands before continuing candidate work.
