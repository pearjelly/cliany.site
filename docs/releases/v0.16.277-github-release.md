# v0.16.277

Generated browser commands now preserve more of the workflows that `explore` discovers, while data commands must prove their own extraction result before an adapter is created.

## What Changed

- Added semantic `browser select` and `browser submit` commands so generated adapters can replay dropdown choices and Enter-based form submission.
- `submit` means Enter for a focused form. Explicit page buttons remain `click` actions.
- Before `explore` creates a `list-`, `search-`, `read-`, or `extract-` command, it now requires an owned `extract` action and real, quality-checked extraction evidence.
- Missing evidence, extraction failures, empty results where data is required, and partial rows trigger one bounded correction round. A remaining failure returns `E_EMPTY_RESULT` with machine-readable `data_commands` details instead of writing an action-only adapter.
- Python SDK callers receive the same diagnostic details in their error envelope.

## Compatibility

Existing installed adapters are not silently rewritten. Re-run `explore` to generate commands that use the new action routing and completion gate.

## Trust Boundaries

This release does not run a live LLM request, create a candidate adapter package, or claim a third-party online result. `expects_nonempty=false` accepts only a genuine zero-match extraction; it never accepts missing fields or partial data.

## Verification

Regression coverage exercises select and submit dispatch, extraction-error recording, one-round data-command repair, action ownership, CLI/SDK `E_EMPTY_RESULT` envelopes, code generation, and the full offline suite. GitHub CI and Embodied CI validated the source commit before release finalization. No live LLM or external data success is claimed.
