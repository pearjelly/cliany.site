# v0.16.2 发布草案

**目标版本：** `0.16.2`
**提交范围：** `v0.16.1..HEAD`
**当前阻塞：** 需要提交下个迭代计划脚本、文档入口和测试；最新本地 release 仍需维护者确认后 push 到远端。

## 用户价值

`v0.16.2` 聚焦维护者下个迭代计划。过去几轮已经有 `release_readiness.py`、`check_release_publication.py`、`check_release_cadence.py` 和案例库 report，但这些信号分散在不同命令里。这个 patch 新增一个计划入口，把“最新 release 是否公开可见”“本周提交日是否足够”“案例库 candidate 差什么”“下个草案路径是什么”合并成一个可复制的下一步清单。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.2 --json` 输出 `recommended_theme`、`recommended_slice`、`next_actions`、`validation_commands` 和 `release_draft_path`；`--report` 能生成 Markdown artifact，供周初选题和周末发版复盘使用。

## 变更分组

### Next iteration planner

- 新增 `scripts/plan_next_iteration.py`。
- 默认目标版本为当前 `pyproject.toml` 版本的下一个 patch，例如 `0.16.1` 之后建议 `0.16.2`。
- 聚合 `release_readiness.py` 和 `check_release_publication.py` 的当前证据，优先提示发布可见性、readiness blocker、commit days 和 candidate cases。
- 输出人读文本、`--json` 机器结果和 `--report` Markdown 报告。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 的周初命令加入 `plan_next_iteration.py`。
- `tests/test_plan_next_iteration.py` 固定计划排序、JSON 字段和当前仓库 CLI 输出。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环里的新入口。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例数量和状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。计划器会显示 active、candidate、known-gap 和 total 汇总，并列出 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 等 candidate cases，帮助维护者选择下一块可晋级案例。

`search-extraction-gap` 继续作为抽取质量 known-gap；计划器只引用现有案例库证据，不访问第三方站点，不执行真实 adapter 包构建。

## 风险与兼容性

- 这是维护脚本和文档 patch，不改变 CLI runtime、adapter schema、JSON envelope 或生成代码。
- `plan_next_iteration.py` 只读取本地 git、`pyproject.toml`、release readiness 和 publication audit 结果；不会 push、tag、访问远端或写入 `~/.cliany-site/`。
- 默认验证继续离线运行，不需要真实 LLM key。

## 发版前验证

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.2 --json
python scripts/plan_next_iteration.py --target-version 0.16.2 --report /tmp/cliany-next-iteration.md
python scripts/release_readiness.py --target-version 0.16.2 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.2 --json` 能给出当前仓库的下一步切片。
2. 运行计划脚本、维护者循环文档和 release draft 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.2`。
4. 将 `pyproject.toml` 版本更新为 `0.16.2`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.2`。
6. 创建 tag：`git tag v0.16.2`。

## Release Notes 摘要

`v0.16.2` adds a next-iteration planner for maintainers. It turns release readiness, publication status, weekly cadence, and candidate case evidence into one recommended release slice with JSON and Markdown outputs.
