# v0.16.125 发布草案

**目标版本：** `0.16.125`
**提交范围：** `v0.16.124..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 publication visibility summary 补进 candidate issue gate quick summary。

## 用户价值

`v0.16.124` 已经让 candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 展示 blocker 和 next action 数量。维护者可以先判断待办规模，但如果只看 `visibility` 状态码，仍需要展开 `Publication Handoff` 或完整 `Artifact Bundle Summary` 才能读到发布可见性的可读说明。

`v0.16.125` 在 `Candidate Issue Gate Quick Summary` 中新增 `visibility_summary`。维护者打开 README 时，可以直接看到例如“本地分支领先远端多少提交，需要先 push branch/tag”或“工作区有未提交改动”的完整说明，再决定下一步动作。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.125 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 出现在 `Artifact Bundle Summary` 之前，并展示 `visibility` 和 `visibility_summary`。

## 变更分组

### Candidate issue gate quick summary visibility summary

- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `visibility_summary`。
- 该字段来自已有 `publication_visibility.summary`。
- 使用现有 inline code formatting，兼容 summary 中包含反引号的情况。
- 现有 `publication_visibility`、`Publication Handoff` 和 `artifact_bundle_summary` 不改名、不删除。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 quick summary 会展示 visibility summary。
- `tests/test_plan_next_iteration.py` 覆盖 README quick summary 的 visibility summary 行。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新增 README 行只读取已有 `publication_visibility.summary`。
- 现有 `Artifact Bundle Summary` 字段不删除、不改名，机器读取路径保持兼容。
- `visibility_summary` 是人类可读辅助说明，不替代稳定的 `visibility` 状态码。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.125..HEAD`。最终带 `--release-tag v0.16.125` 的 strict gate 仍以本草案顶部的 `v0.16.124..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.125 --json
python scripts/plan_next_iteration.py --target-version 0.16.125 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.125 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.125 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 `visibility` 和 `visibility_summary`。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.125`。
7. 将 `pyproject.toml` 版本更新为 `0.16.125`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.125`。
9. 创建 tag：`git tag v0.16.125`。

## Release Notes 摘要

`v0.16.125` adds the publication visibility summary to the Candidate Issue Gate Quick Summary in generated candidate issue artifacts README files, helping maintainers read the human-facing release visibility explanation before expanding the full artifact bundle summary.
