# v0.16.146 发布草案

**目标版本：** `0.16.146`
**提交范围：** `v0.16.145..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把第一条 publication publish command 补进 artifact bundle summary。

## 用户价值

`v0.16.145` 已经让 `artifact_bundle_summary` 能直接展示第一条 publication next action。维护者和自动化只读整包摘要时，能看到发布同步第一步的文字说明，但仍需要展开 `publication-handoff.json` 或完整 command 列表，才能拿到第一条可执行发布命令。

`v0.16.146` 在 `artifact_bundle_summary` 中新增 `publication_primary_publish_command`，并在 artifacts `README.md` 的 `Artifact Bundle Summary` 段落展示同源字段。这样工具只读 manifest summary 或 README summary，就能直接显示第一条发布同步命令，同时仍可用 `publication_publish_command_count` 和 `publication_publish_commands_sha256` 判断是否展开完整命令列表。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.146 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 `publication_primary_publish_command`，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且该值与 `publication_publish_commands[0]` 同源；无 publication publish commands 时字段为 `null`。

## 变更分组

### Artifact bundle publication primary publish command

- `artifact_bundle_summary` 新增 `publication_primary_publish_command`。
- 有 publication publish commands 时，该字段取第一条 command。
- 无 publication publish command 时，该字段为 `null`；README summary 用 `(none)` 展示。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示第一条发布同步命令。
- 现有 `publication_publish_command_count`、`publication_publish_commands_sha256`、`publication_primary_next_action`、`publication_handoff_key_count`、`publication_handoff_sha256` 和 `Publication Handoff` 保持兼容。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示第一条 publication publish command。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的新字段。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 整包摘要的 publication primary publish command 可见性。

## 风险与兼容性

- 新字段只派生自既有 `publication_publish_commands`，不新增网络访问或外部状态读取。
- 无 publication publish command 时字段为 `null`，方便机器读取。
- Primary command 只用于快速展示第一条可执行命令，不替代完整 `publication-handoff.json` 审阅。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.146 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `publication_primary_publish_command`。
3. 确认 artifacts README 的 `Artifact Bundle Summary` 展示 `publication_primary_publish_command`。
4. 确认有 publication publish command 时，该字段等于 `publication_publish_commands[0]`。
5. 确认无 publication publish command 时，JSON 字段为 `null` 且 README 展示 `(none)`。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.146`。
8. 将 `pyproject.toml` 版本更新为 `0.16.146`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.146`。
10. 创建 tag：`git tag v0.16.146`。

## Release Notes 摘要

`v0.16.146` adds the primary publication publish command to artifact bundle summaries, helping tools show the first executable release publishing command without expanding the full publication handoff.
