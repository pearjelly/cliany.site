# v0.16.161 发布草案

**目标版本：** `0.16.161`
**提交范围：** `v0.16.160..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 artifact manifest 顶层字段边界显式化到整包摘要。

## 用户价值

`v0.16.160` 已经让 `artifact_bundle_summary` 输出 summary 自身字段清单的 first/last key 和 boundary hash，维护工具可以快速确认 summary 边界。但 `artifact-manifest.json` 顶层字段虽然已有 key count/hash，仍缺少首尾边界字段；工具若只想确认 manifest 顶层字段入口和出口是否保持预期，仍要展开完整 key list。

`v0.16.161` 在 `artifact_bundle_summary` 中新增 `artifact_manifest_first_key`、`artifact_manifest_last_key` 和 `artifact_manifest_key_boundary_sha256`，并在 artifacts `README.md` 的 `Artifact Bundle Summary` 段落展示同源字段。这样维护工具可以只读整包摘要，先确认 manifest 顶层字段清单边界，再决定是否展开完整 manifest。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.161 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 manifest key boundary fields，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且 first/last key 等于 `artifact-manifest.json` 实际顶层字段清单首尾。

## 变更分组

### Artifact manifest key boundary

- `artifact_bundle_summary` 新增 `artifact_manifest_first_key`。
- `artifact_bundle_summary` 新增 `artifact_manifest_last_key`。
- `artifact_bundle_summary` 新增 `artifact_manifest_key_boundary_sha256`。
- Boundary 从 `ARTIFACT_MANIFEST_KEYS` 的首尾字段派生。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示 manifest first/last key 和 boundary hash。
- 现有 artifact manifest schema/key/payload 摘要、summary key boundary、handoff 摘要和 gate 摘要保持兼容。
- 不改变 release draft validation、publication audit 或 candidate issue gate 的判断逻辑。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示 manifest key boundary fields。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的 manifest boundary 字段，并断言 first/last key 与 manifest 顶层字段清单首尾同源。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的整包摘要可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有 artifact manifest 顶层字段清单，不新增网络访问或外部状态读取。
- 旧工具忽略未知 summary 字段即可继续读取现有 manifest。
- 新工具可以只读 `artifact_manifest_first_key` 和 `artifact_manifest_last_key` 快速确认 manifest 顶层字段清单边界。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.161 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `artifact_manifest_first_key`、`artifact_manifest_last_key` 和 `artifact_manifest_key_boundary_sha256`。
3. 确认 artifacts README 的 `Artifact Bundle Summary` 展示这三个字段。
4. 确认 first/last key 等于 `artifact-manifest.json` 实际顶层字段清单首尾。
5. 确认 `artifact_manifest_key_count` 和 `artifact_manifest_keys_sha256` 仍与完整 manifest key 清单同源。
6. 运行计划器、维护者循环文档和 release draft 相关测试。
7. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.161`。
8. 将 `pyproject.toml` 版本更新为 `0.16.161`。
9. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.161`。
10. 创建 tag：`git tag v0.16.161`。

## Release Notes 摘要

`v0.16.161` adds artifact manifest first/last key and boundary hash fields to the artifact bundle summary, helping tools confirm manifest key-list boundaries before expanding the full manifest.
