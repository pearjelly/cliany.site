# v0.16.267 发布草案

**目标版本：** `0.16.267`
**提交范围：** `v0.16.266..HEAD`
**当前状态：** 2026-07-20 release slice；Python SDK 版本与已安装发行包统一。

## 用户价值

`from cliany_site import __version__` 现在与 `cliany-site --version` 和 PyPI 发行版本一致。SDK 集成可据此可靠地记录诊断、做最低版本判断和生成支持信息，不再错误报告历史 `0.14.0`。

## 案例库映射

本版不改变案例状态或 adapter 资产。`cases/README.md` 和 `cases/manifest.json` 仍是案例目录的唯一来源；`search-extraction-gap` 与 `llm_live_preflight_not_ready` 继续约束候选案例晋级，未产生虚构的 online smoke 或 package 证据。

## 风险与兼容性

- `__version__` 保持字符串 API；在未安装发行元数据的源码环境中返回 `0+unknown`，避免导入失败。
- 已安装发行包通过 `importlib.metadata` 获得权威版本，消除将来升级时遗漏手工常量的风险。

## 发版步骤

1. 在 `[Unreleased]` 保留本版条目、compare link 为 `v0.16.266...HEAD` 时运行 target-mode readiness。
2. Finalize Changelog：清空 `[Unreleased]`，新增 `## [0.16.267] - 2026-07-20`，并更新 compare links。
3. 提交并推送 `master`，等待 CI 成功。
4. 创建 `git tag v0.16.267`，然后运行 `uv run --extra dev --frozen python scripts/release_readiness.py --strict --release-tag v0.16.267 --remote --remote-name origin`。
5. 仅在该 gate 通过后推送 tag，等待 GitHub Release 与 PyPI 发布。
6. 在 `site/` 运行 `vercel link --yes --project cliany.site`、`vercel --prod --yes` 和 `vercel inspect www.cliany.site --wait --timeout 90s`。
7. 运行 `uv run --extra dev --frozen python scripts/check_release_publication.py --strict --remote --distribution --json`。
