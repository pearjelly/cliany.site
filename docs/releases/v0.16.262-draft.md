# v0.16.262 发布草案

**目标版本：** `0.16.262`
**提交范围：** `v0.16.261..HEAD`
**当前状态：** 2026-07-14 release execution；`v0.16.261` 已完整发布，今天尚无 release tag，本版是今日首个 release。

## 用户价值

`v0.16.262` 把上一版修正的 nullable doctor evidence 语义变成自动化可直接读取的契约。Standalone extractor 现在输出 `null_count` 与 `null_fields`；cases evidence bundle、planner issue metadata 和 compact handoff 输出 `doctor_preflight_evidence_null_count` 与 `doctor_preflight_evidence_null_fields`。

机器不再需要遍历全部 evidence values 来猜测某个 `null` 是真实连接级结果还是 selector 缺失。`missing_fields` 只表示 doctor JSON 中没有匹配 selector；`null_fields` 只表示 selector 存在且值为 JSON `null`。例如 `checks[llm_live].details.status_code: null` 会得到 `null_count=1`、`missing_count=0`，preflight state 仍按 readiness 信号判为 `blocked`。

Human 和 Markdown 输出也会展示 null count，standalone Markdown 额外列出精确的 nullable selectors，便于维护者把无需二次解释的 blocker evidence 贴回 candidate issue。

当前 live LLM preflight 仍未 ready，因此本版不运行真实 `explore`，也不伪造 `pypi-project-search` adapter package、metadata validation 或 online smoke 证据。维护者可以先保存准确的 nullable blocker evidence，provider 恢复后再继续晋级。

## 变更范围

- `scripts/extract_doctor_preflight_evidence.py`：增加 `null_count` / `null_fields`，并在 Markdown 中展示计数与字段清单。
- `src/cliany_site/commands/cases.py`：evidence bundle、primary task 与 human output 暴露 namespaced null metadata。
- `scripts/plan_next_iteration.py`：candidate issue metadata、issue body 与 compact handoff 暴露同一 metadata。
- README、candidate promotion runbook、案例目录和官网基线同步到 v0.16.262 契约。
- 回归覆盖显式 JSON `null`、真实缺失 selector、cases bundle、compact handoff、issue artifacts 和 Markdown 输出。

## 风险与兼容性

- 仅新增输出字段；既有字段、selector 列表、hash、`doctor_preflight_state` 与 exit code 语义不变。
- `missing_fields` 与 `null_fields` 互斥统计；缺失值即使在 `values` 中序列化为 JSON `null`，也不会进入 `null_fields`。
- 不写入 repo runtime 状态；adapter/session/snapshot 仍位于 `~/.cliany-site/`。
- 不修改 AXTree selector 或 action replay，不引入 CSS selector fallback。
- `pypi-project-search` 保持 candidate，直到 adapter package、metadata validation 和 online smoke 三项证据齐全。

## Release Notes 摘要

v0.16.262 adds explicit nullable-field metadata to doctor-backed candidate promotion evidence. Automation can now read `null_count` / `null_fields` or their namespaced equivalents to distinguish present JSON `null` values from absent selectors across the standalone extractor, cases CLI, planner issue artifacts, and compact handoffs. Existing preflight states and missing-field behavior are unchanged.
