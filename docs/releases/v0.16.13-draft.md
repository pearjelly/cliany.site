# v0.16.13 发布草案

**目标版本：** `0.16.13`
**提交范围：** `v0.16.12..HEAD`
**当前阻塞：** 需要提交 candidate handoff matrix、案例库文档和测试；最新本地 release 仍需维护者确认后 push 到远端。

## 用户价值

真实案例库已经有三个 package registry candidate：`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search`。它们都有 target URL、推荐命令、离线样例和 promotion 清单，但贡献者查看 `scripts/validate_cases.py --report` 时，仍需要回到 `cases/manifest.json` 才能拿到基本复现上下文。

`v0.16.13` 在案例库 Markdown report 中新增 `Candidate Handoff Matrix`。报告会把每个 candidate 的 target URL、recommended commands 和 offline validation commands 放进同一张表，方便维护者把 candidate 晋级拆成 issue，也方便首次贡献者不翻 JSON 就知道从哪里开始。

成功标准是：`python scripts/validate_cases.py --report /tmp/cliany-case-catalog-report.md` 生成 `Candidate Handoff Matrix`，并列出 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 的 target URL、命令和离线验证命令。

## 变更分组

### Candidate handoff matrix

- `scripts/validate_cases.py` 的 `CaseCheck` 保留 `target_url` 和 `commands`。
- JSON 输出包含每个 case 的 `target_url` 和 `commands`，便于后续自动化读取。
- Markdown report 新增 `Candidate Handoff Matrix`。
- 该矩阵只读取 `cases/manifest.json`，不访问第三方站点，也不运行 candidate 命令。

### 文档与门禁

- [cases/README.md](../../cases/README.md) 说明 `Candidate Handoff Matrix` 的用途。
- `tests/test_validate_cases.py` 固定 JSON 字段和 Markdown report 输出。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例数量和状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本让 candidate 更容易被贡献者接手，但不会提前把 candidate 标记为 active；adapter package、metadata validation 和 online smoke 仍必须按 promotion 清单逐项完成。

## 风险与兼容性

- `target_url` 和 `commands` 是 `CaseCheck.to_dict()` 的新增字段；已有字段保持兼容。
- Markdown report 只新增小节，不改变现有 `Offline Validation Commands` 或 `Candidate Promotion Tasks`。
- 默认验证继续离线运行，不需要真实 LLM key，也不会写入 `~/.cliany-site/`。


## 发版步骤

1. 确认 `python scripts/validate_cases.py --report /tmp/cliany-case-catalog-report.md` 输出 `Candidate Handoff Matrix`。
2. 确认矩阵包含 candidate 的 target URL、recommended commands 和 offline validation commands。
3. 运行案例报告和 release draft 相关测试。
4. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.13`。
5. 将 `pyproject.toml` 版本更新为 `0.16.13`。
6. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.13`。
7. 创建 tag：`git tag v0.16.13`。

## Release Notes 摘要

`v0.16.13` adds a Candidate Handoff Matrix to the case catalog report. Contributors can now see each candidate's target URL, recommended commands, and offline validation commands in one report before picking up promotion work.
