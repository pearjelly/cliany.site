# v0.16.30 发布草案

**目标版本：** `0.16.30`
**提交范围：** `v0.16.29..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 publish script 在执行前拒绝过期脚本，避免误推和脚本上下文不一致。

## 用户价值

`v0.16.29` 已经让 `scripts/check_release_publication.py --publish-script` 生成的脚本带上 `Publication context`。但脚本生成后，维护者可能继续提交、移动 tag 或切换分支；如果之后误运行旧脚本，脚本里的注释虽然能提示上下文，仍然需要人工肉眼发现它已经过期。

`v0.16.30` 给生成的 publish script 增加本地 stale preflight。脚本会记录 `EXPECTED_LOCAL_HEAD`、`EXPECTED_LATEST_TAG` 和 `EXPECTED_TAG_COMMIT`，执行时先重新读取当前 HEAD、latest tag 和 tag commit；如果任一项不一致，会打印 `Publish script is stale` 并退出，不执行 `git push origin master`、`git push origin v0.16.30` 或远端复核命令。

成功标准是：`python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh` 写出的脚本包含 `EXPECTED_LOCAL_HEAD`、`EXPECTED_LATEST_TAG`、`EXPECTED_TAG_COMMIT` 和 stale preflight；如果脚本生成后 HEAD 发生变化，直接运行脚本会失败，并且不会推进远端分支。

## 变更分组

### Publication publish script stale preflight

- `scripts/check_release_publication.py --publish-script` 写出的脚本会记录生成时的 HEAD、latest tag 和 tag commit。
- 脚本执行 push 前会重新读取本地 git 状态，检测旧脚本。
- 检测到 stale 状态时输出 `Publish script is stale` 并退出，后续 publish commands 不会执行。

### 文档与门禁

- [docs/release-cadence.md](../release-cadence.md) 说明 publish script 会执行本地 stale preflight。
- `tests/test_release_publication.py` 覆盖脚本字段和 HEAD 变化后的拒绝执行行为。
- `tests/test_release_cadence_docs.py` 固定发布节奏文档说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强 release publication handoff 脚本的执行前保护。

## 风险与兼容性

- 这是 generated shell script 的本地 preflight 增强，不改变 `publish_commands` 的 JSON schema 或命令顺序。
- 如果维护者在生成脚本后继续提交或移动 tag，旧脚本会失败；这是预期保护。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- `--remote` 行为保持不变；stale preflight 只读取本地 git 状态。

## 发版前验证

```bash
pytest tests/test_release_publication.py tests/test_release_cadence_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/check_release_publication.py tests/test_release_publication.py tests/test_release_cadence_docs.py tests/test_release_draft_docs.py
python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh
python scripts/release_readiness.py --target-version 0.16.30 --json
```

## 发版步骤

1. 确认 `python scripts/check_release_publication.py --json --publish-script /tmp/cliany-publish-release.sh` 写出的脚本包含 `EXPECTED_LOCAL_HEAD`、`EXPECTED_LATEST_TAG` 和 `EXPECTED_TAG_COMMIT`。
2. 确认脚本包含 `Publish script is stale` 失败提示。
3. 确认 HEAD 变化后的脚本执行测试会在 push 前退出。
4. 运行 publication、release cadence 文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.30`。
6. 将 `pyproject.toml` 版本更新为 `0.16.30`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.30`。
8. 创建 tag：`git tag v0.16.30`。

## Release Notes 摘要

`v0.16.30` adds a local stale preflight to generated publish scripts, refusing to run branch/tag pushes when HEAD or tag context no longer matches the script generation state.
