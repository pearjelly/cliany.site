# v0.16.245 发布草案

**目标版本：** `0.16.245`
**提交范围：** `v0.16.244..HEAD`
**当前阻塞：** 本版需要完成 release readiness、完整离线测试、CHANGELOG / 版本号同步，并在最终 readiness 通过后创建并推送 `v0.16.245` tag。

## 用户价值

`v0.16.244` 发布后，`master` 上的主 CI 暴露出一组 post-release 稳定性问题：`explore --json` 的离线 QA 错误顺序会被 Chrome preflight 抢先拦截，Obscura smoke 会通过 `uv run` 创建没有 pytest coverage plugin 的临时环境，Windows pytest job 又同时暴露出 UTF-8、`Path.home()` 隔离、POSIX mode bit、CRLF artifact 和 PowerShell native-command fail-fast 等平台差异。

`v0.16.245` 把这些发布后修复收敛成一个 patch：离线 QA gate 在没有 fake LLM 配置时先返回确定性错误；CI smoke 复用已安装测试环境；Windows 测试只承担跨平台 smoke 职责，设置 UTF-8 / Windows home 隔离，跳过 POSIX-only 断言和 stdout-heavy planner 路径，并最终改用 `cmd` 包住 smoke suite，显式读取 pytest exit code 与 JUnit XML，避免 PowerShell 在 pytest 已通过后把 step 标成失败。

## 变更分组

- Offline QA preflight：`explore --json` 在 `CLIANY_QA_OFFLINE=1` 且 fake LLM 未配置时先报告缺失配置，避免测试结果依赖本机 Chrome 是否可用。
- Obscura CI smoke：mock-based smoke 不再用 `uv run` 创建新环境，直接复用 workflow 已安装的 test extras。
- Windows isolation：`tmp_home` fixture 同步设置 `USERPROFILE`、`HOMEDRIVE`、`HOMEPATH`，避免 Windows `Path.home()` 泄漏到 runner 用户目录。
- Cross-platform artifacts：candidate issue artifact 输出固定 LF，Windows 不再校验 POSIX executable bit。
- Windows test scope：Windows job 明确跑跨平台 smoke 文件列表，完整 pytest matrix 仍由 Linux Python 3.11/3.12/3.13 覆盖。
- Windows shell control：smoke suite 通过 `cmd` 调用 pytest，并用 JUnit XML 作为零失败/零错误的权威结果，真实测试失败仍会让 job 失败。

## 案例库映射

- 本次不修改 [cases/manifest.json](../../cases/manifest.json) 的案例状态；相关说明仍落在 [cases/README.md](../../cases/README.md)。
- 当前案例库仍保持 active 4 个、candidate 3 个、known gap 1 个。
- Known gap 保持 1 个：`search-extraction-gap`。
- Candidate 晋级仍需要 adapter package、metadata validation 和 online smoke 三段真实证据。
- 本版不生成 candidate adapter package，也不把任何 candidate 标记为 active。

## 风险与兼容性

- 这是 CI、测试隔离和 release gate 稳定性修复，不改变 CLI 参数、adapter metadata schema、action runtime 语义、AXTree 语义匹配、session 存储或 marketplace package 格式。
- Windows job 从完整 pytest 缩小为明确 smoke suite；完整测试覆盖仍由 Linux Python matrix 承担，Windows 保留跨平台路径、home 隔离、manifest、doctor、codegen、Obscura 基础能力等代表性覆盖。
- `cmd` wrapper 会在 JUnit XML 存在且 failures/errors 均为 0 时接受 smoke 结果；如果 pytest 产生真实失败或没有生成报告，job 仍返回 pytest 的非零退出码。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.244..HEAD`。最终带 `--release-tag v0.16.245` 的 strict gate 仍以本草案顶部的 `v0.16.244..HEAD` 作为实际 release diff 范围。

```bash
ruff check src/
mypy src/cliany_site/
python scripts/validate_cases.py --strict
pytest tests/ -q
git diff --check
```

正式 tag 前再运行：

```bash
pytest tests/ -q
python scripts/release_readiness.py --strict --release-tag v0.16.245
```

## Release note draft

v0.16.245 stabilizes the post-v0.16.244 release pipeline by fixing deterministic offline QA preflight behavior, reusing the installed CI test environment for Obscura smoke checks, hardening Windows home isolation and artifact comparisons, and running the Windows smoke suite through a `cmd` wrapper that honors zero-failure JUnit results.
