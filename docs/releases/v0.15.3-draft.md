# v0.15.3 发布草案

**目标版本：** `0.15.3`
**提交范围：** `v0.15.2..HEAD`
**当前阻塞：** 需要提交 publication report 输出、文档入口和 release readiness 元数据 gate。

## 用户价值

`v0.15.3` 聚焦发布可见性证据留档。`v0.15.2` 已能告诉维护者本地 release commit/tag 是否公开可见，但如果只能看终端或 JSON，一次发布复盘里仍然缺少可直接保存、上传或粘贴的报告。这个 patch 给 `scripts/check_release_publication.py` 增加 Markdown report，让 branch/tag 发布状态像 release readiness 一样能沉淀成 artifact。

成功标准是：运行 `python scripts/check_release_publication.py --report /tmp/cliany-release-publication.md` 后，报告应包含 Summary、Refs 和 Next Actions；传入 `--remote` 时报告还应记录实时远端 branch/tag refs。维护者可以把这份报告和 `release-readiness-report.md` 一起保存，用来解释“本地已 tag 但尚未公开”或“远端 tag 已可见”的状态。

## 变更分组

### Publication report

- `scripts/check_release_publication.py` 新增 `--report` 参数，写出 Markdown 报告。
- 报告包含 `ok`、branch、upstream、ahead/behind、latest tag、branch/tag published、remote checked 等 Summary 字段。
- 报告包含本地 HEAD、upstream HEAD、latest tag commit、remote branch HEAD 和 remote tag commit 等 Refs 字段。
- 报告包含 `Next Actions`，保持和 JSON/文本输出中的 `next_actions` 一致。

### 文档与门禁

- [docs/release-cadence.md](../release-cadence.md) 增加 `check_release_publication.py --remote --report` 示例和使用说明。
- [docs/contributor-starter.md](../contributor-starter.md) 把 publication report 纳入发布脚本贡献验证路径。
- `release_readiness.py` 的项目元数据 gate 应检查 release cadence 文档保留 publication report 入口。

## 案例库映射

本版本不改变案例库内容，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。`suitecrm-accounts`、`apache-jira-issues`、`apache-confluence-search` 和 `apache-jenkins-jobs` 仍是 active；`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是维护者脚本和文档层面的 patch，不改变 CLI runtime、adapter schema 或 JSON envelope。
- `--report` 只写调用者指定的 Markdown 文件，不创建 release、不推送 tag、不访问 PyPI。
- 默认模式仍不访问网络；只有显式传入 `--remote` 时才读取实时远端 refs。

## 发版前验证

```bash
python scripts/check_release_publication.py --json --report /tmp/cliany-release-publication.md
pytest tests/test_release_publication.py tests/test_release_readiness.py tests/test_contributor_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/check_release_publication.py scripts/release_readiness.py tests/test_release_publication.py tests/test_release_readiness.py tests/test_contributor_docs.py tests/test_release_draft_docs.py
python scripts/release_readiness.py --target-version 0.15.3 --json
```

如需要发布前实时确认远端 refs：

```bash
python scripts/check_release_publication.py --remote --report /tmp/cliany-release-publication.md
```

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.15.3 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行 `python scripts/check_release_publication.py --json --report /tmp/cliany-release-publication.md`。
3. 运行相关 pytest 与 ruff 检查。
4. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.15.3`。
5. 将 `pyproject.toml` 版本更新为 `0.15.3`。
6. 运行 `python scripts/release_readiness.py --strict --target-version 0.15.3`。
7. 创建 tag：`git tag v0.15.3`。

## Release Notes 摘要

`v0.15.3` adds Markdown report output to the release publication audit. Maintainers can now save branch/tag publication status, refs, and next actions as a review artifact before or after pushing a release.
