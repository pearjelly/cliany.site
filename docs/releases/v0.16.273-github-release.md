# v0.16.273

`doctor` now gives new users a real, pinned active-demo path instead of sending them to an unavailable historical archive.

## What Changed

- `demo_adapter_quickstart` reads the packaged active case catalog and returns ordered install, verify, and read-only commands when a real no-login demo is available.
- The current path selects Apache Jira from the maintained active cases, using its pinned GitHub Release HTTPS asset and SHA-256 digest.
- Candidate cases and login-required demos stay out of this first-success path.
- Metadata schema v3 continues to validate supported API-capability adapters, including legacy string endpoint and command declarations.

## Trust Boundaries

The install and verify path proves the published archive can be downloaded and validated. No third-party business result is claimed until the user runs the final read-only command in their own browser and permission context. Live LLM preflight remains independent; `E_LLM_UNAVAILABLE` still blocks candidate generation and promotion.
