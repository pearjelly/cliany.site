# v0.16.44 发布草案

**目标版本：** `0.16.44`
**提交范围：** `v0.16.43..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 candidate issue artifacts manifest 直接展示 publication ref、worktree 和 publish-script 上下文。

## 用户价值

`v0.16.43` 已经让 `artifact-manifest.json` 展示 publication ok、visibility、publication next actions 和 publish commands。维护者或工具可以先读 manifest 判断是否应该暂停候选 issue 创建，但如果要看到 latest tag、local HEAD、ahead count、worktree 状态和生成发布脚本的命令，仍然需要继续读取 `publication-handoff.json`。

`v0.16.44` 让 `artifact-manifest.json` 直接包含 `publication_ref_context`、`publication_worktree_clean`、`publication_worktree_status` 和 `publication_publish_script_command`。这样候选 issue artifacts 的机器可读入口可以完整表达“当前 release 是否可见、为什么不可见、应该审阅哪些 refs、是否有本地改动、如何生成发布脚本”。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.44 --issues-dir /tmp/cliany-candidate-issues --json` 写出的 `artifact-manifest.json` 包含 publication context 字段；artifacts `README.md` 的 Files 小节说明 manifest 会包含 publication ref context、worktree status 和 publish commands。

- `artifact-manifest.json` 新增 `publication_ref_context`。
- `artifact-manifest.json` 新增 `publication_worktree_clean`。
- `artifact-manifest.json` 新增 `publication_worktree_status`。
- `artifact-manifest.json` 新增 `publication_publish_script_command`。
- Artifacts `README.md` 的 Files 小节说明 manifest 包含 publication ref context 和 worktree status。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 manifest 会复用 `publication-handoff.json` 的 publication ref context、publication worktree status 和 publication publish script command。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 内容和 README 文件列表。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只增强候选 issue artifacts manifest 的 publication 上下文摘要。

## 风险与兼容性

- `publication_ref_context`、`publication_worktree_clean`、`publication_worktree_status` 和 `publication_publish_script_command` 是 `artifact-manifest.json` 的新增字段，不改变 plan JSON schema。
- 既有 `publication_ok`、`publication_visibility`、`publication_next_actions`、`publication_publish_commands`、`files`、`review_order` 和 `validation_commands` 字段保持兼容。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- Publication context 字段只复用当前 plan 的既有字段和 publication audit 结果。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.44..HEAD`。最终带 `--release-tag v0.16.44` 的 strict gate 仍以本草案顶部的 `v0.16.43..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov

python scripts/plan_next_iteration.py --target-version 0.16.44 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.44 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.44 --issues-dir /tmp/cliany-candidate-issues --json` 写出 `artifact-manifest.json`。
2. 确认 manifest 包含 `publication_ref_context`、`publication_worktree_clean`、`publication_worktree_status` 和 `publication_publish_script_command`。
3. 确认 artifacts `README.md` 的 Files 小节说明 manifest 包含 publication ref context 和 worktree status。
4. 运行计划器、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.44`。
6. 将 `pyproject.toml` 版本更新为 `0.16.44`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.44`。
8. 创建 tag：`git tag v0.16.44`。

## Release Notes 摘要

`v0.16.44` adds publication ref, worktree, and publish-script context to generated `artifact-manifest.json` files, making candidate issue artifact bundles easier to inspect from one machine-readable entrypoint.
