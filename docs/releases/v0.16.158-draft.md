# v0.16.158 发布草案

**目标版本：** `0.16.158`
**提交范围：** `v0.16.157..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 artifact bundle summary 自身字段预览显式化。

## 用户价值

`v0.16.157` 已经让 `artifact_bundle_summary` 输出 summary 自身 key count/hash，维护工具可以判断字段清单是否漂移。但只看 hash 时，人工审阅者仍需要展开完整 summary key list 才能快速确认前几个关键字段是否保持预期顺序。

`v0.16.158` 在 `artifact_bundle_summary` 中新增 `artifact_bundle_summary_key_preview_count`、`artifact_bundle_summary_key_preview` 和 `artifact_bundle_summary_key_preview_sha256`，并在 artifacts `README.md` 的 `Artifact Bundle Summary` 段落展示同源字段。这样维护工具和人工审阅者可以先看 summary 顶部字段预览，再决定是否展开完整字段清单或 manifest。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.158 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 summary key preview count/list/hash，artifacts README 的 `Artifact Bundle Summary` 展示同一字段，且 preview 等于 `artifact_bundle_summary` 实际字段清单的前几个 key。

## 变更分组

### Artifact bundle summary key preview

- `artifact_bundle_summary` 新增 `artifact_bundle_summary_key_preview_count`。
- `artifact_bundle_summary` 新增 `artifact_bundle_summary_key_preview`。
- `artifact_bundle_summary` 新增 `artifact_bundle_summary_key_preview_sha256`。
- Preview 从 `ARTIFACT_BUNDLE_SUMMARY_KEYS` 的前几个字段派生。
- Artifacts `README.md` 的 `Artifact Bundle Summary` 展示 preview count/list/hash。
- 现有 summary key count/hash、manifest schema/key/payload 摘要、handoff 摘要和 gate 摘要保持兼容。
- 不改变 release draft validation、publication audit 或 candidate issue gate 的判断逻辑。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 `artifact_bundle_summary` 会展示 summary key preview count/list/hash。
- `tests/test_plan_next_iteration.py` 覆盖 manifest summary 与 README summary 的 preview 字段，并断言 preview 与 summary key 清单前缀同源。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善 candidate issue artifacts 的整包摘要可读性和机器可读入口。

## 风险与兼容性

- 新字段只派生自既有 artifact bundle summary 字段清单，不新增网络访问或外部状态读取。
- 旧工具忽略未知 summary 字段即可继续读取现有 manifest。
- 新工具可以只读 `artifact_bundle_summary_key_preview` 快速确认 summary 入口字段顺序。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
