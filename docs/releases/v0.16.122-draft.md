# v0.16.122 发布草案

**目标版本：** `0.16.122`
**提交范围：** `v0.16.121..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 candidate issue gate quick summary 的发布状态字段真正放进生成的 README 第一屏。

## 用户价值

`v0.16.121` 已经把维护者循环文档中的 quick summary 字段族说明补完整。维护者知道第一屏应该覆盖 upstream HEAD、tag state、published state 和 remote refs，但生成的 `Candidate Issue Gate Quick Summary` 仍只展示 local HEAD 和 tag commit，仍需要继续展开 `Artifact Bundle Summary` 才能看到这些关键发布状态。

`v0.16.122` 在 candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 中新增 `publication_upstream_head`、`publication_tag_points_at_head`、`publication_tag_commit_in_upstream`、`publication_branch_published`、`publication_tag_published`、`publication_remote_branch_head` 和 `publication_remote_tag_commit`。维护者打开 README 时，可以直接判断本地 HEAD、上游 HEAD、tag 是否指向 HEAD、tag commit 是否进入上游，以及远端分支/tag 在 remote check 时是否可见。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.122 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 出现在 `Artifact Bundle Summary` 之前，并展示 upstream HEAD、tag state、published state 和 remote refs。

## 变更分组

### Candidate issue gate quick summary publication state

- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_upstream_head`。
- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_tag_points_at_head` 和 `publication_tag_commit_in_upstream`。
- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_branch_published` 和 `publication_tag_published`。
- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_remote_branch_head` 和 `publication_remote_tag_commit`。
- 这些字段均来自已有 `publication_ref_context`，不新增 schema，不改变 `artifact_bundle_summary`。

### 文档与门禁

- `scripts/plan_next_iteration.py` 生成 quick summary 的发布状态字段。
- `tests/test_plan_next_iteration.py` 覆盖 README quick summary 的发布状态字段。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新增 README 行只读取已有 `publication_ref_context` 字段。
- 现有 `Artifact Bundle Summary` 字段不删除、不改名，机器读取路径保持兼容。
- 新字段可能显示 `(none)`，用于明确 remote check 未提供对应远端引用。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.122..HEAD`。最终带 `--release-tag v0.16.122` 的 strict gate 仍以本草案顶部的 `v0.16.121..HEAD` 作为实际 release diff 范围。

```bash
python scripts/plan_next_iteration.py --target-version 0.16.122 --json
python scripts/plan_next_iteration.py --target-version 0.16.122 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.122 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.122 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 `publication_upstream_head`、`publication_tag_points_at_head`、`publication_tag_commit_in_upstream`、`publication_branch_published`、`publication_tag_published`、`publication_remote_branch_head` 和 `publication_remote_tag_commit`。
5. 运行计划器和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.122`。
7. 将 `pyproject.toml` 版本更新为 `0.16.122`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.122`。
9. 创建 tag：`git tag v0.16.122`。

## Release Notes 摘要

`v0.16.122` adds upstream HEAD, tag state, published state, and remote-ref fields to the Candidate Issue Gate Quick Summary in generated candidate issue artifacts README files, helping maintainers judge release visibility before expanding the full artifact bundle summary.
