# v0.16.261 发布草案

**目标版本：** `0.16.261`
**提交范围：** `v0.16.260..HEAD`
**当前状态：** 2026-07-10 release execution；`v0.16.260` 已发布，今天尚无 release tag，daily release cap 未触发。

## 用户价值

`v0.16.261` 修复 candidate promotion 的真实 blocker evidence 语义。`cliany-site doctor --llm-live --json` 在连接级失败时会返回 `E_LLM_UNAVAILABLE` 和 `status_code: null`；这里的 `null` 是字段存在且没有 HTTP 响应码，不是字段缺失。

此前 standalone extractor、`cliany-site cases --doctor-json` 和 `scripts/plan_next_iteration.py --doctor-json` 都用值是否为 `None` 判断 selector 是否缺失，因此把 present JSON `null` 错判为 `missing_fields`。本版改为分别追踪 selector presence 与 value：真实缺键仍是 `missing_fields`，显式 `null` 保留在 evidence values 中，并根据 doctor readiness 信号得到 `doctor_preflight_state.status=blocked`。

2026-07-10 的真实 preflight 证据为：CDP `ok`、`summary.ready_for_explore=false`、`summary.llm_live_preflight.ready=false`、`summary.capabilities.generate_adapters.ready=false`、`checks[llm_live].status=warning`、`checks[llm_live].details.error_code=E_LLM_UNAVAILABLE`、`checks[llm_live].details.status_code=null`。修复后 extractor 输出 `missing_count=0`、`doctor_preflight_state.status=blocked`，reason codes 包含 `llm_live_preflight_not_ready` 与 `llm_live_status_warning`。

由于 live preflight 未 ready，本版不运行真实 `explore`，也不伪造 `pypi-project-search` adapter package、metadata validation 或 online smoke 证据。维护者可以把准确的 blocked evidence 贴回 candidate issue，并在 provider 恢复后继续晋级。

## 变更范围

- `scripts/extract_doctor_preflight_evidence.py`：区分 selector 缺失与 present JSON `null`。
- `scripts/plan_next_iteration.py`：doctor-json handoff 使用相同 presence 语义。
- `src/cliany_site/commands/cases.py`：evidence bundle 和 issue template 使用相同 presence 语义。
- 回归覆盖 standalone extractor、compact planner handoff 与 cases evidence bundle，同时保留 named check 缺失时的 `missing_fields` 行为。
- 官网 baseline 与 package version 在同一 release commit 更新到 `v0.16.261`。

## 风险与兼容性

- JSON 输出字段结构不变；仅修正 `missing_fields` 与 `blocked` 的分类。
- 显式 `null` 继续序列化为 JSON `null`，不会引入 sentinel 或 Python 对象到外部 payload。
- 不写入 repo runtime 状态；doctor evidence 保存在 `/tmp`，adapter/session/snapshot 仍位于 `~/.cliany-site/`。
- 不修改 AXTree selector 或 action replay，不引入 CSS selector fallback。
- `pypi-project-search` 保持 candidate，直到 adapter package、metadata validation 和 online smoke 三项证据齐全。

## Release Notes 摘要

v0.16.261 fixes doctor-backed candidate promotion evidence for connection-level LLM failures. A present `checks[llm_live].details.status_code: null` now remains valid evidence and produces a blocked preflight state instead of a false missing-fields result across the cases CLI, next-iteration planner, and standalone extractor. Real missing selectors remain blocked as `missing_fields`.
