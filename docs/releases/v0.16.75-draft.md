# v0.16.75 发布草案

**目标版本：** `0.16.75`
**提交范围：** `v0.16.74..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让 `artifact_bundle_summary` 直接携带 review checklist 的数量与稳定 hash。

## 用户价值

`v0.16.74` 已经让 `artifact_bundle_summary` 能检测 validation commands 是否漂移。但候选 issue artifacts 的人工审阅清单也会影响维护者能否安全创建任务；如果 review checklist 变化，自动化需要尽早知道，而不是等到展开完整 manifest 后才发现。

`v0.16.75` 在 `artifact_bundle_summary` 中新增 `review_checklist_count` 和 `review_checklist_sha256`。Artifacts `README.md` 的 `Artifact Bundle Summary` 也展示这些字段，让维护者和自动化能在展开完整 manifest 前判断 review checklist 是否漂移。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.75 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 review checklist count/hash，并能从 manifest 中的 `review_checklist` 重新计算得到；README 的 `Artifact Bundle Summary` 也展示这些字段。

## 变更分组

### Artifact bundle review checklist drift summary

- `artifact_bundle_summary` 新增 `review_checklist_count`。
- `artifact_bundle_summary` 新增 `review_checklist_sha256`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示这些 review checklist 漂移摘要字段。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 review checklist count/hash。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让 candidate issue artifacts 的整包摘要更适合自动化判断审阅清单是否变化。

## 风险与兼容性

- 新字段只出现在 `artifact_bundle_summary`，不改变 `review_checklist` 的既有字段语义。
- Hash 基于现有 review checklist 稳定 JSON 序列化计算，不新增网络访问。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.75..HEAD`。最终带 `--release-tag v0.16.75` 的 strict gate 仍以本草案顶部的 `v0.16.74..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov

python scripts/plan_next_iteration.py --target-version 0.16.75 --json
python scripts/plan_next_iteration.py --target-version 0.16.75 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.75 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.75 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `review_checklist_count` 和 `review_checklist_sha256`。
3. 确认这些摘要能从 manifest 中的 `review_checklist` 重新计算得到。
4. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 展示这些字段。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.75`。
7. 将 `pyproject.toml` 版本更新为 `0.16.75`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.75`。
9. 创建 tag：`git tag v0.16.75`。

## Release Notes 摘要

`v0.16.75` adds review-checklist count and hash fields to artifact bundle summaries, helping tools detect review drift before reading full issue artifact manifests.
