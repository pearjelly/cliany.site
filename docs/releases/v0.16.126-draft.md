# v0.16.126 发布草案

**目标版本：** `0.16.126`
**提交范围：** `v0.16.125..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本修正 quick summary inline code fence 在 summary 含反引号时的渲染边界。

## 用户价值

`v0.16.125` 已经让 candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 展示 `visibility_summary`。当 visibility summary 是 `local_only` 这类包含 `` `master` `` 和 `` `origin` `` 的文本时，原有 `_summary_inline_code` 会直接用双反引号包裹内容；如果内容本身以反引号开头，输出会形成不易阅读的三反引号开头。

`v0.16.126` 把包含反引号的 inline code 统一渲染成带空格的双反引号形式，例如 `` `master` is ahead ``。维护者打开 README 时，可以稳定阅读 `visibility_summary`，不用猜 Markdown fence 是否吞掉了开头反引号。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.126 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 的 `visibility_summary` 在 summary 含反引号时使用清晰的 inline code fence。

## 变更分组

### Quick summary inline code fence

- `_summary_inline_code` 在内容包含反引号时使用 `` `master` is ahead `` 这种带空格的双反引号形式。
- `visibility_summary` 继续来自已有 `publication_visibility.summary`。
- 现有 quick summary 字段不改名、不删除。
- 现有 `Artifact Bundle Summary`、`Publication Handoff` 和 `publication_visibility` 结构不变。

### 文档与门禁

- `tests/test_plan_next_iteration.py` 覆盖普通含反引号文本和以反引号开头的文本。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 这是 Markdown 展示层修复，不改变 JSON schema。
- 已有不含反引号的 summary 仍使用单反引号 inline code。
- 含反引号的 summary 输出会多出前后空格，这是 CommonMark 推荐的可读 code span 形式。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.126 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。

4. 确认 summary 含反引号时使用带空格的双反引号 inline code fence。
5. 运行计划器和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.126`。
7. 将 `pyproject.toml` 版本更新为 `0.16.126`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.126`。
9. 创建 tag：`git tag v0.16.126`。

## Release Notes 摘要

`v0.16.126` stabilizes inline code rendering for quick summary values that contain backticks, keeping generated candidate issue artifacts README files readable when publication visibility summaries mention refs such as `master` and `origin`.
