# v0.16.25 发布草案

**目标版本：** `0.16.25`
**提交范围：** `v0.16.24..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本让候选 issue artifacts README 的审阅清单明确检查 publication next actions。

## 用户价值

`v0.16.24` 已经把 `Publication Next Actions` 展示在 `scripts/plan_next_iteration.py --issues-dir` 生成的 artifacts `README.md` 里。维护者能看到发布阻塞，但执行 `create-issues.sh` 前的 `Review Checklist` 还没有明确要求先处理或有意延后这些 publication next actions。

`v0.16.25` 把这个动作放进 `Review Checklist`：运行 `create-issues.sh` 前，维护者需要确认 `Publication Next Actions` are resolved or intentionally deferred。这样候选 issue 派发前的人工检查顺序更贴近项目策略：先确认最新 release 是否公开可见，再扩大 candidate 工作。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.25 --issues-dir /tmp/cliany-candidate-issues --json` 写出的 `README.md` 包含 `Review Checklist`、`Publication Next Actions`、`resolved or intentionally deferred` 和 `create-issues.sh`；`publication-handoff.json` 继续保留结构化 publication next actions。

## 变更分组

### Candidate issue artifacts review gate

- `scripts/plan_next_iteration.py --issues-dir` 生成的 artifacts `README.md` 在 `Review Checklist` 中新增 publication next actions 检查项。
- 检查项要求维护者在运行 `create-issues.sh` 前确认 `Publication Next Actions` 已处理或明确延后。
- `publication-handoff.json`、`create-issues.sh` 和候选 issue body 文件结构保持兼容。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifacts README 审阅清单会要求先确认 publication next actions。
- `tests/test_plan_next_iteration.py` 固定 artifacts README 输出。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，也不把 candidate 标记为 active；它只让候选 issue artifacts 的人工审阅步骤更明确。

## 风险与兼容性

- 这是 artifacts README 文案增强，不改变 JSON schema 和脚本执行行为。
- 不自动执行 push，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。
- artifacts 继续写入维护者指定的 `--issues-dir`，不会写入 `~/.cliany-site/` 或仓库运行时状态。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.25 --issues-dir /tmp/cliany-candidate-issues --json` 写出的 README 包含 `Review Checklist`。
2. 确认清单要求 `Publication Next Actions` are resolved or intentionally deferred before running `create-issues.sh`。
3. 确认 `publication-handoff.json` 继续包含 `publication_next_actions`。
4. 运行计划脚本、维护者循环文档和 release draft 相关测试。
5. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.25`。
6. 将 `pyproject.toml` 版本更新为 `0.16.25`。
7. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.25`。
8. 创建 tag：`git tag v0.16.25`。

## Release Notes 摘要

`v0.16.25` adds a review checklist gate to generated candidate issue artifacts README files, asking maintainers to resolve or intentionally defer publication next actions before running `create-issues.sh`.
