# v0.16.116 发布草案

**目标版本：** `0.16.116`
**提交范围：** `v0.16.115..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 publication branch 补进 candidate issue gate quick summary。

## 用户价值

`v0.16.115` 已经让 candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 展示 publication worktree clean 状态。维护者可以先判断工作区是否干净，但仍需要展开 `Publication Handoff` 或完整 `Artifact Bundle Summary` 才能确认当前 artifacts 指向哪条发布分支。

`v0.16.116` 在 `Candidate Issue Gate Quick Summary` 中新增 `publication_branch`。维护者打开 README 时，可以先确认待发布分支，再结合 latest tag、ahead/behind 数和 worktree clean 状态决定是否继续执行发布脚本。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.116 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 README 中，`Candidate Issue Gate Quick Summary` 出现在 `Artifact Bundle Summary` 之前，并展示 `publication_branch`、`publication_worktree_clean` 和 `publication_ahead_count`。

## 变更分组

### Candidate issue gate quick summary branch

- Candidate issue artifacts `README.md` 的 `Candidate Issue Gate Quick Summary` 新增 `publication_branch`。
- 该字段来自已有 candidate issue gate evidence。
- 现有 quick summary 的 status、can_create_issues、requires_maintainer_review、reason/action count、primary reason/action、latest tag、publication worktree clean、publication ahead/behind count、publication remote checked、release draft issue count、release draft path 和 visibility 保持不变。
- 现有 `artifact_bundle_summary` 字段保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 quick summary 会展示 publication branch。
- `tests/test_plan_next_iteration.py` 覆盖 README quick summary 的 publication branch 行。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只改善候选 issue artifacts 的维护者阅读路径。

## 风险与兼容性

- 新增的 README 行只读取已有 candidate issue gate evidence。
- `Artifact Bundle Summary` 字段不删除、不改名，机器读取路径保持兼容。
- `publication_branch` 展示当前 publication audit 识别的分支名，帮助维护者在执行 push 前确认上下文。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版前验证

发版前目标版本自检会检查这段锚点：**提交范围：** `v0.16.116..HEAD`。最终带 `--release-tag v0.16.116` 的 strict gate 仍以本草案顶部的 `v0.16.115..HEAD` 作为实际 release diff 范围。

```bash
pytest tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check scripts/plan_next_iteration.py tests/test_plan_next_iteration.py tests/test_weekly_maintainer_loop_docs.py tests/test_release_draft_docs.py
python scripts/plan_next_iteration.py --target-version 0.16.116 --json
python scripts/plan_next_iteration.py --target-version 0.16.116 --issues-dir /tmp/cliany-candidate-issues --json
python scripts/release_readiness.py --target-version 0.16.116 --json
```

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.116 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 artifacts `README.md` 包含 `Candidate Issue Gate Quick Summary`。
3. 确认该段落出现在 `Artifact Bundle Summary` 之前。
4. 确认该段落展示 `publication_branch`。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.116`。
7. 将 `pyproject.toml` 版本更新为 `0.16.116`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.116`。
9. 创建 tag：`git tag v0.16.116`。

## Release Notes 摘要

`v0.16.116` adds publication branch to the Candidate Issue Gate Quick Summary in generated candidate issue artifacts README files, helping maintainers confirm the branch context before publishing.
