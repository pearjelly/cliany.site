# v0.15.7 发布草案

**目标版本：** `0.15.7`
**提交范围：** `v0.15.6..HEAD`
**当前阻塞：** 需要提交 case report 的 Candidate Promotion Tasks 输出、文档入口和验证测试。

## 用户价值

`v0.15.7` 聚焦真实案例库的维护效率。`v0.15.6` 已经把 candidate 晋级拆成 `adapter_package`、`metadata_validation`、`online_smoke` 三类子任务，但维护者仍需要手工从 `cases/manifest.json` 或 `cases/README.md` 复制字段，才能创建 GitHub issue。

成功标准是：`python scripts/validate_cases.py --report /tmp/cliany-case-catalog-report.md` 生成的 Markdown 报告包含 `Candidate Promotion Tasks` 小节；每个 candidate 都有三条可复制任务、对应的验收证据提示，以及 `data.quality.ok=true` / `row_count>0` 的在线只读 smoke 质量要求。

## 变更分组

### Case report issue-ready tasks

- `scripts/validate_cases.py --report` 在案例表格后输出 `Candidate Promotion Tasks` 小节。
- 每个 candidate 的 `adapter_package`、`metadata_validation`、`online_smoke` 都渲染成 checkbox 任务。
- online smoke 任务明确要求粘贴只读 JSON envelope 摘要，并确认 `data.quality.ok=true` 和 `row_count>0`。

### 文档与门禁

- [cases/README.md](../../cases/README.md) 说明 case catalog report 可直接生成 candidate promotion issue 草稿。
- [docs/good-first-issues.md](../good-first-issues.md) 增加把 `case-catalog-report` 中的 `Candidate Promotion Tasks` 复制成 GitHub issue 的候选任务。
- `tests/test_validate_cases.py` 应检查 Markdown report 包含 issue-ready 任务和质量证据提示。
- `tests/test_good_first_issues_docs.py` / `tests/test_contributor_docs.py` 应保留报告入口。

## 案例库映射

本版本不新增案例数量，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。Active 案例保持 4 个，candidate 案例保持 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 三个；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是 report artifact 和文档入口的 patch，不改变 CLI runtime、adapter schema、JSON envelope 或 `validate_cases.py --json` 输出结构。
- 默认验证继续离线运行；`online_smoke` 仍只是维护者手动晋级 active 前需要收集的证据。
- `Candidate Promotion Tasks` 只帮助创建 issue，不自动访问第三方站点、不发布 adapter 包。


## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.15.7 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行 case report、Good First Issues 和 release draft 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.15.7`。
4. 将 `pyproject.toml` 版本更新为 `0.15.7`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.15.7`。
6. 创建 tag：`git tag v0.15.7`。

## Release Notes 摘要

`v0.15.7` makes the case catalog report issue-ready for candidate promotion. Maintainers can now run `scripts/validate_cases.py --report` and copy package, metadata validation, and online smoke tasks directly into GitHub issues.
