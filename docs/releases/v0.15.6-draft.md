# v0.15.6 发布草案

**目标版本：** `0.15.6`
**提交范围：** `v0.15.5..HEAD`
**当前阻塞：** 需要提交 candidate 晋级任务拆分、文档测试和 release readiness 检查。

## 用户价值

`v0.15.6` 聚焦真实案例库的可贡献性。当前 `cases/manifest.json` 已经为 `pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 声明了 `promotion` 清单，但贡献者从 `docs/good-first-issues.md` 进入时仍容易把“晋级 active”理解成一个需要一次性完成的大任务。

成功标准是：`cases/README.md` 明确把 candidate 晋级拆成 `adapter_package`、`metadata_validation`、`online_smoke` 三个可独立验收的 issue；`docs/good-first-issues.md` 提供对应的 Good First Issues 入口；文档测试固定这些入口，避免下一轮维护时又退回笼统描述。

## 变更分组

### Candidate promotion 拆分

- [cases/README.md](../../cases/README.md) 新增 Candidate 晋级任务拆分，说明三个子任务的适合贡献者、完成证据和保持 `candidate` 的边界。
- [docs/good-first-issues.md](../good-first-issues.md) 增加 `adapter_package`、`metadata_validation`、`online_smoke` 三类 candidate 子任务，分别绑定离线或文档验证命令。

### 文档与门禁

- `tests/test_good_first_issues_docs.py` 应检查 Good First Issues 保留三类 candidate promotion 子任务和本地验证命令。
- `tests/test_contributor_docs.py` 应检查贡献者入口仍然保留 candidate promotion 的三个字段。
- `tests/test_release_draft_docs.py` 应检查本草案追踪 candidate promotion 拆分。

## 案例库映射

本版本不新增案例数量，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json) 作为真实案例入口。Active 案例保持 4 个，candidate 案例保持 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 三个；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是贡献者文档和案例库维护路径的 patch，不改变 CLI runtime、adapter schema 或 JSON envelope。
- 新增任务拆分不承诺 candidate 已经具备 release adapter 包；没有完成 `adapter_package`、`metadata_validation` 和 `online_smoke` 前，案例仍保持 `candidate`。
- 所有默认验证继续离线运行，不需要真实 LLM key，也不访问第三方站点。

## 发版前验证

```bash
pytest tests/test_good_first_issues_docs.py tests/test_contributor_docs.py tests/test_release_draft_docs.py -q --no-cov
ruff check tests/test_good_first_issues_docs.py tests/test_contributor_docs.py tests/test_release_draft_docs.py
python scripts/validate_cases.py --strict
python scripts/release_readiness.py --target-version 0.15.6 --json
```

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.15.6 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行 Good First Issues、贡献者文档和 release draft 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.15.6`。
4. 将 `pyproject.toml` 版本更新为 `0.15.6`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.15.6`。
6. 创建 tag：`git tag v0.15.6`。

## Release Notes 摘要

`v0.15.6` turns candidate case promotion into three contributor-sized tasks: adapter package creation, offline metadata validation, and online read-only smoke evidence. This makes the real-case backlog easier to split into issues without weakening the candidate/active boundary.
