# v0.16.107 发布草案

**目标版本：** `0.16.107`
**提交范围：** `v0.16.106..HEAD`
**当前阻塞：** 最新本地 release 仍需维护者确认后 push 到远端；本版本把 candidate issue gate 的首要原因和首个动作补进 candidate issue artifacts 的整包摘要。

## 用户价值

`v0.16.106` 已经让 `artifact_bundle_summary` 能展示 remote check 看到的远端分支和 tag commit。维护者继续审阅候选 issue artifacts 时，除了知道 gate reason/action set 是否漂移，还需要不展开完整 `candidate_issue_gate` 就能直接展示“首要阻塞原因”和“第一个维护动作”。

`v0.16.107` 在 `artifact_bundle_summary` 中新增 `candidate_issue_gate_primary_reason_code` 和 `candidate_issue_gate_primary_required_action`。当 `candidate_issue_gate.reason_codes` 或 `candidate_issue_gate.required_actions` 非空时，这两个字段分别取列表第一项；列表为空时保持 `null`/`None`，避免工具把空 gate 误判成需要执行动作。

成功标准是：`python scripts/plan_next_iteration.py --target-version 0.16.107 --issues-dir /tmp/cliany-candidate-issues --json` 生成的 `artifact-manifest.json.artifact_bundle_summary` 包含 `candidate_issue_gate_primary_reason_code` 和 `candidate_issue_gate_primary_required_action`，且这些值分别等于 manifest 中 `candidate_issue_gate.reason_codes[0]` 和 `candidate_issue_gate.required_actions[0]`；README 的 `Artifact Bundle Summary` 也展示这些字段。

## 变更分组

### Artifact bundle gate primary summary

- `artifact_bundle_summary` 新增 `candidate_issue_gate_primary_reason_code`。
- `artifact_bundle_summary` 新增 `candidate_issue_gate_primary_required_action`。
- Candidate issue artifacts `README.md` 的 `Artifact Bundle Summary` 展示首要 gate reason 和首个 required action。
- 现有 `candidate_issue_gate_reason_code_count`、`candidate_issue_gate_reason_codes_sha256`、`candidate_issue_gate_required_action_count` 和 `candidate_issue_gate_required_actions_sha256` 保持不变。

### 文档与门禁

- [docs/weekly-maintainer-loop.md](../weekly-maintainer-loop.md) 说明 artifact bundle summary 会携带 gate primary reason/action。
- `tests/test_plan_next_iteration.py` 覆盖 manifest 和 README。
- `tests/test_weekly_maintainer_loop_docs.py` 固定维护者循环说明。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不改变案例库状态，继续依赖 [cases/README.md](../../cases/README.md) 和 [cases/manifest.json](../../cases/manifest.json)。`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate cases；`search-extraction-gap` 继续作为抽取质量 known-gap。

本版本不创建真实 GitHub issue，不创建真实 GitHub Release，不推送 tag，也不把 candidate 标记为 active；它只让候选 issue artifacts 的整包摘要更容易被自动化工具读取。

## 风险与兼容性

- 新字段只来自已有 `candidate_issue_gate.reason_codes` 和 `candidate_issue_gate.required_actions`。
- 当 gate 允许创建 issue 且列表为空时，新字段为 `null`/`None`。
- 字段值可能包含维护者动作文本中的反引号；README 继续使用 Markdown code span 展示。
- 不访问 GitHub，不创建远端 tag，不访问第三方业务站点，不使用真实 LLM key。

## 发版步骤

1. 确认 `python scripts/plan_next_iteration.py --target-version 0.16.107 --issues-dir /tmp/cliany-candidate-issues --json` 生成 artifacts。
2. 确认 `artifact-manifest.json.artifact_bundle_summary` 包含 `candidate_issue_gate_primary_reason_code` 和 `candidate_issue_gate_primary_required_action`。
3. 确认这些值分别等于 `artifact-manifest.json.candidate_issue_gate.reason_codes[0]` 和 `artifact-manifest.json.candidate_issue_gate.required_actions[0]`。
4. 确认 artifacts `README.md` 的 `Artifact Bundle Summary` 展示这些字段。
5. 运行计划器、维护者循环文档和 release draft 相关测试。
6. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.107`。
7. 将 `pyproject.toml` 版本更新为 `0.16.107`。
8. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.107`。
9. 创建 tag：`git tag v0.16.107`。

## Release Notes 摘要

`v0.16.107` adds primary candidate issue gate reason/action fields to artifact bundle summaries, helping tools display the first blocker and next maintainer action without expanding the full gate object.
