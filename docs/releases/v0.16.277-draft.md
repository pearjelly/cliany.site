# v0.16.277 发布草案

**目标版本：** `0.16.277`
**提交范围：** `v0.16.276..HEAD`
**发布主题：** 让生成的浏览器命令完整重放 `select` / `submit`，并在写入 adapter 前以真实提取证据阻止数据命令的假完成。

完成 version bump 后，target-mode readiness 还会检查占位锚点 **提交范围：** `v0.16.277..HEAD`；该锚点不改变实际 release diff。

## 用户价值

`explore` 以前可以建议带有 `select` 或 `submit` 的工作流，但 generated adapter 的原子命令路径没有对应实现；执行时会落入未知动作。更重要的是，提示词虽然要求 `list-`、`search-`、`read-` 与 `extract-` 命令拥有 `extract` action，LLM 仍可能把纯点击或其他命令的提取结果当成某个数据命令的完成证据。

本版补齐语义化 `browser select` 与 `browser submit` 原子命令，并在探索完成前按命令自己的 `action_steps` 对齐真实 extraction result。命名数据命令必须拥有自己的 extract action 与可接受的提取质量；默认空结果、缺字段、partial 行、缺证据和执行失败都不能生成 adapter。允许合法零匹配的 `expects_nonempty=false` 只接受真实 empty extraction，绝不放过 partial 或缺失字段。

第一次不满足门禁时，explorer 只进行一次明确的修正回合；仍失败时 CLI 与 SDK 返回带 `data_commands` 详情的 `E_EMPTY_RESULT`，不会写入 action-only adapter。这个版本不运行 live LLM、不生成外部 adapter package，也不声称任何第三方站点产生了在线结果。

## 变更分组

### Generated browser actions

- 新增 `cliany-site browser select`，基于 AXTree `--ref` / `--text` 解析 combobox，再使用 browser-use 的下拉选择事件。
- 新增 `cliany-site browser submit`；可先语义化聚焦输入元素，再发送 Enter。显式按钮仍由 `click` 处理，避免把 submit 误当成点击操作。
- generated adapter 的 runtime helper 现在能将 `select` 和 `submit` action 路由到上述原子命令，而不是报告未知动作。

### Data-command completion gate

- 每个 `list-`、`search-`、`read-`、`extract-` 命令只接受其自身 `action_steps` 中 extract 的证据，不能借用其他命令的结果。
- extraction evidence 保留全局 action index；`evaluate_extract_quality` 对实际 data 评估 empty、partial 与字段缺失。
- 在 `continue_on_error=True` 的探索路径中，extract evaluate 失败会被记录为证据交给 completion gate，而不是绕过门禁或生成假成功。
- SDK 旧式 error envelope 也携带可选 `details`，使程序化调用能读取同一 `data_commands` 诊断信息。

## 案例库映射

`pypi-project-search`、`npm-package-search` 和 `crates-io-crate-search` 仍是 candidate。`search-extraction-gap` 的真实性边界不变：candidate 只有在 real doctor preflight、adapter package、metadata validation 和 online smoke 证据齐全后才能晋级。

本版只提高未来 generated data command 的完成判定与动作重放完整性；不写入 `~/.cliany-site/`，不生成 session、snapshot、adapter archive 或第三方业务数据，也不将 `llm_live_preflight_not_ready` 或 `E_LLM_UNAVAILABLE` 伪装为成功。

## 风险与兼容性

- 已安装 adapter 不会被静默重写；需要重新运行 `explore` 才会得到新动作路由与 completion gate。
- `submit` 的语义限定为 Enter 型表单。历史上把 submit 用于显式按钮的工作流应在重新探索时使用 `click`。
- 数据命令会更早拒绝未验证的完成结果；这是消除假成功的有意收紧。调用方应处理 `E_EMPTY_RESULT` 与其 `data_commands` details。
- `expects_nonempty=false` 仍只代表合法零匹配，不代表浏览器执行、字段选择或页面结构已验证。

## 发版步骤

1. 在下一个允许的发版窗口完成 version bump、CHANGELOG、官网基线和本草案的 reviewed GitHub Release Notes；保持工作树干净并通过 target-mode readiness。
2. 推送最终 `master` 后，等待 GitHub CI 与 Embodied CI 都成功；任一失败都不创建 tag。
3. 仅在 CI 成功后创建本地 `git tag v0.16.277`，运行 `uv run --extra dev --frozen python scripts/release_readiness.py --strict --release-tag v0.16.277 --remote --remote-name origin`。
4. 通过 tag gate 后推送 tag，等待 GitHub Release workflow，逐版本检查 GitHub Release 与 PyPI `0.16.277`。
5. 运行 `gh release edit v0.16.277 --repo pearjelly/cliany.site --notes-file docs/releases/v0.16.277-github-release.md`，再用 `gh release view v0.16.277 --repo pearjelly/cliany.site --json body` 检查 release notes。
6. 在 `site/` 运行 `vercel link --yes --project cliany.site`、`vercel --prod --yes` 与 `vercel inspect www.cliany.site --wait --timeout 90s`，然后浏览 www.cliany.site。
7. 运行 `uv run --extra dev --frozen python scripts/check_release_publication.py --strict --remote --distribution --json`；audit 成功后才报告完成。

## Release Notes 摘要

`v0.16.277` makes generated adapters replay select and Enter-submit actions, and refuses to generate a named data command until that command owns verified extraction evidence.
