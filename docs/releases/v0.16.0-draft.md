# v0.16.0 发布草案

**目标版本：** `0.16.0`
**提交范围：** `v0.15.9..HEAD`
**当前阻塞：** 需要提交案例库结构化离线验证命令、报告输出和相关测试。

## 用户价值

`v0.16.0` 启动真实案例库一期。`cases/manifest.json` 已经有 active、candidate 和 known-gap 状态，也有离线样例输出；但 `validation.offline` 仍是自然语言说明，贡献者和维护者要从文档里猜应该运行哪条命令。这个版本把离线验证命令结构化，作为案例资产的一部分。

成功标准是：每个案例都声明 `validation.offline_commands`；`scripts/validate_cases.py` 校验这些命令是本地验证命令；`scripts/validate_cases.py --report` 输出 `Offline Validation Commands` 小节，便于 PR、CI artifact 和 release notes 复盘直接复制。

## 变更分组

### Structured offline validation

- [cases/manifest.json](../../cases/manifest.json) 为 4 个 active、3 个 candidate 和 1 个 known-gap 案例补充 `validation.offline_commands`。
- `scripts/validate_cases.py` 要求每个案例提供非空 `validation.offline_commands`。
- offline command 只允许本地验证入口，例如 `python`、`pytest`、`CLIANY_QA_OFFLINE=1` 或 `bash` 开头的命令。
- Markdown report 新增 `Offline Validation Commands` 小节。

### 文档与门禁

- [cases/README.md](../../cases/README.md) 说明结构化离线命令如何进入案例分层、维护规则和报告 artifact。
- `tests/test_cases_manifest.py` 检查真实 manifest 中每个案例都有本地离线验证命令。
- `tests/test_validate_cases.py` 检查缺失或非本地命令会被拒绝，并固定 Markdown report 输出。
- `tests/test_release_draft_docs.py` 固定本草案。

## 案例库映射

本版本不新增案例数量，但提高现有 8 个案例资产的可验证性。Active 案例保持 4 个，candidate 案例保持 `pypi-project-search`、`npm-package-search`、`crates-io-crate-search` 三个；`search-extraction-gap` 继续作为抽取质量 known-gap。

## 风险与兼容性

- 这是案例 manifest、报告和校验规则的 minor release，`validate_cases.py --json` 会新增 `offline_commands` 字段，但不改变现有 `cases`、`promotion` 或 package check 结构。
- 命令只用于展示和校验形状，不会由 `validate_cases.py` 自动执行。
- 默认验证继续离线运行，不访问第三方站点，不需要真实 LLM key。

```bash
pytest tests/test_cases_manifest.py tests/test_validate_cases.py tests/test_release_readiness.py tests/test_release_draft_docs.py -q --no-cov

python scripts/validate_cases.py --report /tmp/cliany-case-catalog-report.md --strict
python scripts/release_readiness.py --target-version 0.16.0 --json
```

## 发版步骤

1. 确认 `python scripts/release_readiness.py --target-version 0.16.0 --json` 只剩可解释的节奏或包资产阻塞。
2. 运行案例库、release readiness 和 release draft 相关测试。
3. 将 `CHANGELOG.md` 的 Unreleased 移动到 `0.16.0`。
4. 将 `pyproject.toml` 版本更新为 `0.16.0`。
5. 运行 `python scripts/release_readiness.py --strict --target-version 0.16.0`。
6. 创建 tag：`git tag v0.16.0`。

## Release Notes 摘要

`v0.16.0` starts the first real-case catalog iteration by making offline validation commands part of every case asset. Maintainers can now copy per-case local validation commands from the case catalog report when reviewing PRs or preparing release notes.
