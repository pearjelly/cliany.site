# ADR 0007: v0.12.0 稳定性加固迭代架构决策与脆弱点跟踪

**状态**: 已接受

**日期**: 2026-05-20

## 背景

在 v0.11.0 引入 Obscura 浏览器提供商和多平台支持后，系统在极端场景下的稳定性问题开始显现。为了确保 v0.12.0 能够达到生产级交付标准，需要对现有的 P0/P1 级安全和稳定性脆弱点进行系统性加固。

## 决策

### 1. P0 级高危脆弱点修复策略

**[T03] tar.extractall 路径穿越修复**
- **文件**: `tui/screens/adapter_list.py`
- **描述**: 在安装适配器包时，直接使用 `tar.extractall` 可能导致路径穿越攻击。
- **修复**: 引入安全检查逻辑，验证解压路径是否始终位于目标目录内。

**[T05] PID 文件写入竞态加固**
- **文件**: `binary/process.py`
- **描述**: 在多进程高频启动/停止时，PID 文件的写入和读取存在竞态条件。
- **修复**: 使用文件锁（Portalocker）确保 PID 文件的原子性读写。

**[T06] Manifest 竞态与异常处理加固**
- **文件**: `loader.py`
- **描述**: 适配器加载时 manifest 文件读写无保护，且存在 `broad except` 吞掉关键错误的问题。
- **修复**: 引入文件锁保护 manifest 读写，并将模糊的 `except Exception` 细化为具体异常。

### 2. P1 级稳定性加固策略

**[T07] 原子写增强 (fsync)**
- **文件**: `codegen/merger.py`
- **描述**: 代码合并后写入文件未调用 `fsync`，断电或崩溃可能导致文件损坏。
- **修复**: 在关键文件关闭前强制执行 `os.fsync`。

**[T08] 网络请求重试机制**
- **文件**: `commands/obscura.py`
- **描述**: 二进制文件下载等网络操作缺乏重试，在不稳定网络环境下极易失败。
- **修复**: 引入指数退避重试逻辑。

**[T09] Session 存储并发安全**
- **文件**: `session.py`, `atomic_io.py`
- **描述**: 多实例并发运行时，Session 文件的持久化存在冲突。
- **修复**: 全面接入基于文件的互斥锁。

**[T10] 链式异常透明化**
- **文件**: `explorer/interactive.py`
- **描述**: 交互式探索中的异常链被切断，导致根因难以追踪。
- **修复**: 使用 `raise ... from ...` 保持完整的异常上下文。

### 3. 依赖与集成策略

- **引入 pytest-cov (>=5.0)**: 为了量化加固效果，强制要求测试覆盖率统计。
- **引入 portalocker (>=2.8)**: 作为跨平台（Linux/macOS/Windows）文件锁的标准方案。
- **拒绝重型监控库**: 考虑到 CLI 工具的轻量化需求，明确拒绝引入 OpenTelemetry、Sentry 或 Prometheus client。
- **保留公共契约**: 严禁修改 CLI JSON 输出的 `{ok, data, error, meta}` (v1) 信封协议，确保向后兼容性。
- **保留 action_runtime.py 结构**: 暂不进行大规模重构，通过增加单元测试和集成测试来验证其稳定性。

## 后果

- **正面**: 显著提升了系统在多并发、弱网及非预期崩溃场景下的健壮性；消除了潜在的安全漏洞。
- **负面/权衡**: 引入了 `portalocker` 依赖；由于增加了锁等待和重试，部分 I/O 操作的耗时会有轻微增加。
- **跨平台验证**: 决策要求所有加固项必须在 Linux、macOS 和 Windows 三端通过 CI 验证，特别是 `portalocker` 的三平台一致性。
