# 安全漏洞报告政策 / Security Policy

我们非常重视安全问题，并致力于及时响应和修复安全漏洞。如果您发现了 cliany-site 项目中的安全漏洞，请通过 GitHub 的私有安全报告功能向我们报告。

We take security issues seriously and are committed to responding to and fixing security vulnerabilities in a timely manner. If you discover a security vulnerability in the cliany-site project, please report it to us through GitHub's private security reporting feature.

# 支持版本 / Supported Versions

我们仅对最新版本提供安全补丁支持。以下是当前支持的版本：

We only provide security patch support for the latest version. The following are the currently supported versions:

| 版本 / Version | 接受安全补丁 / Accepts Security Patches |
|----------------|------------------------------------------|
| 0.6.x（最新） / 0.6.x (Latest) | ✅ |
| < 0.6.0 / < 0.6.0 | ❌ |

# 报告漏洞 / Reporting a Vulnerability

请通过 GitHub Security Advisories 私下报告安全漏洞。请勿在公开的 Issue 或 Pull Request 中披露安全漏洞详情。

Please report security vulnerabilities privately through GitHub Security Advisories. Do not disclose security vulnerability details in public Issues or Pull Requests.

报告链接 / Report Link: https://github.com/pearjelly/cliany.site/security/advisories/new

# 处理流程 / Process

我们将按照以下流程处理安全漏洞报告：

We will handle security vulnerability reports according to the following process:

1. **确认收到 / Acknowledgment**: 我们将确认收到您的报告。
2. **评估严重性 / Severity Assessment**: 我们将评估漏洞的严重性和影响。
3. **修复 / Fix**: 我们将开发并测试修复方案。
4. **发布安全版本 / Security Release**: 我们将发布包含修复的安全版本。
5. **公开披露 / Public Disclosure**: 在修复发布后，我们将公开披露漏洞详情。
